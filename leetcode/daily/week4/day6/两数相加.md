# 两数相加

- **题目名称**：两数相加
- **链接**：[2. 两数相加](https://leetcode.cn/problems/add-two-numbers/)
- **难度**：中等
- **标签**：递归、链表、数学

## 1. 题目概述

给你两个**非空**的链表，表示两个非负整数。每位数字都是按照**逆序方式存储**的，并且每个节点只能存储**一位数字**。请将两个数相加，并以相同形式返回一个表示和的链表。

**示例 1**：

```text
输入：l1 = [2,4,3], l2 = [5,6,4]
输出：[7,0,8]
解释：342 + 465 = 807，返回链表 7 → 0 → 8。
```

**示例 2**：

```text
输入：l1 = [0], l2 = [0]
输出：[0]
```

**示例 3**：

```text
输入：l1 = [9,9,9,9,9,9,9], l2 = [9,9,9,9]
输出：[8,9,9,9,0,0,0,1]
解释：9999999 + 9999 = 10009998。
```

**约束条件**：

- 每个链表中的节点数在范围 `[1, 100]` 内
- `0 <= Node.val <= 9`
- 题目数据保证链表表示的数字没有前导零

> 💡 这是一道链表题，但本质是**模拟竖式加法**。逆序存储是关键——链表头是个位，从左到右正好对应"从低位到高位"的加法顺序，无需反转链表即可逐位相加。核心是处理好**进位（carry）**，尤其是两链表长度不等、以及最高位产生新进位的边界情况。

---

## 2. 解题思路

### 2.1 暴力思路：转数字相加再转链表（会溢出）

最直白的想法：遍历两条链表还原成整数 `a` 和 `b`，相加得 `sum = a + b`，再把 `sum` 逐位拆解成新链表。

```text
a = 0; 遍历 l1: a = a * 10 + val   # 注意逆序，要倒推权重
b = 0; 遍历 l2: b = b * 10 + val
sum = a + b
if sum == 0: return ListNode(0)
while sum > 0:
    append(sum % 10); sum //= 10
```

> ⚠️ 致命缺陷：链表最长 100 位，而 `int64` 最多容纳约 19 位十进制数，`Python` 的大整数虽不溢出，但 `C++/Java` 必然溢出。即使题目保证 `Python` 不溢出，"转数字"也偏离了**链表逐节点操作**的本意，面试官不会认可。正确做法是直接在链表上模拟。

### 2.2 核心观察：模拟竖式加法

既然链表**逆序存储**（头节点是个位），那么从两个头节点开始同步向后遍历，就等同于"从个位到最高位"做竖式加法。每一位的运算规则：

- 当前位的和 = `l1 当前位 + l2 当前位 + carry（来自低位的进位）`
- 新节点值 = `sum % 10`
- 向高位的进位 = `sum / 10`（下一位带上）

![模拟竖式加法：逐位相加，进位向高位传递](images/add_two_numbers_column_addition.svg)

上图展示了示例 1 的逐位运算：

- 个位：`2 + 5 + 0 = 7`，写 7 进 0
- 十位：`4 + 6 + 0 = 10`，写 0 进 1
- 百位：`3 + 4 + 1 = 8`，写 8 进 0
- 结果链表：`7 → 0 → 8`

> 💡 **关键洞察**：逆序存储天然契合"从低位到高位"的加法方向——这正是本题把链表题降维成"逐节点扫描"的根本原因。若改为正序存储（如 [445 两数相加 II](https://leetcode.cn/problems/add-two-numbers-ii/)），则需要先反转链表或用栈，难度立刻上升一档。

**两个边界细节**：

1. **长度不等**：较短的链表遍历完后，其对应位按 `0` 参与运算，长链表剩余位继续与 carry 相加。
2. **最高位仍有进位**：两链表都遍历完，但 `carry == 1` 时，需额外追加一个值为 1 的节点（示例 3 末尾的 `1` 即此情况）。

### 2.3 算法流程图

![算法流程：哑节点起步，逐位相加取模，carry 收尾追加](images/add_two_numbers_algorithm_flow.svg)

**完整步骤**：

1. **建哑节点**：`dummy` 作为结果链表头前的哨兵，`tail = dummy` 指向结果链表当前末尾。
2. **初始化进位**：`carry = 0`。
3. **同步遍历**：`while l1 != null or l2 != null or carry != 0`：
   - 取 `x = l1 ? l1.val : 0`，`y = l2 ? l2.val : 0`
   - `sum = x + y + carry`
   - 新建节点 `tail.next = ListNode(sum % 10)`，`tail = tail.next`
   - `carry = sum / 10`
   - `l1`、`l2` 非空则各前进一位
4. **返回**：`dummy.next`。

> ⚠️ 循环条件里**带上 `carry != 0`** 是最容易漏的细节。若只写 `while l1 or l2`，示例 3 的最高位进位 `1` 会被丢弃，结果少一位。把 carry 纳入循环条件可统一处理"末尾追加进位"的边界，无需在循环外再补一个 `if`。

### 2.4 示例演算

以 `l1 = [9,9,9,9,9,9,9]`，`l2 = [9,9,9,9]`（示例 3）为例：

![示例演算：逐位相加，最终 carry=1 追加最高位节点](images/add_two_numbers_example_walkthrough.svg)

| 步骤 | l1 当前 | l2 当前 | carry（入） | sum | 新节点 | carry（出） |
|------|---------|---------|------------|-----|--------|------------|
| 1 | 9 | 9 | 0 | 18 | 8 | 1 |
| 2 | 9 | 9 | 1 | 19 | 9 | 1 |
| 3 | 9 | 9 | 1 | 19 | 9 | 1 |
| 4 | 9 | 9 | 1 | 19 | 9 | 1 |
| 5 | 9 | 0（已尽） | 1 | 10 | 0 | 1 |
| 6 | 9 | 0 | 1 | 10 | 0 | 1 |
| 7 | 9 | 0 | 1 | 10 | 0 | 1 |
| 8 | 0（已尽） | 0 | 1 | 1 | 1 | 0 |

最终结果链表：`8 → 9 → 9 → 9 → 0 → 0 → 0 → 1`。

> 💡 第 5~7 步体现了"短链表已尽按 0 参与运算"；第 8 步两链表都尽但 `carry = 1`，因循环条件含 `carry != 0` 而追加最后一个节点 1。这正是题目最长用例的边界，务必覆盖。

## 3. 参考代码

### C++

```cpp
// 两数相加.cpp —— 模拟竖式加法 + 哑节点
// 编译: g++ -O2 -std=c++17 两数相加.cpp -o add_two_numbers
struct ListNode {
    int val;
    ListNode* next;
    ListNode() : val(0), next(nullptr) {
    }
    ListNode(int x) : val(x), next(nullptr) {
    }
    ListNode(int x, ListNode* n) : val(x), next(n) {
    }
};

class Solution {
  public:
    ListNode* addTwoNumbers(ListNode* l1, ListNode* l2) {
        ListNode dummy(0); // 哑节点，统一尾部追加逻辑
        ListNode* tail = &dummy;
        int carry = 0;

        // 循环条件带 carry，统一处理"最高位仍有进位"的边界
        while (l1 != nullptr || l2 != nullptr || carry != 0) {
            int x = (l1 != nullptr) ? l1->val : 0;
            int y = (l2 != nullptr) ? l2->val : 0;
            int sum = x + y + carry;

            carry = sum / 10;                    // 向高位的进位
            tail->next = new ListNode(sum % 10); // 当前位写入新节点
            tail = tail->next;

            if (l1 != nullptr)
                l1 = l1->next;
            if (l2 != nullptr)
                l2 = l2->next;
        }

        return dummy.next;
    }
};
```

### Python

```python
from typing import Optional

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

class Solution:
    def addTwoNumbers(self, l1: Optional[ListNode], l2: Optional[ListNode]) -> Optional[ListNode]:
        dummy = ListNode(0)                 # 哑节点，统一尾部追加逻辑
        tail = dummy
        carry = 0

        # 循环条件带 carry，统一处理"最高位仍有进位"的边界
        while l1 or l2 or carry:
            x = l1.val if l1 else 0
            y = l2.val if l2 else 0
            s = x + y + carry

            carry = s // 10                 # 向高位的进位
            tail.next = ListNode(s % 10)    # 当前位写入新节点
            tail = tail.next

            if l1:
                l1 = l1.next
            if l2:
                l2 = l2.next

        return dummy.next
```

> 💡 两种语言结构完全对应。注意 C++ 用 `sum / 10` 和 `sum % 10`（整数除法），Python 用 `//` 和 `%`。`carry` 初始为 0，循环结束后必为 0（因为循环条件保证），无需额外清理。

## 4. 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| **时间复杂度** | `O(max(m, n))` | 逐位遍历，循环次数 = 较长链表长度 + 1（可能的最高位进位），`m`、`n` 分别为两链表长度 |
| **空间复杂度** | `O(max(m, n))` | 结果链表长度最多 `max(m, n) + 1`，除返回链表外只用 `O(1)` 辅助变量 |

> ⚠️ 注意空间复杂度含结果链表本身。若按"除答案外额外空间"口径，则是 `O(1)`。面试时说清口径即可，通常写作 `O(max(m, n))`。

## 5. 扩展：445 两数相加 II（正序存储）

[445 题](https://leetcode.cn/problems/add-two-numbers-ii/) 把存储顺序改为**正序**（头节点是最高位），要求同样返回正序链表。难度立刻上升：加法必须从最低位开始，但链表头却是最高位。

**两种通用解法**：

| 解法 | 思路 | 复杂度 |
|------|------|--------|
| **反转链表** | 先反转 `l1`、`l2` 化为本题（逆序），相加后再反转结果 | `O(m + n)` 时间，`O(1)` 额外空间 |
| **双栈模拟** | 两条链表各压栈，出栈顺序即从低位到高位，逐位相加后头插法建结果 | `O(m + n)` 时间，`O(m + n)` 空间 |

> 💡 本题（逆序）是 445 题（正序）的**简化版**——逆序存储省去了"先反转/压栈对齐低位"的步骤，直接顺序扫描即可。面试中若被问 445，先说"本题（逆序）做法"，再说"正序需先对齐低位，反转或入栈皆可"，体现从简单到复杂的递进思维。两题共用同一套"逐位相加 + carry"的内核，差别仅在"如何从低位开始遍历"。

## 6. 面试要点

1. **为什么循环条件要带上 `carry != 0`？**

   - 若只写 `while l1 or l2`，当两链表都遍历完但仍有进位（如示例 3 的最高位 `1`）时，循环已退出，进位被丢弃，结果少一位。
   - 带上 `carry != 0` 后，循环会多执行一次，把最后的进位作为新节点追加。这统一了"末尾进位"的边界处理，无需在循环外补 `if (carry) tail.next = ...`，代码更简洁且不易漏。

2. **两链表长度不等时怎么处理？**

   - 较短链表遍历完后，其当前位按 `0` 参与运算：`x = l1 ? l1.val : 0`。
   - 长链表剩余位继续与 `carry` 相加，逐位生成新节点。
   - 这是竖式加法的自然延伸——短的数高位补零即可，无需特判。

3. **为什么用哑节点（dummy）？**

   - 哑节点统一了"首节点的创建"和"后续节点的追加"逻辑：每步都是 `tail.next = new ListNode(...)`，无需为第一个节点单独 `if`。
   - 最终返回 `dummy.next`，避免维护"head 是否已赋值"的状态变量。这是链表构建题的通用模板。

4. **进位 carry 何时归零？如何保证不会漏掉？**

   - `carry = sum / 10`，最大 `9 + 9 + 1 = 19`，故 `carry ∈ {0, 1}`，不会超过 1。
   - 循环条件含 `carry != 0`，只要还有进位就继续循环，最后一次循环 `sum = carry`（两位链表已尽，`x = y = 0`），`carry = sum / 10 = 0`，循环自然终止。数学上保证不会漏。

5. **如果链表是正序存储（445 题）怎么做？**

   - 加法必须从最低位开始，正序存储的头是最高位，方向相反。
   - 解法一：先反转 `l1`、`l2` 化为本题，相加后反转结果，`O(1)` 额外空间。
   - 解法二：两条链表各入栈，出栈即从低位到高位，逐位相加后用头插法建结果链表，`O(m + n)` 空间但代码更直观。
   - 内核仍是"逐位相加 + carry"，差别仅在"如何从低位开始遍历"。

> 💡 **一句话总结**：两数相加是"链表模拟竖式加法"的招牌题——逆序存储让加法方向与链表遍历方向天然一致，哑节点统一尾部追加，循环条件带 `carry` 统一末尾进位边界。`O(max(m, n))` 时间、`O(max(m, n))` 空间。核心模板是"逐位相加取模、进位向高位传递"，掌握后可迁移到 445 正序版、字符串相加（415）、字符串相乘（43）等所有"逐位运算"类问题。
