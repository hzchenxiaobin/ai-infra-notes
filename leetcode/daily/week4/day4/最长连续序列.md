# 最长连续序列

- **题目名称**：最长连续序列
- **链接**：[128. 最长连续序列](https://leetcode.cn/problems/longest-consecutive-sequence/)
- **难度**：中等
- **标签**：并查集、数组、哈希表

## 1. 题目概述

给定一个**未排序**的整数数组 `nums`，找出数字连续的最长序列（不要求在原数组中连续）的长度。要求算法的时间复杂度为 **`O(n)`**。

**示例 1**：

```text
输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。
```

**示例 2**：

```text
输入：nums = [0,3,7,2,5,8,4,6,0,1]
输出：9
```

**约束条件**：

- `0 <= nums.length <= 10^5`
- `-10^9 <= nums[i] <= 10^9`

> ⚠️ **关键约束**：题目明确要求 `O(n)` 时间复杂度，这排除了直接排序（`O(n log n)`）的常规做法，必须借助**哈希表**来换取线性时间。

---

## 2. 解题思路

### 2.1 暴力思路：排序 O(n log n)，不满足要求

最直观的做法是先对数组**排序**，再线性扫描相邻元素统计连续段长度：

```text
排序后：1, 2, 3, 4, 100, 200  →  连续段 [1,2,3,4] 长度 4
```

时间复杂度 `O(n log n)`，空间复杂度 `O(log n)`（排序栈）。虽然能 AC，但**不满足题目 O(n) 的硬性要求**，面试中若被追问"能否 O(n)？"就要换思路。

### 2.2 核心观察：哈希集合 + 序列起点判断

把所有数放入**哈希集合**（`unordered_set` / `set`），即可在 `O(1)` 时间内判断某个数是否存在。关键观察是：

> 💡 **只有"序列起点"才需要向右扩展计数**。如果 `num - 1` 也在集合中，那么 `num` 一定属于某个更长的序列，从 `num` 开始计数是重复劳动，跳过即可。

![哈希集合与序列起点判断](images/longest_consecutive_overview.svg)

上图展示了示例 1 的判定过程：

- `100`：`99` 不在集合中 → 是起点 → 向右找 `101`，不存在 → 长度 1。
- `4`：`3` 在集合中 → **跳过**（不是起点）。
- `200`：`199` 不在集合中 → 是起点 → 长度 1。
- `1`：`0` 不在集合中 → 是起点 → `2,3,4` 都在 → 长度 4 ✓。
- `3`、`2`：前驱都在集合中 → **跳过**。

**为什么是 O(n)？** 虽然外层遍历 `n` 个元素，内层 `while` 看似会嵌套，但每个元素**至多被作为起点扩展一次**（只有起点会触发内层循环，且 `num+1`、`num+2`… 这些后续元素永远不会再次成为起点，因为它们的 `num-1` 都在集合中）。因此所有内层 `while` 的总步数上界为 `n`，整体 `O(n)`。

> ⚠️ **常见误区**：如果不加 `num - 1` 的起点判断，直接对每个元素向右扩展，最坏情况（如 `1,2,3,...,n`）会退化为 `O(n^2)`。

### 2.3 算法流程图

![哈希集合解法流程图](images/longest_consecutive_flow.svg)

流程要点：

1. **建集合**：把 `nums` 全部放入哈希集合，顺便去重。
2. **遍历集合**（遍历去重后的元素可减少外层次数）。
3. **起点判定**：若 `num - 1` 在集合中 → 跳过；否则进入扩展。
4. **向右扩展**：从 `num` 开始，每次 `cur + 1`，统计序列长度。
5. **更新答案**：`ans = max(ans, length)`。

### 2.4 示例演算

以 `nums = [100,4,200,1,3,2]` 为例，集合为 `{1,2,3,4,100,200}`：

| 当前 num | num-1 在集合？ | 是否起点 | 向右扩展 | 当前长度 | ans |
|----------|----------------|----------|----------|----------|-----|
| 100      | 否 (99)        | ✓        | 101 ✗    | 1        | 1   |
| 4        | 是 (3)         | ✗ 跳过   | —        | —        | 1   |
| 200      | 否 (199)       | ✓        | 201 ✗    | 1        | 1   |
| 1        | 否 (0)         | ✓        | 2→3→4→5✗ | 4        | 4   |
| 3        | 是 (2)         | ✗ 跳过   | —        | —        | 4   |
| 2        | 是 (1)         | ✗ 跳过   | —        | —        | 4   |

最终返回 `ans = 4`。

## 3. 参考代码

### C++

```cpp
class Solution {
public:
    int longestConsecutive(vector<int>& nums) {
        unordered_set<int> num_set(nums.begin(), nums.end());
        int ans = 0;

        for (int num : num_set) {
            // 只有当 num-1 不在集合中时，num 才是某个连续序列的起点
            if (num_set.find(num - 1) == num_set.end()) {
                int cur = num;
                int length = 1;

                while (num_set.find(cur + 1) != num_set.end()) {
                    cur += 1;
                    length += 1;
                }

                ans = max(ans, length);
            }
        }

        return ans;
    }
};
```

### Python

```python
class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        num_set = set(nums)
        ans = 0

        for num in num_set:
            # 只有当 num-1 不在集合中时，num 才是某个连续序列的起点
            if num - 1 not in num_set:
                cur = num
                length = 1

                while cur + 1 in num_set:
                    cur += 1
                    length += 1

                ans = max(ans, length)

        return ans
```

## 4. 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| 时间复杂度 | O(n) | 建集合 O(n)；外层遍历去重后的元素，内层 `while` 总扩展步数上界为 n，每个元素至多被起点扩展一次 |
| 空间复杂度 | O(n) | 哈希集合存储所有元素，最坏情况下全部互不相同 |

> 💡 **关键证明**：设元素 `x` 被内层 `while` 访问（即作为某个起点的 `cur` 被检查过）。由于 `x` 是从某个起点 `s` 逐步 `+1` 走到的，说明 `s < x` 且 `s..x` 都在集合中，因此 `x - 1` 一定在集合中 → `x` 在外层循环中会被跳过（不触发新的内层循环）。这意味着内层 `while` 检查过的元素与外层真正扩展的元素**不重叠**，总访问次数 ≤ `2n`。

## 5. 扩展：并查集解法

题目标签中含**并查集**，本节给出并查集思路，适合用来练习 Union-Find 模板。

**思路**：把每个数看作图中的一个节点。若 `num + 1` 也在集合中，则把 `num` 和 `num + 1` 合并到同一个集合（并查集连通分量）。最终答案就是所有连通分量中**规模最大的那个**。

- **查找**：路径压缩 `find(x)`。
- **合并**：`union(x, x+1)`，同时维护每个根节点对应的集合大小 `size[root]`。
- **查询**：遍历所有元素，取 `size[find(x)]` 的最大值。

![并查集合并相邻数](images/longest_consecutive_unionfind.svg)

时间复杂度：建图 O(n)，每次合并/查找近似 `O(α(n))`（α 是阿克曼反函数，常数级），整体 `O(n α(n)) ≈ O(n)`。空间 O(n)。

### 并查集 C++ 代码

```cpp
class Solution {
    unordered_map<int, int> parent, size_;

    int find(int x) {
        if (parent[x] != x) parent[x] = find(parent[x]);
        return parent[x];
    }

    void unite(int x, int y) {
        int rx = find(x), ry = find(y);
        if (rx == ry) return;
        if (size_[rx] < size_[ry]) swap(rx, ry);
        parent[ry] = rx;
        size_[rx] += size_[ry];
    }

public:
    int longestConsecutive(vector<int>& nums) {
        if (nums.empty()) return 0;
        for (int x : nums) {
            if (parent.count(x)) continue;       // 去重
            parent[x] = x;
            size_[x] = 1;
            if (parent.count(x - 1)) unite(x, x - 1);
            if (parent.count(x + 1)) unite(x, x + 1);
        }
        int ans = 0;
        for (auto& [x, _] : parent) ans = max(ans, size_[find(x)]);
        return ans;
    }
};
```

> ⚠️ 并查集解法常数较大、实现更重，面试中**首选哈希集合解法**；并查集版本适合作为"标签对应模板"的练习。

## 6. 面试要点

1. **为什么这个算法是 O(n) 而不是 O(n^2)？**
   - 关键在于 `num - 1` 的起点判断：只有真正的序列起点才会触发内层 `while`。每个元素至多在"作为起点的扩展链"中被访问一次，而后续元素因前驱存在必被跳过，因此内层总步数 ≤ n。

2. **如果不加 `num - 1` 判断会怎样？**
   - 最坏情况如 `1,2,...,n`：从 1 扩展 n 步，从 2 扩展 n-1 步……总复杂度退化为 `O(n^2)`，会 TLE。

3. **为什么要遍历集合而不是原数组？**
   - 原数组可能有重复元素（如示例 2 有两个 0）。遍历集合可以避免对重复元素重复扩展，减少常数开销；即便遍历原数组答案仍正确，但效率略低。

4. **题目要求 O(n)，为什么不能用排序？**
   - 排序下界 `O(n log n)`，违反题目硬性约束。但若面试官放宽要求，排序 + 一次扫描是更易写的方案，可作为"先给出可行解再优化"的过渡。

5. **如何返回最长序列本身（而不仅是长度）？**
   - 在更新 `ans` 时，记录对应的起点 `start_num` 和长度，最后用 `[start_num, start_num + ans - 1]` 还原序列即可，无需改动主算法。
