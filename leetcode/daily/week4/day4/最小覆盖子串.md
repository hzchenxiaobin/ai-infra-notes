# 最小覆盖子串

- **题目名称**：最小覆盖子串
- **链接**：[76. 最小覆盖子串](https://leetcode.cn/problems/minimum-window-substring/)
- **难度**：困难
- **标签**：哈希表、字符串、滑动窗口

## 1. 题目概述

给定字符串 `s` 和 `t`，返回 `s` 中涵盖 `t` 中所有字符的**最小子串**。如果不存在这样的子串，返回空字符串 `""`。

> ⚠️ **注意**：`t` 中的字符（含重复字符）都必须被覆盖，且子串是 `s` 中的连续片段。

**示例 1**：

```text
输入：s = "ADOBECODEBANC", t = "ABC"
输出："BANC"
解释：最小覆盖子串是 "BANC"，它包含 t 中的 'A'、'B'、'C' 各一个。
```

**示例 2**：

```text
输入：s = "a", t = "a"
输出："a"
解释：整个字符串 s 即为最小覆盖子串。
```

**示例 3**：

```text
输入：s = "a", t = "aa"
输出：""
解释：s 中只有一个 'a'，无法满足 t 需要的两个 'a'，返回空串。
```

**约束条件**：

- `1 <= s.length, t.length <= 10^5`
- `s` 和 `t` 由英文字母（大小写）组成

---

## 2. 解题思路

### 2.1 暴力思路

枚举 `s` 的所有子串 `[i, j]`（共 `O(n^2)` 个），对每个子串检查是否涵盖 `t` 的全部字符（检查 `O(n + m)`），记录满足条件的最短子串。总复杂度 `O(n^2 · (n + m))`，必然超时。

### 2.2 核心观察：滑动窗口 + 哈希计数

![滑动窗口核心概念](images/mws_sliding_window.svg)

关键直觉是维护一个**可能涵盖 t 的窗口** `[left, right]`：

- `right` 不断右移，把 `s[right]` 纳入窗口，**扩张探索**。
- 一旦窗口满足条件（涵盖 `t` 的所有字符），就尝试**收缩 `left`**——只要收缩后仍满足条件，就继续收缩，从而找到「以当前 `right` 结尾的最短可行子串」。
- 在收缩过程中持续更新全局最小窗口。

> 💡 **如何 O(1) 判断窗口是否涵盖 t？** 用两张哈希表：`need` 记录 `t` 中每个字符的需求量，`window` 记录当前窗口的字符量。再用一个计数器 `valid` 表示「窗口中已满足需求量的字符种类数」。当 `valid == need.size()` 时窗口合法。这样每次扩张/收缩只需 `O(1)` 更新，无需遍历整张表。

### 2.3 算法流程图

![滑动窗口算法流程](images/mws_algorithm_flow.svg)

整体只有一重 `for` 循环驱动 `right`，内层 `while` 收缩 `left`。`left`、`right` 都只增不减，因此总移动次数不超过 `2n`，复杂度 `O(n)`。

### 2.4 示例演算

![示例演算：ADOBECODEBANC](images/mws_example_walkthrough.svg)

以 `s = "ADOBECODEBANC"`，`t = "ABC"` 为例（`needCnt = 3`）：

| step | right | 新字符 | valid | 窗口 [left,right]      | 动作                       | minLen |
|------|-------|--------|-------|------------------------|----------------------------|--------|
| 1    | 0     | A      | 1     | [0,0] A                | 扩张                       | ∞      |
| 2    | 5     | C      | 3     | [0,5] ADOBEC           | 合法 → 收缩，左端 A 移出   | 6      |
| 3    | 10    | A      | 3     | [1,10] DOBECODEBA      | 合法 → 收缩到 [5,10] CODEBA，C 移出 | 6      |
| 4    | 12    | C      | 3     | [6,12] ODEBANC         | 合法 → 收缩到 [9,12] BANC  | 4      |

最终最小窗口为 `s[9..12] = "BANC"`。

## 3. 参考代码

### C++

```cpp
class Solution {
  public:
    string minWindow(string s, string t) {
        unordered_map<char, int> need, window;
        for (char c : t)
            need[c]++;
        int needCnt = need.size(); // 需要满足的字符种类数
        int left = 0, valid = 0;
        int start = 0, minLen = INT_MAX;

        for (int right = 0; right < (int)s.size(); right++) {
            char c = s[right];
            if (need.count(c)) { // 只关心 t 中的字符
                window[c]++;
                if (window[c] == need[c])
                    valid++; // 该字符刚好满足
            }
            while (valid == needCnt) { // 窗口合法，尝试收缩
                if (right - left + 1 < minLen) {
                    minLen = right - left + 1;
                    start = left;
                }
                char d = s[left];
                if (need.count(d)) {
                    if (window[d] == need[d])
                        valid--; // 该字符将不再满足
                    window[d]--;
                }
                left++;
            }
        }
        return minLen == INT_MAX ? "" : s.substr(start, minLen);
    }
};
```

### Python

```python
class Solution:
    def minWindow(self, s: str, t: str) -> str:
        from collections import Counter
        need = Counter(t)
        window = {}
        need_cnt = len(need)              # 需要满足的字符种类数
        left = valid = 0
        start, min_len = 0, float('inf')

        for right, c in enumerate(s):
            if c in need:                  # 只关心 t 中的字符
                window[c] = window.get(c, 0) + 1
                if window[c] == need[c]:
                    valid += 1
            while valid == need_cnt:       # 窗口合法，尝试收缩
                if right - left + 1 < min_len:
                    min_len = right - left + 1
                    start = left
                d = s[left]
                if d in need:
                    if window[d] == need[d]:
                        valid -= 1
                    window[d] -= 1
                left += 1

        return "" if min_len == float('inf') else s[start:start + min_len]
```

> ⚠️ **两个易错点**：(1) 收缩时必须**先判断 `window[d] == need[d]` 再 `window[d]--`**，顺序反了会让 `valid` 错乱；(2) `valid` 统计的是「种类数」而非「字符数」，只有当某字符数量**恰好等于**需求时才 `valid++`，严格大于时不再加。

## 4. 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| 时间复杂度 | O(n + m) | `right`、`left` 各至多遍历 `s` 一次（`n = len(s)`），预处理 `t` 为 `O(m)` |
| 空间复杂度 | O(m) | 哈希表 `need`/`window` 最多存 `t` 的不同字符种类数，上界 `O(m)` |

## 5. 扩展：数组代替哈希表

题目保证 `s`、`t` 仅含英文字母（ASCII 范围），可用定长数组 `int[128]` 代替 `unordered_map`，省去哈希开销、常数更小：

```cpp
class Solution {
  public:
    string minWindow(string s, string t) {
        int need[128] = {0}, window[128] = {0};
        int needCnt = 0;
        for (char c : t)
            if (need[c]++ == 0)
                needCnt++;
        int left = 0, valid = 0, start = 0, minLen = INT_MAX;

        for (int right = 0; right < (int)s.size(); right++) {
            char c = s[right];
            if (need[c]) {
                if (++window[c] == need[c])
                    valid++;
            }
            while (valid == needCnt) {
                if (right - left + 1 < minLen) {
                    minLen = right - left + 1;
                    start = left;
                }
                char d = s[left++];
                if (need[d]) {
                    if (window[d]-- == need[d])
                        valid--;
                }
            }
        }
        return minLen == INT_MAX ? "" : s.substr(start, minLen);
    }
};
```

> 💡 **相关题目**：[438. 找到字符串中所有字母异位词](https://leetcode.cn/problems/find-all-anagrams-in-a-string/) 用的是同一套「`need` + `window` + `valid`」模板，只是把「最小窗口」换成「固定长度窗口的起点列表」，可直接迁移。

## 6. 面试要点

1. **为什么左右指针都只增不减就能保证不漏解？**

   - 对任意一个合法窗口，`right` 第一次到达使其合法的位置时，`left` 会收缩到「以该 `right` 结尾的最短合法窗口」。后续 `right` 右移后，更优解只可能出现在更靠右的位置，因此 `left` 无需回退——这保证了总移动次数 `O(n)`。

2. **`valid` 用「种类数」而不是「字符总数」有什么好处？**

   - 用种类数可以**只在 `window[c] == need[c]` 这一个临界点**触发 `valid++`，避免每次加减字符都要比较全部需求；判断合法只需 `valid == needCnt`，O(1)。若用字符总数则要频繁比较两张表，常数更大。

3. **收缩时为什么要先判等再 `--`？**

   - `window[d] == need[d]` 表示「该字符当前刚好满足，移出一个后将不再满足」，必须先把 `valid` 减 1，再让 `window[d]` 跌破需求。若先 `--` 再判等，`window[d]` 已变成 `need[d] - 1`，永远判不相等，`valid` 漏减导致死循环或错误答案。

4. **如果 `t` 中有字符不在 `s` 中怎么办？**

   - 该字符的 `need[c] > 0` 但 `window[c]` 永远到不了 `need[c]`，`valid` 永远小于 `needCnt`，`minLen` 保持 `INT_MAX`，最终返回 `""`。无需特判。

5. **和「无重复字符的最长子串」(3) 的滑动窗口有何异同？**

   - 都是同向双指针 + 哈希表。区别在于**收缩条件**：第 3 题窗口必须「无重复」，遇到重复立即收缩 `left`，目标是**最大化**窗口；本题窗口必须「涵盖 t」，合法时才收缩 `left`，目标是**最小化**窗口。前者是「扩张为主、收缩为辅」，后者是「合法即收缩」。
