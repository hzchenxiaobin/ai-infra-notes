# 跳跃游戏 II

- **题目名称**：跳跃游戏 II
- **链接**：[45. 跳跃游戏 II](https://leetcode.cn/problems/jump-game-ii/)
- **难度**：中等
- **标签**：贪心、数组、动态规划

## 1. 题目概述

给定一个长度为 `n` 的 **0 索引**整数数组 `nums`。一开始位于数组的第一个位置 `nums[0]`。

数组中的每个元素 `nums[i]` 表示从位置 `i` 可以向前跳跃的最大长度。也就是说，如果你位于 `nums[i]`，你可以跳到任意 `nums[i + j]` 处，其中：

- `0 <= j <= nums[i]`
- `i + j < n`

返回到达 `nums[n - 1]` 的最小跳跃次数。生成的测试用例保证可以到达 `nums[n - 1]`。

**示例 1**：

```text
输入：nums = [2,3,1,1,4]
输出：2
解释：跳到最后一个位置的最小跳跃次数是 2。
     从下标 0 跳到下标 1，然后再跳到下标 4。
```

**示例 2**：

```text
输入：nums = [2,3,0,1,4]
输出：2
```

**约束条件**：

- `1 <= nums.length <= 10^4`
- `0 <= nums[i] <= 1000`
- 题目保证可以到达 `nums[n-1]`

> 💡 这是 **贪心算法** 的经典例题。与 [Week5/Day3 任务调度器](../week5/day3/任务调度器.md) 这种需要排序/堆的贪心不同，跳跃游戏的贪心体现在"每一步都尽可能扩展能到达的最远边界"。

## 2. 解题思路

### 2.1 贪心：维护当前边界与下一跳最远边界

想象我们在一条路上跳跃，每一步都要做出最优选择：

- 在当前这一步的覆盖范围内，选择一个位置作为下一跳的起跳点，使得下一跳能到达最远。
- 当遍历完当前这一步能覆盖的所有位置后，必然要再跳一次，把边界扩展到下一跳的最远位置。

具体做法：

- `end`：当前这一步能到达的最远边界
- `maxPos`：从当前这一步范围内起跳，能到达的最远位置
- `steps`：已经使用的跳跃次数

遍历数组时不断更新 `maxPos`，当遍历到 `end` 时，说明必须再跳一次，于是 `steps++` 并把 `end` 更新为 `maxPos`。

### 2.2 动态规划（较慢，供对比）

设 `dp[i]` 表示到达位置 `i` 的最小跳跃次数。对于每个位置 `i`，枚举所有能跳到 `i` 的前驱位置 `j`，取最小值：

```
dp[i] = min(dp[j] + 1)  for all j < i and j + nums[j] >= i
```

时间复杂度 `O(n^2)`，对于 `n = 10^4` 会超时，仅用于理解。

## 3. 参考代码

### 3.1 贪心解法

```cpp
class Solution {
public:
    int jump(vector<int>& nums) {
        int n = nums.size();
        if (n == 1) return 0;

        int steps = 0;          // 已用跳跃次数
        int end = 0;            // 当前这一步能覆盖的最远边界
        int maxPos = 0;         // 下一步能到达的最远位置

        for (int i = 0; i < n - 1; ++i) {
            maxPos = max(maxPos, i + nums[i]);
            // 到达当前边界，必须再跳一次
            if (i == end) {
                steps++;
                end = maxPos;
                // 贪心保证：一旦 end 已经覆盖终点，可以提前结束
                if (end >= n - 1) break;
            }
        }
        return steps;
    }
};
```

### 3.2 Python 版本

```python
class Solution:
    def jump(self, nums: list[int]) -> int:
        n = len(nums)
        if n == 1:
            return 0

        steps = 0
        end = 0
        max_pos = 0

        for i in range(n - 1):
            max_pos = max(max_pos, i + nums[i])
            if i == end:
                steps += 1
                end = max_pos
                if end >= n - 1:
                    break
        return steps
```

## 4. 复杂度分析

- **时间复杂度**：`O(n)`，只需遍历一次数组。
- **空间复杂度**：`O(1)`，只使用常数额外空间。

---

## 5. 面试要点

1. **为什么遍历到 `n-2` 而不是 `n-1`**：因为终点不需要再跳，一旦 `end` 覆盖到 `n-1` 就可以结束。
2. **为什么 `i == end` 时步数加 1**：说明当前这一步能覆盖的范围已经全部考察完毕，必须发动下一次跳跃。
3. **贪心正确性证明**：在当前覆盖范围内，我们总是记录能到达的最远位置；当必须跳时，选择这个最远位置一定不劣于任何其他选择，因此得到最少步数。
4. **与跳跃游戏 I 的区别**：[55. 跳跃游戏](https://leetcode.cn/problems/jump-game/) 只需判断是否能到达终点，本题要求最小步数。
