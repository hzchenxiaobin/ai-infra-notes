# 买卖股票的最佳时机

- **题目名称**：买卖股票的最佳时机
- **链接**：[121. 买卖股票的最佳时机](https://leetcode.cn/problems/best-time-to-buy-and-sell-stock/)
- **难度**：简单
- **标签**：数组、动态规划

## 1. 题目概述

给定一个数组 `prices`，其中 `prices[i]` 表示第 `i` 天的股票价格。你只能选择**某一天买入**并在**未来的另一天卖出**，求能获得的最大利润。如果不能获得任何利润，返回 `0`。

**示例 1**：

```text
输入：prices = [7,1,5,3,6,4]
输出：5
解释：在第 2 天（价格 = 1）买入，在第 5 天（价格 = 6）卖出，最大利润 = 6 - 1 = 5。
```

**示例 2**：

```text
输入：prices = [7,6,4,3,1]
输出：0
解释：在这种情况下，没有交易完成，所以最大利润为 0。
```

**约束条件**：

- `1 <= prices.length <= 10^5`
- `0 <= prices[i] <= 10^4`

---

## 2. 解题思路

### 2.1 暴力思路

枚举所有可能的买入日 `i` 和卖出日 `j`（`j > i`），计算 `prices[j] - prices[i]` 取最大值。时间复杂度 `O(n^2)`，在 `n = 10^5` 时会**超时**。

```text
ans = 0
for i in range(n):
    for j in range(i+1, n):
        ans = max(ans, prices[j] - prices[i])
```

> ⚠️ 暴力法的瓶颈在于：对每个卖出日都重复扫描了之前的所有买入日，存在大量冗余比较。

### 2.2 核心观察：一次遍历维护最小买入价

![一次遍历维护最小买入价](images/stock_max_profit.svg)

关键洞察：**如果我们想在第 `i` 天卖出获得最大利润，那么最优买入日一定是 `0..i-1` 中价格最低的那一天。**

于是只需在遍历过程中维护一个变量 `min_price`（历史最低买入价），每到一天 `i`：

1. 用 `prices[i] - min_price` 计算若今天卖出的利润，更新 `ans`。
2. 用 `prices[i]` 更新 `min_price`。

> 💡 这相当于把「枚举买入日」这一维压缩成了「维护历史最小值」，时间复杂度从 `O(n^2)` 降到 `O(n)`。

**DP 视角**：定义 `dp[i]` 为前 `i` 天能获得的最大利润，则：

```text
dp[i] = max(dp[i-1], prices[i] - min_price_before_i)
```

由于 `dp[i]` 只依赖 `dp[i-1]`，可以滚动成一个变量。

### 2.4 示例演算

以 `prices = [7,1,5,3,6,4]` 为例：

![示例演算过程](images/stock_example_walkthrough.svg)

| i | prices[i] | min_price | 今日卖出利润 | ans |
|---|-----------|-----------|--------------|-----|
| 0 | 7         | 7         | 0            | 0   |
| 1 | 1         | 1         | 0            | 0   |
| 2 | 5         | 1         | 4            | 4   |
| 3 | 3         | 1         | 2            | 4   |
| 4 | 6         | 1         | 5            | 5   |
| 5 | 4         | 1         | 3            | 5   |

最终答案 `ans = 5`（第 2 天买入 1，第 5 天卖出 6）。

## 3. 参考代码

### C++

```cpp
class Solution {
  public:
    int maxProfit(vector<int>& prices) {
        int min_price = INT_MAX;
        int ans = 0;

        for (int price : prices) {
            ans = max(ans, price - min_price);
            min_price = min(min_price, price);
        }

        return ans;
    }
};
```

### Python

```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        min_price = float('inf')
        ans = 0

        for price in prices:
            ans = max(ans, price - min_price)
            min_price = min(min_price, price)

        return ans
```

> 💡 代码里**先更新 `ans` 再更新 `min_price`**，保证同一天不会既买入又卖出（虽然本题这样做结果也一样，因为差为 0，但顺序写对更严谨）。

**122 无限次买卖（贪心）参考代码**：

```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        ans = 0
        for i in range(1, len(prices)):
            if prices[i] > prices[i - 1]:
                ans += prices[i] - prices[i - 1]
        return ans
```

> 💡 通法：所有股票题都可以用「状态机 DP」统一建模——用 `hold`（持有）和 `cash`（不持有）两个状态滚动转移，再按题意加上交易次数 / 冷冻期 / 手续费的限制。

## 6. 面试要点

1. **为什么只需维护历史最小值？**
   - 对于任意卖出日 `i`，最大利润必然来自 `0..i-1` 中的最小买入价。所以遍历时实时维护 `min_price` 即可，无需回头扫描。

2. **这题和动态规划是什么关系？**
   - 可视为一维 DP：`dp[i] = max(dp[i-1], prices[i] - min_price)`。由于只依赖前一项，可滚动压缩为 `O(1)` 空间，即本题写法。

3. **先更新 `ans` 还是先更新 `min_price` 有区别吗？**
   - 本题结果相同（同日买卖利润为 0）。但先算 `ans` 再更新 `min_price` 语义更清晰，也方便迁移到「含冷冻期」等变体。

4. **如果要求输出买卖的具体日期怎么办？**
   - 在更新 `min_price` 时记录对应的买入日 `buy_day`，在更新 `ans` 时记录卖出日 `sell_day = i`，最后返回两者即可。

5. **数组单调递减时返回什么？**
   - 返回 `0`。因为每次 `price - min_price <= 0`，`ans` 始终保持 0，符合「不能获利时返回 0」的要求。
