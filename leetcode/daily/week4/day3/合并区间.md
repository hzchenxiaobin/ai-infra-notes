# 合并区间

- **题目名称**：合并区间
- **链接**：[56. 合并区间](https://leetcode.cn/problems/merge-intervals/)
- **难度**：中等
- **标签**：数组、排序

## 1. 题目概述

给定一个区间集合 `intervals`，其中 `intervals[i] = [start_i, end_i]`，合并所有**重叠**的区间，返回一个不重叠的区间数组，使得覆盖的范围与输入完全相同。

**示例 1**：

```text
输入：intervals = [[1,3],[2,6],[8,10],[15,18]]
输出：[[1,6],[8,10],[15,18]]
解释：[1,3] 与 [2,6] 重叠（3 >= 2），合并为 [1,6]；其余区间互不重叠。
```

**示例 2**：

```text
输入：intervals = [[1,4],[4,5]]
输出：[[1,4]]
解释：[1,4] 与 [4,5] 在端点 4 处相接，视为重叠，合并为 [1,4]。
```

**约束条件**：

- `1 <= intervals.length <= 10^4`
- `intervals[i].length == 2`
- `0 <= start_i <= end_i <= 10^4`

---

## 2. 解题思路

### 2.1 暴力思路（O(n²) 两两比较）

最直接的做法：对每个区间，遍历其余所有区间，看是否相交（`a.start <= b.end && b.start <= a.end`），相交则合并。重复多轮直到没有可合并的区间为止。

> ⚠️ 这种做法时间复杂度高达 **O(n²)** 甚至 O(n³)，当 `n = 10^4` 时会超时。更糟糕的是，区间顺序混乱时，一次扫描往往无法合并完，需要多轮迭代。

### 2.2 核心观察：排序后贪心合并

**关键洞察**：如果按 `start` 升序排序，那么所有可能重叠的区间必然**相邻排列**。

![排序后贪心合并示意图](images/merge_intervals_overview.svg)

排序后，遍历区间时只需维护一个"当前合并区间"：

- 若下一个区间的 `start <= 当前区间的 end` → **重叠**，更新 `end = max(end, next.end)`。
- 否则 → **不重叠**，把当前区间推入结果，开启新的合并区间。

> 💡 **为什么排序就够了？** 排序后，后一个区间的 `start` 一定 >= 前一个的 `start`。若它不超过当前合并区间的 `end`，就一定落入当前区间内部或与之相接；否则它一定与当前区间分离，且之后的所有区间 `start` 更大，更不可能再与当前区间重叠。

### 2.3 算法流程图

![合并区间算法流程](images/merge_intervals_greedy.svg)

整体步骤：

1. **排序**：按 `start` 升序（`start` 相同则按 `end` 升序）。
2. **初始化**：把第一个区间放入结果数组 `merged`。
3. **遍历**：对每个后续区间 `cur`，与 `merged` 中最后一个区间 `last` 比较：
   - `cur[0] <= last[1]` → 合并：`last[1] = max(last[1], cur[1])`。
   - 否则 → `merged.push_back(cur)`。
4. **返回** `merged`。

### 2.4 示例演算

以 `[[1,3],[2,6],[8,10],[15,18]]` 为例（已按 `start` 排序）：

![示例演算过程](images/merge_intervals_walkthrough.svg)

```text
初始 merged = [[1,3]]

i=1: cur=[2,6], 2 <= 3 → 重叠, end = max(3,6)=6 → merged=[[1,6]]
i=2: cur=[8,10], 8 > 6 → 不重叠, push → merged=[[1,6],[8,10]]
i=3: cur=[15,18], 15 > 10 → 不重叠, push → merged=[[1,6],[8,10],[15,18]]

结果：[[1,6],[8,10],[15,18]]
```

## 3. 参考代码

### C++

```cpp
class Solution {
  public:
    vector<vector<int>> merge(vector<vector<int>>& intervals) {
        if (intervals.empty())
            return {};

        // 1. 按起点升序排序
        sort(intervals.begin(), intervals.end(),
             [](const vector<int>& a, const vector<int>& b) { return a[0] < b[0]; });

        // 2. 贪心合并
        vector<vector<int>> merged;
        merged.push_back(intervals[0]);

        for (int i = 1; i < intervals.size(); ++i) {
            vector<int>& last = merged.back();
            if (intervals[i][0] <= last[1]) {
                // 重叠：扩展右端点
                last[1] = max(last[1], intervals[i][1]);
            } else {
                // 不重叠：新增区间
                merged.push_back(intervals[i]);
            }
        }

        return merged;
    }
};
```

### Python

```python
from typing import List

class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        if not intervals:
            return []

        # 1. 按起点升序排序
        intervals.sort(key=lambda x: x[0])

        # 2. 贪心合并
        merged = [intervals[0]]
        for cur in intervals[1:]:
            last = merged[-1]
            if cur[0] <= last[1]:
                # 重叠：扩展右端点
                last[1] = max(last[1], cur[1])
            else:
                # 不重叠：新增区间
                merged.append(cur)

        return merged
```

## 4. 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| 时间复杂度 | O(n log n) | 排序 O(n log n) + 一次遍历合并 O(n) |
| 空间复杂度 | O(log n) | 排序递归栈空间（结果数组不计入额外空间，部分判定为 O(n)） |

> 💡 排序是瓶颈。合并阶段每个区间只访问一次，已是最优。若输入本身有序，可降到 O(n)。

## 6. 面试要点

1. **为什么要先排序？不排序会怎样？**
   - 排序后所有可能重叠的区间必然相邻，只需一次线性扫描即可完成合并。
   - 不排序时，重叠区间可能分散在任意位置，需要 O(n²) 两两比较，甚至多轮迭代。

2. **判断重叠的条件是 `cur[0] <= last[1]`，为什么用 `<=` 而不是 `<`？**
   - 题目规定端点相接（如 `[1,4]` 与 `[4,5]`）算重叠，需合并为 `[1,5]`（示例 2 即 `[1,4]`，因 `4<=4` 合并）。
   - 用 `<=` 涵盖相接情况；若题目改为严格相交，则用 `<`。

3. **合并时为什么取 `max(last[1], cur[1])` 而不是直接用 `cur[1]`？**
   - 排序只保证 `start` 升序，不保证 `end` 升序。例如 `[1,5],[2,3]`，`cur[1]=3 < last[1]=5`，直接赋值会**缩小**区间范围，丢失覆盖。
   - 取 `max` 保证合并后的区间始终覆盖所有原区间。

4. **空间复杂度到底是 O(1) 还是 O(n)？**
   - 若结果数组视为输出不算额外空间，则为 O(log n)（排序栈）。
   - 若结果数组计入空间，则为 O(n)。面试时主动说明口径即可。

5. **如果区间数量极大（10^7 级），还能用这个方法吗？**
   - 排序 O(n log n) 仍是主流。若区间值域有限（如本题 `end_i <= 10^4`），可用**计数排序/桶排序**降到 O(n + V)，再做扫描线合并。
