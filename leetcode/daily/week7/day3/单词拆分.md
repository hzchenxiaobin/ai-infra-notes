# 单词拆分

- **题目名称**：单词拆分
- **链接**：[139. 单词拆分](https://leetcode.cn/problems/word-break/)
- **难度**：中等
- **标签**：动态规划、哈希表、字符串

## 1. 题目概述

给定一个字符串 `s` 和一个字符串字典 `wordDict`，判断 `s` 是否可以被拆分为一个或多个字典中出现的单词的序列。

**示例 1**：

```text
输入：s = "leetcode", wordDict = ["leet","code"]
输出：true
解释：s 可以拆分为 "leet code"
```

**示例 2**：

```text
输入：s = "applepenapple", wordDict = ["apple","pen"]
输出：true
解释：s 可以拆分为 "apple pen apple"（可重复使用字典中的词）
```

**约束条件**：

- `1 <= s.length <= 300`
- `1 <= wordDict.length <= 1000`
- `1 <= wordDict[i].length <= 20`
- `s` 和 `wordDict[i]` 仅由小写英文组成

---

## 2. 解题思路

### 2.1 暴力递归

对每个位置尝试所有可能的拆分点 → `O(2^n)` 指数级，超时。

### 2.2 核心观察：DP + 前缀缓存

![DP 前缀缓存求单词拆分](images/word_break.svg)

关键洞察：`dp[i]` **表示前 i 个字符能否拆分**——本质是缓存"前缀 `s[0..i]` 的计算结果"。新位置 `i` 只需检查：是否存在 `j < i` 使得 `dp[j]` 为 true 且 `s[j..i]` 在字典中。

> 💡 与 [Week7 Day3 Prefix Caching](../../aiinfra/daily/week7/day3/README.md) 同构——`dp[j]` 是前缀缓存命中（前 j 个字符已解决），`s[j..i]` 是增量计算（只处理新增部分）。Prefix Caching 缓存 KV Cache，DP 缓存子问题答案，两者都是**避免重复计算前缀**的核心思想。

### 2.3 算法流程

1. `dp[0] = True`（空前缀可拆分）
2. 把 `wordDict` 转为 `set`（O(1) 查词）
3. 对每个 `i` 从 1 到 `n`：
   - 对每个 `j` 从 0 到 `i-1`：
     - 若 `dp[j]` 且 `s[j:i]` 在 set 中 → `dp[i] = True`，break
4. 返回 `dp[n]`

### 2.4 示例演算

`s = "leetcode"`, `wordDict = ["leet","code"]`，set = `{"leet","code"}`：

| i | j | dp[j] | s[j:i] | in set? | dp[i] |
|---|---|-------|--------|---------|-------|
| 0 | — | — | — | — | True |
| 4 | 0 | True | "leet" | ✓ | True |
| 8 | 0 | True | "leetcode" | ✗ | — |
| 8 | 4 | True | "code" | ✓ | True |

输出 `True`。

## 3. 参考代码

### C++

```cpp
class Solution {
  public:
    bool wordBreak(string s, vector<string>& wordDict) {
        unordered_set<string> dict(wordDict.begin(), wordDict.end());
        int n = s.size();
        vector<bool> dp(n + 1, false);
        dp[0] = true;

        for (int i = 1; i <= n; i++) {
            for (int j = 0; j < i; j++) {
                if (dp[j] && dict.count(s.substr(j, i - j))) {
                    dp[i] = true;
                    break;
                }
            }
        }
        return dp[n];
    }
};
```

### Python

```python
class Solution:
    def wordBreak(self, s: str, wordDict: List[str]) -> bool:
        word_set = set(wordDict)
        n = len(s)
        dp = [False] * (n + 1)
        dp[0] = True

        for i in range(1, n + 1):
            for j in range(i):
                if dp[j] and s[j:i] in word_set:
                    dp[i] = True
                    break

        return dp[n]
```

## 4. 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| 时间复杂度 | `O(n² × m)` | n=字符串长度，m=平均词长（substr + hash） |
| 空间复杂度 | `O(n + k×m)` | dp 数组 O(n) + set O(k×m) |

> 💡 可优化到 `O(n × max_word_len)`：内层循环只遍历 `j = i - max_word_len` 到 `i`，因为字典中最长的词长度有限。

## 5. 扩展：BFS / 记忆化 DFS

### BFS 解法

把字符串看作图，位置为节点，字典中的词为边。BFS 从位置 0 出发，看能否到达位置 n。

```python
def wordBreak_bfs(s, wordDict):
    word_set = set(wordDict)
    from collections import deque
    queue = deque([0])
    visited = set()

    while queue:
        start = queue.popleft()
        if start in visited:
            continue
        visited.add(start)
        for end in range(start + 1, len(s) + 1):
            if s[start:end] in word_set:
                if end == len(s):
                    return True
                queue.append(end)
    return False
```

## 6. 面试要点

1. **dp[i] 的含义是什么？为什么是"前缀缓存"？**

   - `dp[i]` = 前 i 个字符能否被拆分
   - 类比 Prefix Caching：`dp[j]` 是"前 j 个字符的前缀结果已缓存"，`s[j:i]` 是"增量部分"
   - 查 `dp[j]` = 缓存命中（O(1)），查 `s[j:i] in set` = 增量计算
   - 两者都是"避免重复计算前缀"的核心思想

2. **这题和 Prefix Caching 有什么共同模式？**

   - DP 缓存子问题答案，Prefix Caching 缓存 KV Cache
   - DP 的 `dp[j]` 对应 Prefix Caching 的 cache hit（前缀已解决）
   - DP 的 `s[j:i]` 对应 Prefix Caching 的增量 prefill（只处理新增 tokens）
   - 都是"前缀复用 + 增量计算"的模式

3. **为什么用 set 存字典？**

   - `set` 的查找是 O(1)（哈希表），`list` 的查找是 O(k)
   - DP 内层循环频繁查词，O(1) 查找是性能关键
   - 对应 Prefix Caching 中用 hash 做 O(1) 缓存查找

4. **内层循环能优化吗？**

   - 可以只遍历 `j = max(0, i - max_word_len)` 到 `i`，因为字典中最长的词长度有限
   - 时间从 `O(n²)` 降到 `O(n × max_word_len)`
   - 也可按字典中的词长度去重，只尝试可能的长度

5. **BFS 和 DP 哪个好？**

   - DP 更直观，适合面试首选
   - BFS 可以提前终止（找到终点即返回），但需要 visited 防重复
   - 记忆化 DFS 也可以，但 Python 递归深度可能受限
