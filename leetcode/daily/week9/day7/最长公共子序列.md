# 最长公共子序列

- **题目名称**：最长公共子序列
- **链接**：[1143. 最长公共子序列](https://leetcode.cn/problems/longest-common-subsequence/)
- **难度**：中等
- **标签**：动态规划、二维 DP、字符串

## 1. 题目概述

给定两个字符串 `text1` 和 `text2`，返回这两个字符串的**最长公共子序列**的长度。如果不存在公共子序列，返回 `0`。

**子序列**：从一个字符串中删除某些（也可不删）字符而不改变剩余字符相对顺序得到的新字符串。例如 `"ace"` 是 `"abcde"` 的子序列，但 `"aec"` 不是。

**示例 1**：

```text
输入：text1 = "abcde", text2 = "ace"
输出：3
解释：最长公共子序列是 "ace"，长度为 3。
```

**示例 2**：

```text
输入：text1 = "abc", text2 = "abc"
输出：3
解释：最长公共子序列是 "abc"，长度为 3。
```

**示例 3**：

```text
输入：text1 = "abc", text2 = "def"
输出：0
解释：两字符串无公共子序列。
```

**约束条件**：

- `1 <= text1.length, text2.length <= 1000`
- `text1` 和 `text2` 仅由小写英文字母组成

> 💡 这是**二维 DP 的经典模板**。与 [72. 编辑距离](../../week8/day4/编辑距离.md) 同构——两者都是「两序列逐位置交互」的二维表格 DP，只是转移语义不同：LCS 在字符匹配时 `+1`（求最长），编辑距离在不匹配时 `+1`（求最少操作）。掌握 LCS 的二维 DP 模板，就能迁移到编辑距离、最长重复子数组、不同的子序列等一整类问题。

## 2. 解题思路

### 2.1 暴力思路：递归枚举

对 `text1` 和 `text2` 的每个位置，若字符相同则计入并递归剩余部分，若不同则分别尝试跳过 `text1` 或 `text2` 的当前字符，取较大值。时间 $O(2^{m+n})$，指数级超时，且有大量重复子问题。

### 2.2 核心观察：二维 DP

![核心观察：dp[i][j] = text1[0..i-1] 与 text2[0..j-1] 的 LCS 长度](images/lcs_dp_table.svg)

**定义**：`dp[i][j]` = `text1[0..i-1]` 与 `text2[0..j-1]` 的最长公共子序列长度。

**边界**：

```text
dp[0][j] = 0      # text1 为空，LCS = 0
dp[i][0] = 0      # text2 为空，LCS = 0
```

**转移**（核心）：

```text
若 text1[i-1] == text2[j-1]：
    dp[i][j] = dp[i-1][j-1] + 1        # 字符匹配，LCS 延长 1

若 text1[i-1] != text2[j-1]：
    dp[i][j] = max(dp[i-1][j],          # 跳过 text1[i-1]
                   dp[i][j-1])           # 跳过 text2[j-1]
```

**两种转移的直觉**：

| 情况 | DP 来源 | 含义 |
|------|---------|------|
| **字符匹配** | `dp[i-1][j-1] + 1` | 两字符相同，都纳入 LCS，各前进一步 |
| **字符不匹配** | `max(dp[i-1][j], dp[i][j-1])` | 至少一个字符不在 LCS 中，尝试跳过各自取较大 |

> 💡 **为什么字符匹配时不取 `max(dp[i-1][j], dp[i][j-1])`？** 因为 `dp[i-1][j-1] + 1` 恒 $\geq$ `max(dp[i-1][j], dp[i][j-1])`。直觉：匹配一对字符至少不比跳过其中一个差。证明：`dp[i-1][j] ≤ dp[i-1][j-1] + 1`（在 `text1[0..i-2]` 与 `text2[0..j-1]` 的 LCS 基础上，`text2[j-1]` 最多再贡献 1），同理 `dp[i][j-1] ≤ dp[i-1][j-1] + 1`。

**答案**：`dp[m][n]`。

### 2.3 算法流程图

![算法流程：初始化边界 → 逐行填表 → 返回 dp[m][n]](images/lcs_algorithm_flow.svg)

**完整步骤**：

1. **初始化**：`dp` 为 `(m+1) × (n+1)` 矩阵，第 0 行和第 0 列全 0（空串边界）
2. **逐行填表**：`i` 从 1 到 `m`，`j` 从 1 到 `n`：
   - `text1[i-1] == text2[j-1]` → `dp[i][j] = dp[i-1][j-1] + 1`
   - `text1[i-1] != text2[j-1]` → `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`
3. **返回** `dp[m][n]`

> ⚠️ **填表顺序**：必须从左到右、从上到下。因为 `dp[i][j]` 依赖左上 (`dp[i-1][j-1]`)、正上 (`dp[i-1][j]`)、左方 (`dp[i][j-1]`)，三者都要先算出来。这正是二维 DP 的典型依赖方向。

### 2.4 示例演算

以 `text1 = "abcde"`, `text2 = "ace"` 为例：

![示例演算：abcde 与 ace 的 dp 填表，答案 dp[5][3]=3](images/lcs_example_walkthrough.svg)

**dp 表**（行 = text1 字符，列 = text2 字符，首行/首列为空串边界）：

| | `""` | `a` | `c` | `e` |
|------|------|-----|-----|-----|
| `""` | 0 | 0 | 0 | 0 |
| `a` | 0 | **1** | 1 | 1 |
| `b` | 0 | 1 | 1 | 1 |
| `c` | 0 | 1 | **2** | 2 |
| `d` | 0 | 1 | 2 | 2 |
| `e` | 0 | 1 | 2 | **3** |

**关键填表步骤**：

| 格子 | 字符比较 | 转移 | 值 | 说明 |
|------|----------|------|----|------|
| `dp[1][1]` | `a` == `a` | `dp[0][0]+1` | 1 | 匹配，LCS="a" |
| `dp[1][2]` | `a` ≠ `c` | `max(dp[0][2], dp[1][1])` | 1 | 跳过 `c`，沿用 "a" |
| `dp[3][2]` | `c` == `c` | `dp[2][1]+1` | 2 | 匹配，LCS="ac" |
| `dp[5][3]` | `e` == `e` | `dp[4][2]+1` | 3 | 匹配，LCS="ace" ✓ |

最终 `dp[5][3] = 3`，最长公共子序列为 `"ace"`。

> 💡 **如何还原 LCS 字符串？** 从 `dp[m][n]` 回溯：若 `text1[i-1] == text2[j-1]` 且 `dp[i][j] == dp[i-1][j-1] + 1`，该字符在 LCS 中，记录并走到 `dp[i-1][j-1]`；否则走到 `dp[i-1][j]` 或 `dp[i][j-1]`（值较大的方向）。逆序拼接即得 LCS。

## 3. 参考代码

### C++

```cpp
// 最长公共子序列.cpp —— 二维 DP
// 编译: g++ -O2 -std=c++17 最长公共子序列.cpp -o lcs
class Solution {
public:
    int longestCommonSubsequence(string text1, string text2) {
        int m = text1.size(), n = text2.size();
        vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0));

        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (text1[i-1] == text2[j-1]) {
                    dp[i][j] = dp[i-1][j-1] + 1;         // 字符匹配
                } else {
                    dp[i][j] = max(dp[i-1][j], dp[i][j-1]); // 跳过其一
                }
            }
        }
        return dp[m][n];
    }
};
```

### Python

```python
class Solution:
    def longestCommonSubsequence(self, text1: str, text2: str) -> int:
        m, n = len(text1), len(text2)
        dp = [[0] * (n + 1) for _ in range(m + 1)]

        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if text1[i-1] == text2[j-1]:
                    dp[i][j] = dp[i-1][j-1] + 1          # 字符匹配
                else:
                    dp[i][j] = max(dp[i-1][j], dp[i][j-1])  # 跳过其一

        return dp[m][n]
```

> 💡 **实现细节**：① `dp` 矩阵大小 `(m+1)×(n+1)`，多出来的首行首列表示空串边界，全初始化为 0。② 循环从 1 开始，因为 0 行/列是边界。③ `text1[i-1]` 对应 `dp[i]` 行——下标偏移 1 是因为 `dp[0]` 代表空串。

## 4. 复杂度分析

| 维度 | 二维 DP | 滚动数组 |
|------|---------|---------|
| **时间** | `O(m × n)` | `O(m × n)` |
| **空间** | `O(m × n)` | `O(n)` |
| **能否还原 LCS 串** | ✓ 回溯 dp 表 | ✗ 只得长度 |

> ⚠️ 时间 `O(m×n)`：双重循环遍历 `(m+1)×(n+1)` 个格子，每个格子 `O(1)` 转移。`m, n ≤ 1000` 时约 $10^6$ 次操作，可接受。

## 5. 扩展：滚动数组空间优化

`dp[i][j]` 只依赖上一行（`dp[i-1][j-1]`、`dp[i-1][j]`）和当前行左边（`dp[i][j-1]`），可用**一维数组 + prev 变量**压缩到 `O(n)` 空间：

```python
class Solution:
    def longestCommonSubsequence(self, text1: str, text2: str) -> int:
        m, n = len(text1), len(text2)
        dp = [0] * (n + 1)

        for i in range(1, m + 1):
            prev = dp[0]                                  # dp[i-1][0] = 0
            for j in range(1, n + 1):
                temp = dp[j]                              # 保存 dp[i-1][j]
                if text1[i-1] == text2[j-1]:
                    dp[j] = prev + 1                      # dp[i-1][j-1] + 1
                else:
                    dp[j] = max(dp[j], dp[j-1])           # max(dp[i-1][j], dp[i][j-1])
                prev = temp                               # 滚动 prev
        return dp[n]
```

> 💡 **prev 的作用**：`dp[j]` 计算需要左上角 `dp[i-1][j-1]`，但一维数组中 `dp[j-1]` 已被更新为当前行的值，左上角丢失。`prev` 在覆盖 `dp[j]` 前保存旧值（上一行的 `dp[j]`），下一次迭代 `j+1` 时作为 `dp[i-1][j]`（即新的左上角）使用。与编辑距离的滚动数组技巧完全一致。

### 与编辑距离的关系

LCS 与 [72. 编辑距离](../../week8/day4/编辑距离.md) 是同构的二维 DP，转移结构对比：

| | 字符匹配时 | 字符不匹配时 | 目标 |
|------|-----------|-------------|------|
| **LCS** | `dp[i-1][j-1] + 1` | `max(dp[i-1][j], dp[i][j-1])` | 求**最长** |
| **编辑距离** | `dp[i-1][j-1]` | `1 + min(dp[i-1][j-1], dp[i-1][j], dp[i][j-1])` | 求**最少** |

若编辑距离只允许插入和删除（不允许替换），则 $\text{edit\_distance} = m + n - 2 \times \text{LCS}$。

## 6. 面试要点

1. **`dp[i][j]` 的定义是什么？为什么下标从 1 开始？**

   > `dp[i][j]` = `text1[0..i-1]` 与 `text2[0..j-1]` 的 LCS 长度。下标从 1 开始是因为 `dp[0][j]` 和 `dp[i][0]` 代表空串边界（LCS=0），需要单独占一行/列。`text1[i-1]` 对应 `dp[i]` 行——下标偏移 1。

2. **字符不匹配时为什么取 `max(dp[i-1][j], dp[i][j-1])` 而非 `dp[i-1][j-1]`？**

   > 不匹配意味着 `text1[i-1]` 和 `text2[j-1]` 至少有一个不在 LCS 中。`dp[i-1][j]` = 跳过 `text1[i-1]` 的 LCS，`dp[i][j-1]` = 跳过 `text2[j-1]` 的 LCS，取较大值。不取 `dp[i-1][j-1]`（两者都跳过）是因为它 $\leq$ 前两者（跳过一个的子问题包含跳过两个的）。

3. **字符匹配时为什么直接 `dp[i-1][j-1]+1` 而不取 `max`？**

   > 因为 `dp[i-1][j-1]+1` 恒 $\geq$ `max(dp[i-1][j], dp[i][j-1])`。匹配一对字符至少不比跳过差。所以匹配时无需比较，直接 `+1` 更高效。

4. **子序列和子串有什么区别？**

   > 子序列可以**不连续**（删除若干字符，保持相对顺序），子串必须**连续**。本题求子序列，DP 转移允许 `max(dp[i-1][j], dp[i][j-1])`（跳过中间字符）；若求最长公共子串，不匹配时 `dp[i][j]=0`（连续中断），不能用 `max`。

5. **如何还原 LCS 字符串（而非仅求长度）？**

   > 从 `dp[m][n]` 回溯：若 `text1[i-1]==text2[j-1]` 且 `dp[i][j]==dp[i-1][j-1]+1`，记录该字符并走到 `[i-1][j-1]`；否则走到 `dp[i-1][j]` 或 `dp[i][j-1]` 中较大者。逆序拼接即得 LCS。滚动数组无法回溯（丢失了历史行），需用完整二维表。

> 💡 **一句话总结**：1143 是二维 DP 的经典模板——`dp[i][j]` 表示两前缀的 LCS 长度，匹配时 `+1`，不匹配时取 `max(上方, 左方)`。`O(m×n)` 时间，滚动数组可压到 `O(n)` 空间。这个「两序列逐位置交互」的二维 DP 模板可迁移到编辑距离（72）、最长重复子数组（718）、不同的子序列（115）等一整类问题，是面试必会的核心套路。
