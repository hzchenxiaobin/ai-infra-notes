# 不同路径

- **题目名称**：不同路径
- **链接**：[62. 不同路径](https://leetcode.cn/problems/unique-paths/)
- **难度**：中等
- **标签**：动态规划、组合数学

## 1. 题目概述

一个机器人位于一个 `m x n` 网格的左上角（起始点在下图中标记为 "Start" ）。

机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 "Finish" ）。

问总共有多少条不同的路径？

**示例 1**：

```text
输入：m = 3, n = 7
输出：28
```

**示例 2**：

```text
输入：m = 3, n = 2
输出：3
解释：
从左上角开始，总共有 3 条路径可到达右下角：
1. 向右 -> 向下 -> 向下
2. 向下 -> 向下 -> 向右
3. 向下 -> 向右 -> 向下
```

**约束条件**：

- `1 <= m, n <= 100`
- 题目数据保证答案小于等于 `2 * 10^9`

> 💡 这是 **二维动态规划** 的入门题。机器人只能向右或向下移动，意味着到达每个格子的路径数只依赖于它上方和左方格子的路径数。状态转移方程非常清晰：`dp[i][j] = dp[i-1][j] + dp[i][j-1]`。

---

## 2. 解题思路

### 2.1 动态规划

设 `dp[i][j]` 表示从左上角到达格子 `(i, j)` 的不同路径数。

- **初始状态**：第一行和第一列都只有一条路径到达（只能一直向右或一直向下），所以 `dp[i][0] = 1`，`dp[0][j] = 1`。
- **状态转移**：对于其他格子 `(i, j)`，机器人可以从上方 `(i-1, j)` 或左方 `(i, j-1)` 到达，因此：
  ```
  dp[i][j] = dp[i-1][j] + dp[i][j-1]
  ```
- **答案**：`dp[m-1][n-1]`

### 2.2 空间优化

由于每个状态只依赖于上一行和当前行，可以将二维数组优化为一维滚动数组：

```
dp[j] = dp[j] + dp[j-1]
```

其中 `dp[j]` 在更新前代表上一行第 `j` 列的值，`dp[j-1]` 代表当前行第 `j-1` 列的值。

## 3. 参考代码

### 3.1 二维 DP

```cpp
class Solution {
  public:
    int uniquePaths(int m, int n) {
        vector<vector<int>> dp(m, vector<int>(n, 1));
        for (int i = 1; i < m; ++i) {
            for (int j = 1; j < n; ++j) {
                dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
            }
        }
        return dp[m - 1][n - 1];
    }
};
```

### 3.2 一维滚动数组优化

```cpp
class Solution {
  public:
    int uniquePaths(int m, int n) {
        vector<int> dp(n, 1);
        for (int i = 1; i < m; ++i) {
            for (int j = 1; j < n; ++j) {
                dp[j] += dp[j - 1];
            }
        }
        return dp[n - 1];
    }
};
```

### 3.3 组合数学

从左上角到右下角，无论怎么走，都需要向右走 `n-1` 步、向下走 `m-1` 步，总共 `m+n-2` 步。问题等价于在这么多步中选择哪些步向右（或向下）：

```cpp
class Solution {
  public:
    int uniquePaths(int m, int n) {
        long long ans = 1;
        int N = m + n - 2;
        int k = min(m - 1, n - 1);
        for (int i = 1; i <= k; ++i) {
            ans = ans * (N - k + i) / i;
        }
        return (int)ans;
    }
};
```

## 4. 复杂度分析

- **时间复杂度**：
  - 二维 DP：`O(m * n)`
  - 一维滚动数组：`O(m * n)`
  - 组合数学：`O(min(m, n))`
- **空间复杂度**：
  - 二维 DP：`O(m * n)`
  - 一维滚动数组：`O(n)`
  - 组合数学：`O(1)`

---

## 5. 面试要点

1. **为什么初始化为 1**：第一行和第一列任何一个格子都只有一条路径到达。
2. **滚动数组的理解**：`dp[j]` 在更新前是上一行的值，更新后是当前行的值；`dp[j-1]` 已经提前更新为当前行的值。
3. **组合数学的适用场景**：当只需要计数而不关心具体路径时，组合数公式可以提供 `O(min(m,n))` 的解法，但要注意整数溢出。
