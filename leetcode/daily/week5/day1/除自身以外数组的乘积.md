# 除自身以外数组的乘积

- **题目名称**：除自身以外数组的乘积
- **链接**：[238. 除自身以外数组的乘积](https://leetcode.cn/problems/product-of-array-except-self/)
- **难度**：中等
- **标签**：数组、前缀和（前缀积）

## 1. 题目概述

给定一个整数数组 `nums`，返回数组 `answer`，其中 `answer[i]` 等于 `nums` 中除 `nums[i]` 之外其余各元素的乘积。**要求 O(n) 时间且不使用除法**。

**示例 1**：

```text
输入：nums = [1,2,3,4]
输出：[24,12,8,6]
解释：
  answer[0] = 2*3*4 = 24
  answer[1] = 1*3*4 = 12
  answer[2] = 1*2*4 = 8
  answer[3] = 1*2*3 = 6
```

**示例 2**：

```text
输入：nums = [-1,1,0,-3,3]
输出：[0,0,9,0,0]
```

**约束条件**：

- `2 <= nums.length <= 10^5`
- `-30 <= nums[i] <= 30`
- **保证** `nums` 中任意前缀/后缀的乘积在 32 位整数范围内
- 进阶：O(1) 额外空间（输出数组不计）

> 💡 题目明确"不用除法"，是为了堵住"先求总乘积再逐个除"的取巧做法（而且一旦数组里有 0，除法就直接失效）。这逼着我们用**前缀积**思想。

---

## 2. 解题思路

### 2.1 暴力思路

对每个位置 `i`，把 `nums[0..i-1]` 和 `nums[i+1..n-1]` 各乘一遍。时间复杂度 `O(n²)`，`n = 10^5` 时**超时**。

```text
for i in range(n):
    ans[i] = 1
    for j in range(n):
        if j != i:
            ans[i] *= nums[j]
```

> ⚠️ 暴力法的瓶颈：对每个 `i` 都重复扫了"左边"和"右边"的所有元素，存在大量重复乘法。

### 2.2 核心观察：前缀积 × 后缀积

![前缀积 × 后缀积：answer[i] = left[i] × right[i]](images/product_except_self_prefix_suffix.svg)

关键洞察：`answer[i]` **=** `nums[0..i-1]` **的乘积 ×** `nums[i+1..n-1]` **的乘积**。把它拆成两段：

- `left[i]` = `nums[0..i-1]` 的乘积（即 `i` 左边所有元素的积，边界 `left[0] = 1`）
- `right[i]` = `nums[i+1..n-1]` 的乘积（即 `i` 右边所有元素的积，边界 `right[n-1] = 1`）

则 `answer[i] = left[i] × right[i]`。

两段都可以用**一遍扫描递推**得到：

```text
正序扫：left[i]   = left[i-1]   × nums[i-1]     （i 从 1 到 n-1）
逆序扫：right[i]  = right[i+1]  × nums[i+1]     （i 从 n-2 到 0）
```

> 💡 这相当于把"对每个 `i` 重复扫左右"的两维，压缩成"正序扫一遍算左积、逆序扫一遍算右积"——时间从 `O(n²)` 降到 `O(n)`。本质和 [Week1 Day2 最大子数组和](../week1/day2/最大子数组和.md) 的前缀和思想同源：**用预计算把重复扫描压成常数次遍历**。

### 2.3 算法流程

1. **正序一遍**：用 `answer[i]` 先存 `left[i]`，`answer[0]=1`，`answer[i] = answer[i-1] × nums[i-1]`。
2. **逆序一遍**：维护一个滚动变量 `R = 1`（即当前的 `right[i]`），从右往左扫，`answer[i] = answer[i] × R`，然后 `R = R × nums[i]`。
3. 返回 `answer`。

这样**省掉了** `right[]` **数组**，额外空间 `O(1)`（输出数组不计）。

### 2.4 示例演算

以 `nums = [1, 2, 3, 4]` 为例：

![示例演算：nums=[1,2,3,4] 的 left/right/answer](images/product_except_self_walkthrough.svg)

| i | nums[i] | left[i] | right[i] | answer[i] |
|---|---------|---------|----------|-----------|
| 0 | 1       | 1       | 2·3·4=24 | 1×24=24   |
| 1 | 2       | 1       | 3·4=12   | 1×12=12   |
| 2 | 3       | 1·2=2   | 4        | 2×4=8     |
| 3 | 4       | 1·2·3=6 | 1        | 6×1=6     |

最终 `answer = [24, 12, 8, 6]`。

## 3. 参考代码

### C++（O(1) 额外空间滚动写法）

```cpp
class Solution {
  public:
    vector<int> productExceptSelf(vector<int>& nums) {
        int n = nums.size();
        vector<int> answer(n, 1);

        // ① 正序：answer[i] = left[i] = nums[0..i-1] 的乘积
        for (int i = 1; i < n; ++i) {
            answer[i] = answer[i - 1] * nums[i - 1];
        }

        // ② 逆序：用滚动变量 R 累积右积，直接乘进 answer
        int R = 1;
        for (int i = n - 1; i >= 0; --i) {
            answer[i] *= R; // answer[i] = left[i] * right[i]
            R *= nums[i];   // R 推进到 right[i-1]
        }
        return answer;
    }
};
```

### Python

```python
class Solution:
    def productExceptSelf(self, nums: List[int]) -> List[int]:
        n = len(nums)
        answer = [1] * n

        # ① 正序：answer[i] = left[i]
        for i in range(1, n):
            answer[i] = answer[i - 1] * nums[i - 1]

        # ② 逆序：滚动 R 累积右积
        R = 1
        for i in range(n - 1, -1, -1):
            answer[i] *= R
            R *= nums[i]

        return answer
```

> 💡 也可以写显式 `left[]` + `right[]` 两数组的版本，逻辑更直观但多 `O(n)` 空间。面试时先讲清两数组版本，再优化成滚动版，体现"先正确再优化空间"的思路。

## 4. 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| 时间复杂度 | O(n) | 正序扫一遍 + 逆序扫一遍，共两次线性遍历 |
| 空间复杂度 | O(1) | 仅用一个滚动变量 `R`（输出数组 `answer` 按题意不计入额外空间） |

> ⚠️ 注意：若用显式 `left[]` + `right[]`，空间是 `O(n)`；面试要求"进阶 O(1) 额外空间"时必须用滚动写法。

## 5. 扩展：含 0 的情形与乘积溢出

- **含 0**：本算法天然处理 0——一旦某个 `nums[i]=0`，所有 `left[j> i]` 都会是 0，`right[j< i]` 也都会是 0，只有 `answer[i]`（除掉这个 0）可能非 0。这正是题目禁用除法的原因：除法遇到 0 会除零错误，而前缀积法无需特殊判断。
- **多个 0**：若有两个及以上 0，则所有 `answer[i]` 都是 0（因为除掉任一个 0 后，剩下的乘积里仍含 0）。本算法同样自动得到正确结果。
- **溢出**：题目保证任意前缀/后缀乘积在 32 位范围内，所以中间的 `left[i]`、`R` 不会溢出；若放宽约束（如 `nums[i]` 可达 1e9），需改用 `long long`。

## 6. 面试要点

1. **为什么不能用"总乘积 ÷ nums[i]"？**

   - 一是题目明确禁用除法；二是数组里出现 0 时除法失效（除零错误），且无法区分"一个 0"和"多个 0"的情况（前者只有 `answer[i]` 非 0，后者全 0）。前缀积法对 0 天然正确，无需特判。

2. **如何把空间从 O(n) 优化到 O(1)？**

   - 先用 `answer` 数组存 `left[]`（正序递推），再用一个滚动变量 `R` 边逆序扫边把 `right` 乘进去——`right[i]` 只依赖 `right[i+1]`，可压缩成单变量。输出数组不计入额外空间，故为 O(1)。

3. **这题和前缀和是什么关系？**

   - 把"求和"换成"求积"就是同一套前缀/后缀预计算模板：`prefix[i] = prefix[i-1] × nums[i-1]`。本质都是"用 O(n) 预计算消除重复扫描，把区间查询从 O(n) 降到 O(1)"。最大子数组和、一维前缀和、本题同源。

4. **正序/逆序两遍扫描的顺序能换吗？**

   - 可以。可以先逆序算 `right[]` 存进 `answer`，再用滚动 `L` 正序乘进去。结果完全一样，只是 `left/right` 的角色互换。面试讲清"一遍存左积、一遍用滚动右积乘进去"即可。

5. **为什么这题和推理系统的 KV Cache 思想同构？**

   - 都是"**预计算/缓存历史结果，避免重复计算**"：本题把"每个位置左右乘积"预先存好，避免对每个 `i` 重复扫；KV Cache 把"历史 K/V"存好，避免 Decode 每步重算前缀。两者都是用空间换时间，把 O(n²) 的重复工作压成 O(n)。这是 Day1 推理系统优化的算法直觉之一。
