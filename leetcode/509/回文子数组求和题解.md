# 回文子数组求和

## 1. 题目概述

- **题目名称**：Q4. 回文子数组求和
- **链接**：LeetCode（周赛 Q4）
- **难度**：困难
- **标签**：Manacher、前缀和、回文、贪心

**题意简述**：

给你一个整数数组 `nums`，找出一个**回文子数组**的**最大元素和**，返回该和。

- 子数组是数组中连续的非空元素序列。
- 回文子数组指正着读和反着读都相同的子数组。
- 单元素子数组也是回文。

**约束条件**：

- `1 <= nums.length <= 10^5`
- `1 <= nums[i] <= 10^9`

---

## 2. 示例

**示例 1**

```text
输入：nums = [10,10]
输出：20
解释：整个数组 [10,10] 是回文子数组，和 = 10 + 10 = 20。
```

**示例 2**

```text
输入：nums = [1,2,3,2,1,5,6]
输出：9
解释：子数组 [1,2,3,2,1] 是回文，和 = 1+2+3+2+1 = 9，为最大。
```

**示例 3**

```text
输入：nums = [7,1,2,1,7,3,4,3,4]
输出：18
解释：子数组 [7,1,2,1,7] 是回文，和 = 7+1+2+1+7 = 18，为最大。
```

**示例 4**

```text
输入：nums = [1,2,3,4,5]
输出：5
解释：不存在长度 > 1 的回文子数组，最大单元素为 5。
```

**示例 5**

```text
输入：nums = [1000]
输出：1000
解释：单元素回文，答案为 1000。
```

## 3. 解题思路

### 3.1 关键观察：所有元素为正

题目约束 `1 <= nums[i] <= 10^9`，即**所有元素都是正数**。这带来一个关键推论：

> 对于固定的回文中心，**最长的回文子数组**一定拥有**最大的元素和**。

因为从中心向外扩展时，每多包一对元素（左右各一个），和只会增加（元素为正）。所以不需要枚举所有回文子数组，只需对每个中心找**最长回文**，再算它的和，取全局最大即可。

### 3.2 朴素方法：中心扩展 O(n²)

对每个中心（共 2n-1 个：n 个奇数中心 + n-1 个偶数中心），向两边扩展直到不回文，记录最长半径。总复杂度 O(n²)，n=10⁵ 会超时。

### 3.3 优化：Manacher 算法 O(n)

Manacher 算法利用**已计算的回文信息**加速后续中心的扩展，把 O(n²) 降到 O(n)。它对每个中心返回**最长回文半径**，再配合**前缀和** O(1) 算出每个最长回文的元素和，最终取最大值。

## 4. 算法细节

### 4.1 Manacher 算法

![Manacher 对称性加速原理](images/palindrome_manacher_symmetry.svg)

Manacher 维护两个数组：

| 数组 | 含义 | 最长回文子数组 |
|------|------|--------------|
| `d1[i]` | 以 `i` 为中心的**奇数长度**回文个数 | `[i-d1[i]+1, i+d1[i]-1]`，长度 `2·d1[i]-1` |
| `d2[i]` | 以 `i-1` 和 `i` 之间为中心的**偶数长度**回文个数 | `[i-d2[i], i+d2[i]-1]`，长度 `2·d2[i]` |

**奇数长度 Manacher**（计算 `d1`）：

```
维护右端最远的回文 [l, r]，中心 mr = (l+r)/2
对每个 i：
  若 i < r，利用对称性初始化 d1[i] = min(d1[2*mr - i], r - i)
  否则 d1[i] = 1（至少自身是回文）
  尝试扩展：while i-d1[i] >= 0 && i+d1[i] < n && a[i-d1[i]]==a[i+d1[i]]: d1[i]++
  若 i+d1[i]-1 > r，更新 l, r, mr
```

**偶数长度 Manacher**（计算 `d2`）：

```
维护右端最远的回文 [l, r]，中心 mr = (l+r+1)/2
对每个 i：
  若 i < r，初始化 d2[i] = min(d2[2*mr - i - 1], r - i)
  否则 d2[i] = 0
  尝试扩展：while i-d2[i]-1 >= 0 && i+d2[i] < n && a[i-d2[i]-1]==a[i+d2[i]]: d2[i]++
  若 i+d2[i]-1 > r，更新 l, r, mr
```

#### 代码逐行解析（以奇数长度 `d1` 为例）

下面把 C++ 参考代码中的 `d1` 循环逐行拆解，对应伪代码的每一步：

```cpp
// 初始化：l=左边界, r=右边界, 初始 r=-1 表示"还没有已知回文"
for (int i = 0, l = 0, r = -1; i < n; i++) {
```

> `r = -1` 是哨兵值，保证 `i > r` 对 `i=0` 成立，从而第一个中心从 `k=1` 开始朴素扩展。

**第 1 步：利用对称性初始化 k**

```cpp
    int k = (i > r) ? 1 : min(d1[l + r - i], r - i);
```

| 条件 | k 的值 | 对应情况 |
|------|-------|---------|
| `i > r`（i 在已知回文右边界外） | `1`（至少自身是回文） | 无对称信息可用，从 1 开始朴素扩展 |
| `i <= r`（i 在已知回文内） | `min(d1[l+r-i], r-i)` | 利用镜像 `mir = l+r-i` 的已计算值加速 |

> `l + r - i` 就是镜像下标 `mir = 2·mr - i`（因为 `mr = (l+r)/2`，`2·mr = l+r`）。`r - i` 是 i 到右边界的距离——镜像回文最多只能保证到 `r`，超出部分必须自己扩展。取 `min` 恰好覆盖"完全在内"和"超出边界"两种情况（见上方对称性原理图）。

**第 2 步：尝试扩展**

```cpp
    while (i - k >= 0 && i + k < n && nums[i - k] == nums[i + k]) k++;
```

从初始化的 `k` 开始，向左右各扩展一格。只要不越界且左右元素相等就继续。这一步是唯一做实际比较的地方——由于 `r` 单调递增，每个元素最多在此被比较 1 次，保证总 O(n)。

**第 3 步：记录结果 + 后处理 k**

```cpp
    d1[i] = k--;
```

`d1[i] = k` 记录最终半径。`k--` 把 `k` 从"半径"调整为"右端点偏移"（`i + k` 是回文右边界下标），用于下一步比较。

> 注意 `d1[i]` 的语义是"以 i 为中心的奇数回文个数"（含自身），所以半径为 `d1[i]` 的回文实际范围是 `[i-d1[i]+1, i+d1[i]-1]`，长度 `2·d1[i]-1`。

**第 4 步：更新最远右端回文**

```cpp
    if (i + k > r) { l = i - k; r = i + k; }
```

如果当前回文的右边界 `i+k` 超过了已知的 `r`，就更新 `[l, r]` 和隐含的 `mr`。这是 Manacher 的核心贪心：**始终维护右端最远的回文**，因为右端越远，后续 `i` 落在内部的概率越大，能复用的对称信息越多。

**偶数长度 `d2` 的差异**：

```cpp
    int k = (i > r) ? 0 : min(d2[l + r - i + 1], r - i + 1);
    while (i - k - 1 >= 0 && i + k < n && nums[i - k - 1] == nums[i + k]) k++;
    d2[i] = k--;
    if (i + k > r) { l = i - k - 1; r = i + k; }
```

| 差异点 | d1（奇数） | d2（偶数） |
|-------|-----------|-----------|
| 初始 k | `1`（自身是回文） | `0`（空回文，需扩展才有） |
| 镜像下标 | `l + r - i` | `l + r - i + 1`（中心在间隙，偏移 1） |
| 扩展比较 | `a[i-k] == a[i+k]` | `a[i-k-1] == a[i+k]`（左侧多偏移 1） |
| 更新 l | `l = i - k` | `l = i - k - 1`（左边界多偏移 1） |
| 回文范围 | `[i-d1+1, i+d1-1]` | `[i-d2, i+d2-1]` |

> 偶数长度的"中心"定义在 `i-1` 和 `i` 之间，所以左侧下标比右侧多偏移 1。这是 d2 比 d1 容易写错的地方——前缀和求和时也要注意对应的范围。

#### 为什么 `r` 单调递增保证 O(n)？

每次 `while` 扩展成功时，`i + k` 严格递增且一定 `> r`（因为初始化 `k` 已保证 `[i-k+1, i+k-1]` 在 `[l, r]` 内，扩展只在 `r` 之外发生）。更新后新 `r = i + k` 比旧 `r` 大。由于 `r` 从 -1 单调递增到最多 `n-1`，`while` 循环体总共执行不超过 `n` 次。加上外层 `for` 的 `n` 次迭代，总复杂度 **O(n)**。

### 4.2 前缀和求回文子数组和

预处理前缀和 `pref[i] = nums[0] + ... + nums[i-1]`（`pref[0]=0`），则子数组 `[l, r]` 的和 = `pref[r+1] - pref[l]`。

对每个中心的最长回文：

| 类型 | 回文范围 | 和 |
|------|---------|---|
| 奇数，中心 i | `[i-d1[i]+1, i+d1[i]-1]` | `pref[i+d1[i]] - pref[i-d1[i]+1]` |
| 偶数，中心 i | `[i-d2[i], i+d2[i]-1]` | `pref[i+d2[i]] - pref[i-d2[i]]` |

### 4.3 完整算法

```
1. 计算前缀和 pref[0..n]
2. Manacher 求 d1[0..n-1]（奇数长度最长回文半径）
3. Manacher 求 d2[0..n-1]（偶数长度最长回文半径）
4. ans = 0
   对每个 i：ans = max(ans, pref[i+d1[i]] - pref[i-d1[i]+1])       // 奇数
   对每个 i：若 d2[i]>0，ans = max(ans, pref[i+d2[i]] - pref[i-d2[i]])  // 偶数
5. 返回 ans
```

## 5. 正确性证明

**引理**：当所有 `nums[i] > 0` 时，对固定中心 `c`，最长回文子数组的元素和 ≥ 该中心任何更短回文子数组的元素和。

**证明**：设中心 `c` 的最长回文半径为 `R`。对任意 `r < R`，半径 `r` 的回文子数组是半径 `R` 的子数组去掉外层 `R-r` 对元素。由于每对元素 `nums[c-r-1] + nums[c+r+1] > 0`（元素为正），去掉后和减小。故最长回文和最大。□

**推论**：全局最大回文子数组和 = `max over all centers c of sum(longest palindrome at c)`。Manacher 给出每个中心的最长回文，前缀和 O(1) 算和，取 max 即可。□

---

## 6. 复杂度分析

- **时间复杂度**：`O(n)`。Manacher 算法每个元素最多被访问常数次（扩展时 `r` 单调递增），前缀和 O(n)，取 max O(n)。
- **空间复杂度**：`O(n)`。`d1`、`d2`、`pref` 各 O(n)。

## 7. 参考代码

### C++

```cpp
class Solution {
public:
    long long maxPalindromicSum(vector<int>& nums) {
        int n = nums.size();
        // 前缀和
        vector<long long> pref(n + 1, 0);
        for (int i = 0; i < n; i++) pref[i + 1] = pref[i] + nums[i];

        // Manacher 奇数长度 d1[i]
        vector<int> d1(n);
        for (int i = 0, l = 0, r = -1; i < n; i++) {
            int k = (i > r) ? 1 : min(d1[l + r - i], r - i);
            while (i - k >= 0 && i + k < n && nums[i - k] == nums[i + k]) k++;
            d1[i] = k--;
            if (i + k > r) { l = i - k; r = i + k; }
        }

        // Manacher 偶数长度 d2[i]
        vector<int> d2(n);
        for (int i = 0, l = 0, r = -1; i < n; i++) {
            int k = (i > r) ? 0 : min(d2[l + r - i + 1], r - i + 1);
            while (i - k - 1 >= 0 && i + k < n && nums[i - k - 1] == nums[i + k]) k++;
            d2[i] = k--;
            if (i + k > r) { l = i - k - 1; r = i + k; }
        }

        // 取最大和
        long long ans = 0;
        for (int i = 0; i < n; i++) {
            // 奇数：[i-d1[i]+1, i+d1[i]-1]，和 = pref[i+d1[i]] - pref[i-d1[i]+1]
            ans = max(ans, pref[i + d1[i]] - pref[i - d1[i] + 1]);
            // 偶数：[i-d2[i], i+d2[i]-1]，和 = pref[i+d2[i]] - pref[i-d2[i]]
            if (d2[i] > 0)
                ans = max(ans, pref[i + d2[i]] - pref[i - d2[i]]);
        }
        return ans;
    }
};
```

### Python

```python
class Solution:
    def maxPalindromicSum(self, nums: List[int]) -> int:
        n = len(nums)
        # 前缀和
        pref = [0] * (n + 1)
        for i in range(n):
            pref[i + 1] = pref[i] + nums[i]

        # Manacher 奇数长度
        d1 = [0] * n
        l, r = 0, -1
        for i in range(n):
            k = 1 if i > r else min(d1[l + r - i], r - i)
            while i - k >= 0 and i + k < n and nums[i - k] == nums[i + k]:
                k += 1
            d1[i] = k
            k -= 1
            if i + k > r:
                l, r = i - k, i + k

        # Manacher 偶数长度
        d2 = [0] * n
        l, r = 0, -1
        for i in range(n):
            k = 0 if i > r else min(d2[l + r - i + 1], r - i + 1)
            while i - k - 1 >= 0 and i + k < n and nums[i - k - 1] == nums[i + k]:
                k += 1
            d2[i] = k
            k -= 1
            if i + k > r:
                l, r = i - k - 1, i + k

        # 取最大和
        ans = 0
        for i in range(n):
            ans = max(ans, pref[i + d1[i]] - pref[i - d1[i] + 1])
            if d2[i] > 0:
                ans = max(ans, pref[i + d2[i]] - pref[i - d2[i]])
        return ans
```

## 8. 边界情况与易错点

1. **单元素数组**：`n=1`，`d1[0]=1`，奇数回文覆盖，返回 `nums[0]`。

2. **无长度 > 1 的回文**（示例 4）：所有 `d1[i]=1`（仅自身），`d2[i]=0`，答案是最大单元素。代码自动处理（奇数回文长度 1 的和 = `nums[i]`）。

3. **整数溢出**：`nums[i]` 可达 10⁹，n=10⁵，最大和可达 10¹⁴。C++ 必须用 `long long`，Python 无此问题。

4. **Manacher 偶数长度的中心定义**：`d2[i]` 的中心在 `i-1` 和 `i` 之间，回文范围是 `[i-d2[i], i+d2[i]-1]`，不是 `[i-d2[i]+1, i+d2[i]]`。前缀和下标易错。

5. **前缀和下标**：`pref[j]` = 前 j 个元素的和（`pref[0]=0`）。子数组 `[l,r]` 的和 = `pref[r+1] - pref[l]`。代入回文范围时注意 +1 偏移。

6. **所有元素为正是关键前提**：若元素可能为负，最长回文不一定和最大（外层可能是一对负数），本题不适用。需改用更复杂的处理。

- 如果 `nums[i]` 可以为负数，该怎么做？
  - 最长回文不再保证和最大。需要对每个中心枚举所有回文半径（用 Manacher 的 `d1/d2` 知道哪些半径有效），结合前缀和求最大——但最坏仍 O(n²)。可考虑分治或线段树优化。
