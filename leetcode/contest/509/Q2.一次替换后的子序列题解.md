# 一次替换后的子序列

## 1. 题目概述

- **题目名称**：Q2. 一次替换后的子序列
- **链接**：[3983. 一次替换后的子序列](https://leetcode.cn/problems/subsequence-after-one-replacement/)
- **难度**：中等
- **标签**：贪心、双指针、子序列、预处理

**题意简述**：

给你两个由小写英文字母组成的字符串 `s` 和 `t`。你最多可以选择 `s` 中的一个下标，将该下标处的字符替换为任意小写英文字母。如果可以使 `s` 成为 `t` 的一个子序列，则返回 `true`；否则返回 `false`。

**约束条件**：

- `1 <= s.length, t.length <= 10^5`
- `s` 和 `t` 仅由小写英文字母组成

---

## 2. 示例

**示例 1**

```text
输入：s = "cat", t = "chat"
输出：true
解释：将 s[1] 从 'a' 替换为 'h'，得到 "cht"。"cht" 是 "chat" 的子序列。
```

**示例 2**

```text
输入：s = "plane", t = "apple"
输出：false
解释：即使替换 s 中任意一个字符，也无法使 s 成为 t 的子序列。
```

---

## 3. 解题思路

### 3.1 朴素思路：枚举替换位置 + 子序列匹配

对 `s` 的每个位置 `i`，把 `s[i]` 替换成任意字符后检查 `s` 是否为 `t` 的子序列。但替换成"任意字符"无法枚举，朴素做法 O(n²) 会超时。

### 3.2 优化：前后缀预处理 + 枚举断点

**关键观察**：子序列匹配是"从左到右贪心匹配"。把 `s` 分成三段：

```
s = [s[0..i-1]] [s[i]] [s[i+1..n-1]
      前缀        替换点    后缀
```

- **前缀** `s[0..i-1]` 在 `t` 中从左到右贪心匹配，匹配到 `t` 的位置 `pre[i]`
- **后缀** `s[i+1..n-1]` 在 `t` 中从右到左贪心匹配，匹配到 `t` 的位置 `suf[i]`
- **替换点** `s[i]` 替换为任意字符，只需在 `t[pre[i]..suf[i]-1]` 中存在一个位置放下即可

**判定条件**：对替换点 `i`，若 `pre[i] < suf[i]`（即前缀和后缀的匹配区间在 `t` 中留有至少一个字符的空隙），则替换 `s[i]` 后 `s` 是 `t` 的子序列 → `true`。

**特殊情况**：
- 不替换（替换 0 次）：直接检查 `s` 是否为 `t` 的子序列
- `s` 长度为 1：只要 `t` 非空，替换 `s[0]` 为 `t` 的任意字符即可 → `true`

### 3.3 算法流程

![前后缀预处理 + 枚举断点算法流程](images/one_replace_subsequence_flow.svg)

```
1. 前缀预处理 pre[i]：s[0..i-1] 在 t 中贪心匹配后，t 的下一个待匹配位置
2. 后缀预处理 suf[i]：s[i+1..n-1] 在 t 中逆向匹配后，t 的已匹配最小位置
3. 枚举替换点 i：
   若 pre[i] < suf[i] → 替换 s[i] 可填入空隙 → true
4. 若所有 i 都不满足，返回 false
```

## 4. 算法细节

### 4.1 前缀预处理 `pre[i]`

`pre[i]` 表示 `s[0..i-1]` 这 `i` 个字符在 `t` 中从左到右贪心匹配后，`t` 中下一个待匹配位置（即已匹配了 `i` 个字符）。

```
pre[0] = 0  （s 的前 0 个字符，匹配了 t 的 0 个字符）
pre[i] = pre[i-1] 之后 t 中第一个等于 s[i-1] 的位置 + 1
```

用双指针：`j` 在 `t` 上推进，对 `s` 的每个字符找 `t` 中第一个匹配。

### 4.2 后缀预处理 `suf[i]`

`suf[i]` 表示 `s[i+1..n-1]` 在 `t` 中从右到左贪心匹配后，`t` 中已匹配的最小位置。

```
suf[n] = t.length  （s 的后 0 个字符，t 全部可用）
suf[i] = suf[i+1] 之前 t 中最后一个等于 s[i] 的位置
```

从右到左：`j` 在 `t` 上从右往左退，对 `s` 的每个字符从右往左找 `t` 中最后一个匹配。

### 4.3 枚举替换点

对每个 `i`（`0 <= i < n`），替换 `s[i]` 后：
- 前 `i` 个字符匹配到 `t[0..pre[i]-1]`
- 后 `n-1-i` 个字符匹配到 `t[suf[i]..t.length-1]`
- `s[i]`（替换后）需要在 `t[pre[i]..suf[i]-1]` 中占一个位置

**条件**：`pre[i] < suf[i]`（空隙中至少有一个字符可放）

> ⚠️ **注意**：这里 `pre[i]` 是"前 i 个字符匹配后 t 的下一个位置"，`suf[i]` 是"后 n-1-i 个字符匹配时 t 的起始位置"。两者之间是 `s[i]` 可放置的范围。

### 4.4 不替换的情况

还需考虑"不替换任何字符"的情况——即 `s` 本身就是 `t` 的子序列。这等价于 `pre[n] <= t.length`（前缀预处理中 `s` 全部匹配完）。

---

## 5. 正确性证明

**引理**：若存在替换点 `i` 使得 `pre[i] < suf[i]`，则替换 `s[i]` 为 `t[pre[i]]` 后 `s` 是 `t` 的子序列。

**证明**：
- 前 `i` 个字符 `s[0..i-1]` 贪心匹配到 `t[0..pre[i]-1]`，由 `pre` 定义保证
- 替换后的 `s[i]` 放在 `t[pre[i]]`，因为 `pre[i] < suf[i]`，所以 `pre[i]` 在后缀匹配区间之前
- 后 `n-1-i` 个字符 `s[i+1..n-1]` 从 `t[suf[i]` 开始逆向匹配到 `t[t.length-1]`，由 `suf` 定义保证
- 三段不重叠（`pre[i] - 1 < pre[i] < suf[i]`，前缀末尾、替换位、后缀起始依次递进），且顺序正确 → `s` 是 `t` 的子序列 □

**完备性**：若所有 `i` 都不满足 `pre[i] < suf[i]`，且 `s` 本身不是 `t` 的子序列，则替换任意一个字符都无法弥补前后缀之间的空隙 → `false` □

---

## 6. 复杂度分析

- **时间复杂度**：`O(n + m)`，其中 `n = s.length`，`m = t.length`。前缀预处理 O(n + m)，后缀预处理 O(n + m)，枚举断点 O(n)。
- **空间复杂度**：`O(n)`。`pre` 和 `suf` 各 O(n)。

## 7. 参考代码

### C++

```cpp
class Solution {
  public:
    bool canMakeSubsequence(string s, string t) {
        int n = s.size(), m = t.size();
        // 不替换的情况：s 本身是 t 的子序列
        // 前后缀预处理会覆盖这种情况（pre[n] <= m 时 i=n 的"虚拟替换点"成立）
        // 但为清晰起见，统一走前后缀流程

        // 前缀预处理：pre[i] = s[0..i-1] 匹配后 t 的下一个位置
        vector<int> pre(n + 1);
        pre[0] = 0;
        for (int i = 0, j = 0; i < n; i++) {
            while (j < m && t[j] != s[i])
                j++;
            if (j < m) {
                j++;
                pre[i + 1] = j;
            } else {
                pre[i + 1] = m + 1;
            } // 无法匹配，标记为越界
        }

        // 不替换直接是子序列
        if (pre[n] <= m)
            return true;

        // 后缀预处理：suf[i] = s[i..n-1] 逆向匹配时 t 的起始位置
        vector<int> suf(n + 1);
        suf[n] = m;
        for (int i = n - 1, j = m - 1; i >= 0; i--) {
            while (j >= 0 && t[j] != s[i])
                j--;
            if (j >= 0) {
                suf[i] = j;
                j--;
            } else {
                suf[i] = -1;
            } // 无法匹配
        }

        // 枚举替换点 i：替换 s[i]，前 i 个用 pre，后 n-1-i 个用 suf
        for (int i = 0; i < n; i++) {
            // pre[i] = s[0..i-1] 匹配后 t 的位置
            // suf[i+1] = s[i+1..n-1] 匹配时 t 的起始位置
            if (pre[i] < suf[i + 1])
                return true;
        }
        return false;
    }
};
```

### Python

```python
class Solution:
    def canMakeSubsequence(self, s: str, t: str) -> bool:
        n, m = len(s), len(t)

        # 前缀预处理：pre[i] = s[0..i-1] 匹配后 t 的下一个位置
        pre = [0] * (n + 1)
        j = 0
        for i in range(n):
            while j < m and t[j] != s[i]:
                j += 1
            if j < m:
                j += 1
                pre[i + 1] = j
            else:
                pre[i + 1] = m + 1  # 无法匹配

        # 不替换直接是子序列
        if pre[n] <= m:
            return True

        # 后缀预处理：suf[i] = s[i..n-1] 逆向匹配时 t 的起始位置
        suf = [0] * (n + 1)
        suf[n] = m
        j = m - 1
        for i in range(n - 1, -1, -1):
            while j >= 0 and t[j] != s[i]:
                j -= 1
            if j >= 0:
                suf[i] = j
                j -= 1
            else:
                suf[i] = -1  # 无法匹配

        # 枚举替换点 i
        for i in range(n):
            if pre[i] < suf[i + 1]:
                return True
        return False
```

## 8. 边界情况与易错点

1. `s` **本身就是** `t` **的子序列**：`pre[n] <= m` 时直接返回 `true`，无需替换。必须先检查此情况。

2. `s` **长度为 1**：只要 `t` 非空，替换 `s[0]` 为 `t` 的任意字符即可。代码中 `pre[0]=0 < suf[1]=m`（当 `t` 非空时 `suf[1] >= 0`），自动处理。

3. `pre` **越界标记**：当前缀无法匹配时，`pre[i+1] = m+1`（大于 `m`），保证后续 `pre[i] < suf[i+1]` 不会误判为 `true`。

4. `suf` **越界标记**：当后缀无法匹配时，`suf[i] = -1`，保证 `pre[i] < suf[i+1]` 中 `suf` 为 -1 时条件不成立。

5. `pre[i]` **与** `suf[i+1]` **的下标关系**：枚举替换点 `i` 时，前缀用 `pre[i]`（`s[0..i-1]`），后缀用 `suf[i+1]`（`s[i+1..n-1]`）。不要写成 `pre[i]` 和 `suf[i]`——`suf[i]` 包含了 `s[i]` 本身。

6. `t` **比** `s` **短**：若 `m < n`，即使替换一个字符也无法让 `s` 成为 `t` 的子序列（子序列长度不能超过原串）。前后缀预处理会自然处理（`pre` 很快越界）。
