# K 个元素的最大总和

## 1. 题目概述

- **题目名称**：Q1. K 个元素的最大总和
- **链接**：[3974. K 个元素的最大总和](https://leetcode.cn/problems/maximum-total-sum-of-k-selected-elements/)
- **来源**：LeetCode 第 508 场周赛 Q1
- **难度**：中等
- **标签**：贪心、数组、排序

**题意简述**：

给定数组 `nums` 和整数 `k`、`mul`。从 `nums` 中选恰好 `k` 个元素，按任意顺序逐个处理。每个元素可选：**直接加到总和**，或**乘以 `mul` 当前值后加到总和**。每处理一个元素后 `mul` 减 1（可能变为 0 或负数）。返回最大总和。

**约束条件**：

- `1 <= nums.length <= 10^5`
- `1 <= nums[i] <= 10^5`
- `1 <= k <= nums.length`
- `1 <= mul <= 10^5`

## 2. 示例

**示例 1**

```text
输入：nums = [6,1,2,9], k = 3, mul = 2
输出：26
解释：选 9,6,2。9×2=18, 6×1=6, 2×1=2（加）。总和=18+6+2=26。
```

**示例 2**

```text
输入：nums = [3,7,5,2], k = 2, mul = 4
输出：43
解释：选 7,5。7×4=28, 5×3=15。总和=28+15=43。
```

**示例 3**

```text
输入：nums = [4,4], k = 1, mul = 1
输出：4
解释：选 4。4×1=4。总和=4。
```

## 3. 解题思路

### 3.1 暴力思路

枚选取哪 k 个元素、处理顺序、每个选乘或加。组合爆炸，不可行。

### 3.2 核心观察：贪心选最大 + 乘法优先

![贪心策略：选最大k个，大配大乘](images/ksum_greedy_concept.svg)

**观察 1**：选哪 k 个？`nums[i]` 均为正数，选最大的 `k` 个一定最优。

**观察 2**：选乘还是加？当 `mul >= 2` 时，`nums[i] × mul >= nums[i]`，乘法不劣于加法。当 `mul <= 1` 时，`nums[i] × mul <= nums[i]`，加法更优。

**观察 3**：乘法配对顺序。乘法的乘数 `mul, mul-1, ..., 1` 递减。由**排序不等式**，大数配大乘数最优——选中的 k 个元素降序排列，前 `min(k, mul-1)` 个用乘法，其余用加法。

> 💡 `mul-1` 是因为乘数从 `mul` 递减到 `2` 共 `mul-1` 个（乘数 1 和 0 及负数时加法更优）。乘法个数 `t = min(k, mul-1)`，所用乘数为 `mul, mul-1, ..., max(2, mul-t+1)`。

### 3.3 精确公式

设选中元素降序排列为 `a[0] >= a[1] >= ... >= a[k-1]`：

- 乘法个数 `t = min(k, mul - 1)`（乘数从 `mul` 递减，当乘数 `<= 1` 时用加法）

  > 推导：乘数从 `mul` 开始，第 `i` 个乘法用 `mul - i`。当 `mul - i >= 2` 即 `i <= mul - 2` 时乘法有益。因此 `t = min(k, mul - 1)`（i 从 0 到 t-1，乘数从 mul 到 mul-t+1 >= 2）。

- 前 `t` 个（最大）用乘法：贡献 `a[i] × (mul - i)`，`i = 0..t-1`
- 后 `k - t` 个用加法：贡献 `a[i]`，`i = t..k-1`

**答案**：

```text
ans = Σ_{i=0}^{t-1} a[i] × (mul - i)  +  Σ_{i=t}^{k-1} a[i]
```

### 3.4 示例演算

`nums = [6,1,2,9], k = 3, mul = 2`：

![示例演算](images/ksum_greedy_example.svg)

排序降序：`a = [9, 6, 2]`

- `t = min(3, 2-1) = 1`，即只有 1 个乘法
- `a[0] × 2 = 9 × 2 = 18`
- `a[1] + a[2] = 6 + 2 = 8`
- 总和 = `18 + 8 = 26` ✓

## 4. 算法细节

1. **排序**：`nums` 降序排序。
2. **选取**：取前 `k` 个。
3. **乘法个数**：`t = min(k, mul - 1)`。注意 `mul - 1` 可能为 0 或负（`mul=1` 时 `t=0`，全用加法）。
4. **累加**：前 `t` 个用乘法（乘数 `mul, mul-1, ..., mul-t+1`），后 `k-t` 个直接加。

---

## 5. 正确性证明

**引理 1**：选取最大的 `k` 个元素最优。

**证明**：`nums[i] > 0`。无论乘法还是加法，贡献均与元素值正相关（`a×c >= b×c` 当 `a>=b, c>0`；`a >= b`）。替换为更大元素不减少贡献。∎

**引理 2**：乘法只应用于乘数 `>= 2` 的元素。

**证明**：乘数 `c` 时，乘法贡献 `a×c`，加法贡献 `a`。`a×c >= a ⟺ c >= 1`。但 `c=1` 时两者相等，用加法即可。`c <= 0` 时 `a×c <= 0 < a`，加法严格更优。因此乘数 `>= 2` 才用乘法。∎

**引理 3**：大数配大乘数（排序不等式）。

**证明**：设有两个元素 `a >= b` 和两个乘数 `c >= d >= 2`。`a×c + b×d - (a×d + b×c) = (a-b)(c-d) >= 0`。因此大配大最优。∎

**定理**：算法返回最大总和。

**证明**：由引理 1 选最大 k 个，由引理 2 确定乘法个数 `t`，由引理 3 大配大排列。三步均为最优，故总和最大。∎

---

## 6. 复杂度分析

- **时间复杂度**：`O(n log n)`。排序 `O(n log n)`，累加 `O(k)`。
- **空间复杂度**：`O(1)` 额外空间（原地排序，或 `O(n)` 若不修改输入）。

> 💡 `n=10^5`，排序轻松通过。

## 7. 参考代码

### C++

```cpp
class Solution {
public:
    long long maxSum(vector<int>& nums, int k, int mul) {
        sort(nums.begin(), nums.end(), greater<int>());
        int t = min(k, mul - 1);  // 乘法个数（乘数 >= 2）
        if (t < 0) t = 0;
        long long ans = 0;
        for (int i = 0; i < k; i++) {
            if (i < t) {
                ans += (long long)nums[i] * (mul - i);
            } else {
                ans += nums[i];
            }
        }
        return ans;
    }
};
```

### Python

```python
class Solution:
    def maxSum(self, nums: list[int], k: int, mul: int) -> int:
        nums.sort(reverse=True)
        t = max(0, min(k, mul - 1))  # 乘法个数（乘数 >= 2）
        ans = 0
        for i in range(k):
            if i < t:
                ans += nums[i] * (mul - i)
            else:
                ans += nums[i]
        return ans
```

## 8. 边界情况与易错点

1. **`mul = 1`**：`t = 0`，全部用加法，答案 = 前 k 大之和。乘数 1 时乘法 = 加法，不亏不赚。
2. **`mul = 0` 或负数**：`mul >= 1` 由约束保证，但 `mul` 递减后可能为 0/负。乘数 `<= 0` 时乘法贡献 `<= 0`，加法严格更优，`t = min(k, mul-1)` 自动处理。
3. **`k = nums.length`**：全选，排序后全部处理。
4. **整数溢出**：`nums[i] × mul` 可达 `10^5 × 10^5 = 10^10`，`k` 个累加可达 `10^15`，C++ 需 `long long`。
5. **`t = min(k, mul-1)` 不是 `min(k, mul)`**：乘数从 `mul` 开始，第 `t` 个乘数为 `mul - t + 1`。要保证 `mul - t + 1 >= 2` 即 `t <= mul - 1`。
