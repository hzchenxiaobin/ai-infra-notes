# 区间内的兼容数字之和 I

## 1. 题目概述

- **题目名称**：Q1. 区间内的兼容数字之和 I
- **链接**：[3954. 区间内的兼容数字之和 I](https://leetcode.cn/problems/sum-of-compatible-numbers-in-range-i/)
- **来源**：LeetCode 第 505 场周赛 Q1
- **难度**：简单
- **标签**：位运算、枚举

**题意简述**：

给定正整数 `n` 和 `k`。一个正整数 `x` 称为**兼容**的，当且仅当：① `abs(n - x) <= k`；② `(n & x) == 0`（按位与为零）。返回所有兼容整数 `x` 的总和。

**约束条件**：

- `1 <= n <= 100`
- `1 <= k <= 100`

## 2. 示例

**示例 1**

```text
输入：n = 2, k = 3
输出：10
解释：兼容整数为 1, 4, 5
  x=1: abs(2-1)=1<=3, 2&1=0 ✓
  x=4: abs(2-4)=2<=3, 2&4=0 ✓
  x=5: abs(2-5)=3<=3, 2&5=0 ✓
  和 = 1+4+5 = 10
```

**示例 2**

```text
输入：n = 5, k = 1
输出：0
解释：区间 [4,6] 中没有兼容整数
```

## 3. 解题思路

### 3.1 暴力思路

直接枚举区间 `[max(1, n-k), n+k]` 内的每个正整数 `x`，检查 `n & x == 0`，累加符合条件的 `x`。

由于 `n, k <= 100`，区间最多 201 个数，枚举完全可行。

### 3.2 核心观察：`n & x == 0` 的含义

![兼容数字的按位与为零条件](images/compatible_numbers_bitwise.svg)

`n & x == 0` 意味着 `n` 和 `x` 在二进制表示中**没有任何一位同时为 1**。换言之，`x` 的二进制中，`n` 为 1 的那些位必须为 0。

> 💡 **关键观察**：`n & x == 0` 等价于 `x` 只在 `n` 的零位上取 1。对于小 `n`（<=100，最多 7 位），候选 `x` 的范围被 `n+k` 上界限制，直接枚举即可。

### 3.3 算法流程

1. 遍历 `x` 从 `max(1, n-k)` 到 `n+k`
2. 对每个 `x`，检查 `(n & x) == 0`
3. 累加满足条件的 `x`

---

## 4. 算法细节

- 下界取 `max(1, n-k)`：因为 `x` 是正整数，不能为 0
- 上界取 `n+k`：`abs(n-x) <= k` 等价于 `x` 在 `[n-k, n+k]` 范围内
- 按位与用 `&` 运算符，C++/Python 语法一致

---

## 5. 复杂度分析

- **时间复杂度**：O(k)。枚举区间长度最多 `2k+1 <= 201`，每次检查 O(1)。
- **空间复杂度**：O(1)。仅用常数空间。

## 6. 参考代码

### C++

```cpp
class Solution {
public:
    long long sumOfCompatibleNumbers(int n, int k) {
        long long ans = 0;
        for (int x = max(1, n - k); x <= n + k; ++x) {
            if ((n & x) == 0) {
                ans += x;
            }
        }
        return ans;
    }
};
```

### Python

```python
class Solution:
    def sumOfCompatibleNumbers(self, n: int, k: int) -> int:
        ans = 0
        for x in range(max(1, n - k), n + k + 1):
            if (n & x) == 0:
                ans += x
        return ans
```

---

## 7. 边界情况与易错点

1. **下界不能为 0**：`x` 是正整数，`max(1, n-k)` 防止 `n-k < 1` 时枚举到 0
2. **区间闭合**：`abs(n-x) <= k` 包含等号，上界是 `n+k`（含）
3. **和可能超出 int**：`n+k <= 200`，最多 201 个数，但用 `long long` 更安全
