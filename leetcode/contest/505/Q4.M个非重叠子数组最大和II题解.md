# M 个非重叠子数组最大和 II

## 1. 题目概述

- **题目名称**：Q4. M 个非重叠子数组最大和 II
- **链接**：[3957. M 个非重叠子数组最大和 II](https://leetcode.cn/problems/maximum-sum-of-m-non-overlapping-subarrays-ii/)
- **来源**：LeetCode 第 505 场周赛 Q4
- **难度**：困难
- **标签**：队列, 数组, 二分查找, 动态规划, 前缀和, 滑动窗口, 单调队列

**题意简述**：
与 Q3 完全相同：给定数组 `nums` 与整数 `m, l, r`，选择至少 1 个、至多 `m` 个互不重叠且长度在 `[l, r]` 范围内的子数组，使总和最大。

**约束条件**：
- `1 <= n <= 10^5`
- `-10^5 <= nums[i] <= 10^5`
- `1 <= m <= n`
- `1 <= l <= r <= n`

## 3. 解题思路

### 3.1 为什么 Q3 的 O(m·n) 会超时

Q3 中 `n <= 1000`，`O(m·n)` 可以轻松通过。但本题 `n <= 10^5`，`m <= n`，`O(m·n)` 最坏可达 `10^10`，必须优化。

### 3.2 核心观察：WQS 二分 / 拉格朗日松弛

设 `best[t]` 表示「恰好选 `t` 个满足条件子数组」的最大总和。题目要求 `max_{1 <= t <= m} best[t]`。

`best[t]` 关于 `t` 是**凹函数**（边际收益递减：先选总和大的子数组，再增加子数组带来的收益不会更大）。利用这一性质，可以引入一个**惩罚项** `lambda`：

```
f(lambda) = max_t (best[t] - lambda * t)
```

对于固定的 `lambda`，最优解会倾向于选择较少的子数组（因为每个子数组都要付出 `lambda` 的代价）。当 `lambda` 增大时，最优子数组数量单调减少。

如果我们能找到一个 `lambda*`，使得最优解在惩罚后仍选择**不少于 `m` 个子数组**，那么可以通过 `f(lambda*) + lambda* * m` 恢复出 `best[m]`，也就是「至多 `m` 个」的答案。

![WQS 二分思想](images/non_overlapping_subarrays_wqs_concept.svg)

### 3.3 带惩罚的 DP

对于固定 `lambda`，定义 `dp[i]` 为：在前 `i` 个元素中，所有**非空**合法选择的最大「价值和」，其中价值 = 子数组和之和 - lambda × 子数组个数。

转移：

```
dp[i] = max(
    dp[i-1],                                               // 不选以 i-1 结尾的子数组
    prefix[i] + max_{j∈[i-r,i-l]} eff(j) - lambda        // 选一个以 i-1 结尾的子数组
)
```

其中 `eff(j)` 表示「从位置 `j` 之后开始一个新的子数组」还是「继承之前的非空选择」：

```
如果 dp[j] >= 0：  eff(j) = dp[j] - prefix[j]，对应 count = cnt[j]
否则：            eff(j) = -prefix[j]，对应 count = 0   （从 j 开始新的子数组更优）
```

## 4. 算法细节

1. **辅助函数 `solve(lambda)`**：
   - 返回 `(value, count)`，`value` 为带惩罚的最优值，`count` 为该最优解中子数组的数量。
   - 内层转移使用单调队列维护 `eff(j)` 的最大值，均摊 `O(1)`。
   - 当两个决策值相等时，**优先选择子数组数量更多**的那个（这是 WQS 二分能正确恢复答案的关键）。
2. **二分搜索 `lambda`**：
   - 范围 `[0, 10^15]`（足够覆盖所有可能的边际收益）。
   - 找**最大的 `lambda`** 使得 `solve(lambda).count >= m`。
3. **答案恢复**：
   - 若 `solve(0).count < m`：说明即使不惩罚，最优解也用不到 `m` 个子数组，答案就是 `solve(0).value`。
   - 否则：答案 = `solve(lambda).value + lambda * m`。

## 5. 正确性证明

**引理 1**：对于固定 `lambda`，`solve(lambda)` 正确返回带惩罚的最大价值以及对应的子数组数量。

**证明**：递推中 `dp[i]` 考虑两种互斥情况：
- 不选以 `i-1` 结尾的子数组，价值为 `dp[i-1]`。
- 选一个以 `i-1` 结尾、长度在 `[l,r]` 的子数组，左端点为 `j+1`，贡献 `prefix[i] - prefix[j]`。此前部分可以是空选择（价值 0，数量 0）或非空选择（价值 `dp[j]`，数量 `cnt[j]`）。取两者中价值更大者，即 `eff(j)`。

两种情况取最大值，故 `dp[i]` 正确。数量 `cnt[i]` 随最大值同步更新；价值相等时取数量更大者，仍是一种合法的次优/最优方案。∎

**引理 2**：`solve(lambda).count` 关于 `lambda` 单调不增。

**证明**：增大 `lambda` 只会使「增加一个子数组」的代价变大，因此在所有决策中，选择更多子数组的方案相对更差。最优解不会选择比之前更多的子数组。∎

**引理 3**：设 `best[t]` 为恰好 `t` 个子数组的最大原始和，则 `best[t]` 关于 `t` 是凹函数（离散凹）。

**证明**：增加第 `t+1` 个子数组时，它必须与已有的 `t` 个子数组不重叠，因此只能从剩余区间中选择。由于我们已经选了最优的 `t` 个子数组，第 `t+1` 个的边际收益 `best[t+1] - best[t]` 不会超过第 `t` 个的边际收益 `best[t] - best[t-1]`。故二阶差分非正，`best[t]` 凹。∎

**引理 4**：设 `lambda*` 为满足 `solve(lambda*).count >= m` 的最大 `lambda`，则 `solve(lambda*).value + lambda* * m = best[m]`。

**证明**：设 `solve(lambda*)` 选择的子数组数量为 `c >= m`，则 `solve(lambda*).value = best[c] - lambda* * c`。

由 `best[t]` 的凹性，当 `lambda*` 是使最优数量仍 `>= m` 的最大惩罚时，`best[t]` 在 `t = m` 处的值恰好等于连接 `(c, best[c])` 的支撑线在 `t = m` 处的取值，即：

```
best[m] = best[c] + lambda* * (m - c) = solve(lambda*).value + lambda* * m
```

（这里用到了 `lambda*` 恰好对应凹包络在 `c` 到 `m` 区间上的斜率。）∎

**定理**：算法返回的值等于「至多选择 `m` 个满足条件子数组」的最大总和。

**证明**：
- 若 `solve(0).count < m`，则即使不惩罚，最优解也使用少于 `m` 个子数组。由于惩罚只会减少子数组数量，答案就是 `solve(0).value`，算法正确。
- 若 `solve(0).count >= m`，由引理 2 可二分找到最大 `lambda*` 使得 `count >= m`。由引理 4，`solve(lambda*).value + lambda* * m = best[m]`。又因为 `best[t]` 凹，至多 `m` 个的最优值就是 `best[m]`，算法正确。

综上，算法正确。∎

## 6. 复杂度分析

- **时间复杂度**：`O(n log W)`，其中 `W <= 10^15` 为二分范围。
  - 每次 `solve(lambda)` 使用单调队列，复杂度 `O(n)`。
  - 二分约 50 次迭代。
  - 总计算量约为 `50 * 10^5 = 5 * 10^6`。
- **空间复杂度**：`O(n)`。
  - 前缀和、DP 数组、单调队列。

## 7. 参考代码

### C++

```cpp
class Solution {
public:
    long long maxSum(vector<int>& nums, int m, int l, int r) {
        int n = nums.size();
        vector<long long> prefix(n + 1, 0);
        for (int i = 0; i < n; ++i) prefix[i + 1] = prefix[i] + nums[i];

        const long long INF = (1LL << 60);

        auto solve = [&](long long lam) -> pair<long long, int> {
            vector<long long> dp(n + 1, -INF);
            vector<int> cnt(n + 1, 0);
            dp[0] = -INF; // 前 0 个元素无法形成非空选择
            cnt[0] = 0;
            deque<tuple<int, long long, int>> dq; // (j, eff, cnt_eff)

            for (int i = 1; i <= n; ++i) {
                int j = i - l;
                if (j >= 0) {
                    long long eff;
                    int c;
                    if (dp[j] >= 0) {
                        eff = dp[j] - prefix[j];
                        c = cnt[j];
                    } else {
                        eff = -prefix[j];
                        c = 0;
                    }
                    while (!dq.empty()) {
                        auto [&bj, bval, bc] = dq.back();
                        if (bval < eff || (bval == eff && bc < c)) dq.pop_back();
                        else break;
                    }
                    dq.emplace_back(j, eff, c);
                }
                while (!dq.empty() && get<0>(dq.front()) < i - r) dq.pop_front();

                long long bestVal = dp[i - 1];
                int bestCnt = cnt[i - 1];
                if (!dq.empty()) {
                    auto [fj, fval, fc] = dq.front();
                    long long v = prefix[i] + fval - lam;
                    int c = fc + 1;
                    if (v > bestVal || (v == bestVal && c > bestCnt)) {
                        bestVal = v;
                        bestCnt = c;
                    }
                }
                dp[i] = bestVal;
                cnt[i] = bestCnt;
            }
            return {dp[n], cnt[n]};
        };

        auto [v0, c0] = solve(0);
        if (c0 < m) return v0;

        long long lo = 0, hi = (long long)1e15;
        while (lo < hi) {
            long long mid = (lo + hi + 1) / 2;
            auto [v, c] = solve(mid);
            if (c >= m) lo = mid;
            else hi = mid - 1;
        }
        auto [v, c] = solve(lo);
        return v + lo * m;
    }
};
```

### Python

```python
class Solution:
    def maxSum(self, nums: List[int], m: int, l: int, r: int) -> int:
        n = len(nums)
        prefix = [0] * (n + 1)
        for i, x in enumerate(nums):
            prefix[i + 1] = prefix[i] + x

        INF = float('-inf')

        def solve(lam: int):
            dp = [INF] * (n + 1)
            cnt = [0] * (n + 1)
            dp[0] = INF  # 前 0 个元素无法形成非空选择
            from collections import deque
            dq = deque()  # (j, eff, cnt_eff)

            for i in range(1, n + 1):
                j = i - l
                if j >= 0:
                    if dp[j] >= 0:
                        eff = dp[j] - prefix[j]
                        c = cnt[j]
                    else:
                        eff = -prefix[j]
                        c = 0
                    while dq and (dq[-1][1] < eff or (dq[-1][1] == eff and dq[-1][2] < c)):
                        dq.pop()
                    dq.append((j, eff, c))

                while dq and dq[0][0] < i - r:
                    dq.popleft()

                best_val = dp[i - 1]
                best_cnt = cnt[i - 1]
                if dq:
                    v = prefix[i] + dq[0][1] - lam
                    c = dq[0][2] + 1
                    if v > best_val or (v == best_val and c > best_cnt):
                        best_val = v
                        best_cnt = c

                dp[i] = best_val
                cnt[i] = best_cnt

            return dp[n], cnt[n]

        v0, c0 = solve(0)
        if c0 < m:
            return int(v0)

        lo, hi = 0, 10**15
        while lo < hi:
            mid = (lo + hi + 1) // 2
            _, c = solve(mid)
            if c >= m:
                lo = mid
            else:
                hi = mid - 1

        v, _ = solve(lo)
        return int(v + lo * m)
```

## 8. 边界情况与易错点

1. **`dp[0] = -inf` 而非 0**：Q4 的 DP 是**非空**选择，空选择的成本为 0、数量为 0。当所有子数组和为负时，必须强制至少选一个，因此用 `-inf` 并在 `eff(j)` 中通过 `-prefix[j]` 处理「从这里开始第一个子数组」的情况。
2. **tie-breaking 必须选数量更多**：WQS 二分正确性依赖于 `solve(lambda)` 在价值相等时返回更大的 `count`，否则二分边界处的答案恢复会出错。
3. **`solve(0).count < m` 时直接返回 `v0`**：此时所有数的最佳策略用不到 `m` 个子数组，答案就是该最优非空选择。
4. **二分上界**：`10^15` 足够大。`nums[i]` 最大 `10^5`，`n` 最大 `10^5`，任意子数组和不超过 `10^10`，更大的 `lambda` 只会让选择归零。
5. **整数溢出**：C++ 中 `lambda * m` 可能很大，需使用 `long long`。
