<!-- title: 字符串变换后的最少分组数 -->

## 1. 题目概述

- **题目名称**：Q4. 字符串变换后的最少分组数
- **链接**：[3895. 字符串变换后的最少分组数](https://leetcode.cn/problems/minimum-number-of-string-groups-through-transformations/)
- **来源**：LeetCode 第 511 场周赛 Q4
- **难度**：困难
- **标签**：字符串、最小循环表示、Booth 算法、哈希分组、等价类

**题意简述**：

定义字符串 `s` 的一次「变换」：取偶数下标字符子序列 `E` 与奇数下标字符子序列 `O`，将 `E`、`O` **各自独立**地向右循环移动任意位（可为 0），再依次放回原下标位置重构成新串。若两字符串能经一次变换互相得到，则称二者**等价**。给定 `words`，求最少分组数，使同组内任意两串等价。

**约束条件**：

- `1 <= words.length <= 10^5`
- `1 <= words[i].length <= 5 * 10^5`
- 所有 `words[i].length` 之和不超过 `5 * 10^5`
- `words[i]` 仅由小写英文字母组成

## 3. 解题思路

### 3.1 暴力思路：两两判定等价

对每对字符串 `(words[i], words[j])` 判定是否等价：枚举 `E` 的所有循环旋转（共 $|E|$ 种）看能否对上 `E(t)`，再判 `O`。单次判定 $O(|s|)$，两两判定 $O(N^2 \cdot L)$。$N$ 可达 $10^5$，不可行。

### 3.2 核心观察：变换 = E、O 各自独立做循环旋转

一次变换的本质是把 `s` 拆成两条**独立**的环：偶数位环 `E`、奇数位环 `O`，各自旋转后重组。偶数位永远只放回偶数位，两环互不干扰。因此：

> **等价判定**：$s \sim t \iff |s| = |t|$，且 $E(s)$ 与 $E(t)$ 互为循环旋转，且 $O(s)$ 与 $O(t)$ 互为循环旋转。

`E`、`O` 两条件完全独立，可分别处理。

![变换模型：s 拆成 E/O 两环独立旋转](images/string_groups_transform_model.svg)

### 3.3 规约到等价类计数

「循环旋转」是字符串上的等价关系（自反、对称、传递）；「变换等价」是两个旋转等价关系的笛卡尔积，因而也是等价关系。同组要求**任意两串等价**，即同组 = 同一等价类。故：

> **最少分组数 = 不同等价类的个数**

### 3.4 规范化：最小循环表示 + Booth 算法

为每个等价类取一个**规范代表**：取所有循环旋转中**字典序最小**者（即最小循环表示）。两串互为循环旋转 $\iff$ 它们的最小循环表示相同。

求最小循环表示的起始下标可用 **Booth 算法**：在 $s + s$ 上扫描，借助类似 KMP 的失配函数 $f$，利用均摊分析保证每字符 $O(1)$，总 $O(n)$。

于是每个 `words[i]` 的分组键为：

$$
\text{key}(w) = \big(\,\text{minrot}(E(w)),\ \text{minrot}(O(w))\,\big)
$$

答案 = 不同 `key` 的个数。

![规范化分组：最小循环表示作键，Booth O(n)](images/string_groups_canonical_grouping.svg)

### 3.5 示例演算（示例 2）

|------|---------|---------|-----------|-----------|-----|
| abc  | ac      | b       | ac        | b         | (ac, b) |
| cba  | ca      | b       | ac        | b         | (ac, b) |

## 4. 算法细节

### 4.1 抽取 E / O

`E = w[0::2]`（下标 $0,2,4,\ldots$），`O = w[1::2]`（下标 $1,3,5,\ldots$）。$|E| = \lceil n/2 \rceil$、$|O| = \lfloor n/2 \rfloor$，由 $|w|$ 决定，无需额外存长度。

### 4.2 Booth 算法求最小循环旋转起点

将串 doubling 为 $s_2 = s + s$，维护「当前最优起点 $k$」与失配函数 $f$。逐位扫描 $s_2$，当 $s_2[j] \ne s_2[k+i+1]$ 时比较大小决定是否更新 $k$，并沿 $f$ 回退。$f$ 的均摊性质保证每字符均摊 $O(1)$，总 $O(n)$。返回起点 $k \in [0,n)$ 后，最小循环表示为 $s[k{:}] + s[{:}k]$。

```text
Booth(s):              // 返回最小循环旋转的起点下标
  s2 = s + s;  n = |s|
  f[0 .. 2n-1] = -1
  k = 0
  for j = 1 .. 2n-1:
      i = f[j - k - 1]
      while i != -1 and s2[j] != s2[k + i + 1]:
          if s2[j] < s2[k + i + 1]:
              k = j - i - 1
          i = f[i]
      if s2[j] != s2[k + i + 1]:
          if s2[j] < s2[k + i + 1]:
              k = j
          f[j - k] = -1
      else:
          f[j - k] = i + 1
  return k
```

#### 伪代码逐行解释

**目标**：在 $O(n)$ 时间内求 $k \in [0,n)$，使 $s[k{:}]+s[{:}k]$ 是所有循环旋转中字典序最小者。核心技巧：倍长 $s$ 为 $s_2=s+s$（所有旋转都是 $s_2$ 中长度 $n$ 的子串），再用类似 KMP 的失配函数 $f$ 做均摊 $O(1)$ 比较。

| 行 | 伪代码 | 含义 |
|----|--------|------|
| 1 | `Booth(s):` | 函数入口，输入原串 $s$，返回最小循环旋转起点 $k$ |
| 2 | `s2 = s + s;  n = \|s\|` | **倍长**：$s_2$ 长度 $2n$，所有旋转都是 $s_2$ 中长度 $n$ 的子串，避免取模；$n$ 限定返回值范围 $[0,n)$ |
| 3 | `f[0 .. 2n-1] = -1` | **失配函数初始化**：$f[j{-}k]$ 记录 $s_2[k..]$ 与 $s_2[j..]$ 的**已匹配公共前缀长度减 1**（类似 KMP 的 `next`）。$-1$ = 尚未匹配任何字符 |
| 4 | `k = 0` | **初始化最优起点** $k{=}0$，先把不旋转（$s$ 本身）当作暂时最优 |
| 5 | `for j = 1 .. 2n-1:` | **主循环**：扫描指针 $j$ 从 1 到 $2n{-}1$，$j$ 是正在尝试的候选起点。扫到 $2n{-}1$ 保证 $k$ 收敛 |
| 6 | `i = f[j - k - 1]` | **取上一步匹配长度**：$i$ = $s_2[k..]$ 与 $s_2[j..]$ 已匹配了多少位（减 1）。下标用 $j{-}k{-}1$（相对偏移），让 $k$ 更新后 $f$ 语义仍正确 |
| 7 | `while i != -1 and s2[j] != s2[k + i + 1]:` | **失配回退循环**：已有部分匹配（$i{\ne}-1$）且当前位置失配时，沿 $f$ 回退。$k{+}i{+}1$ 是以 $k$ 为起点的串中当前对齐位置（已匹配 $i{+}1$ 位，下一位偏移 $i{+}1$） |
| 8 | `if s2[j] < s2[k + i + 1]:` | **发现更优起点**：失配时若 $s_2[j] < s_2[k{+}i{+}1]$，说明以 $k$ 起点的串在第 $i{+}1$ 位比以 $j$ 起点的大 → $k$ 不是最优 |
| 9 | `k = j - i - 1` | **更新 $k$**：因已匹配 $i{+}1$ 位，$j{-}i{-}1$ 是比 $k$ 更优且位置正确的候选（均摊分析证明不漏最优解）。若 $s_2[j] > s_2[k{+}i{+}1]$ 则 $k$ 不变 |
| 10 | `i = f[i]` | **沿 $f$ 回退**：$i$ 更新为 $f[i]$（更短的已匹配前缀），继续 while 尝试更短对齐。类似 KMP 失配后模式串「自我滑动」 |
| 11 | `if s2[j] != s2[k + i + 1]:` | **终态判断**：while 退出后判断是仍失配（$i{=}-1$ 回退到头）还是已匹配 |
| 12 | `if s2[j] < s2[k + i + 1]:` | **首位定胜负**：回退到 $i{=}-1$ 仍失配，比较 $s_2[j]$ vs $s_2[k]$（首位） |
| 13 | `k = j` | 若 $s_2[j] < s_2[k]$，以 $j$ 开头的串首位就更小，直接 $k \leftarrow j$ |
| 14 | `f[j - k] = -1` | **记录失配**：第 0 位就失配，公共前缀长度 0 → $f[j{-}k] = -1$ |
| 15 | `else:` | **匹配成功分支**：$s_2[j] = s_2[k{+}i{+}1]$ |
| 16 | `f[j - k] = i + 1` | **记录匹配**：公共前缀长度 $+1$，$f[j{-}k] = i{+}1$。后续 $j$ 增大时被 `i = f[j-k-1]` 取出，避免重复比较——**均摊 $O(1)$ 的关键** |
| 17 | `return k` | **返回最优起点** $k \in [0,n)$，最小循环表示为 $s[k{:}]+s[{:}k]$ |

**关键不变式**：

| 不变量 | 含义 |
|--------|------|
| $k < j$ | 最优起点始终在扫描指针之前 |
| $0 \le k < n$ | 返回值限定在原串长度内（$s_2$ 后半段是重复，$k$ 不会超过 $n$） |
| $f[j{-}k]$ = 公共前缀长度 $-1$ | $s_2[k..]$ 与 $s_2[j..]$ 的 LCP 长度减 1 |

> 💡 **为什么是 $O(n)$？** 每步 $i$ 最多 $+1$（匹配时 `f[j-k]=i+1`），整个循环 $i$ 最多增加 $2n$ 次；而每次 while 回退 $i$ 至少 $-1$，回退总次数 $\le$ 增加总次数 $\le 2n$。故总比较次数 $O(n)$，**均摊 $O(1)$/字符**——这正是把朴素 $O(n^2)$ 压到 $O(n)$ 的核心。

### 4.3 Booth 算法图解

#### 核心思想

将 $s$ 倍长为 $s_2 = s + s$，则 $s$ 的所有循环旋转都是 $s_2$ 中长度为 $n$ 的子串（起点 $0, 1, \ldots, n{-}1$）。问题转化为：**在 $s_2$ 中找字典序最小的长度 $n$ 子串的起点**。

维护两个指针：

- $k$：当前已知的最优起点（使旋转字典序最小）
- $j$：扫描指针，逐位尝试找比 $k$ 更优的起点

比较 $s_2[k..]$ 与 $s_2[j..]$ 的字典序：失配时若 $s_2[j] < s_2[k{+}i{+}1]$，则 $j$ 是更优起点，$k \leftarrow j$（或 $j{-}i{-}1$）；否则 $k$ 不变，$j$ 继续前进。

![Booth 算法核心思想](images/booth_algorithm_concept.svg)

#### 失配函数 $f$ 的作用

$f[j{-}k]$ 记录 $s_2[k..]$ 与 $s_2[j..]$ 的**已匹配公共前缀长度**（减 1），类似 KMP 的 `next` 数组。失配时沿 $f$ 回退，避免从头比较。每步 $i$ 最多 $+1$（匹配时），回退总次数 $\le$ 累积增加 $\le 2n$，故总比较次数 $O(n)$，**均摊 $O(1)$/字符**——这正是把朴素 $O(n^2)$ 压到 $O(n)$ 的关键。

#### 逐步演算（$s = $ `"baab"`，$n=4$，$s_2 = $ `"baabbaab"`）

$s_2$ 的 4 个候选旋转起点：$k{=}0$→`"baab"`、$k{=}1$→`"aabb"`、$k{=}2$→`"abba"`、$k{=}3$→`"bbaa"`，最小者为 $k{=}1$ 的 `"aabb"`。

| $j$ | $s_2[j]$ | $i$ 入 | 比较 $s_2[j]$ vs $s_2[k{+}i{+}1]$ | 决策 | $k$ | $f[j{-}k]$ | 说明 |
|-----|----------|--------|-----------------------------------|------|-----|-----------|------|
| 1 | `a` | $-1$ | `a` vs `b` ($s_2[0]$) | `a<b` → $k{=}j$ | $0{\to}1$ | $-1$ | 发现更优起点（首位 `a<b` 已定胜负） |
| 2 | `a` | $-1$ | `a` vs `a` ($s_2[1]$) | 匹配 | $1$ | $0$ | 记录匹配长度 |
| 3 | `b` | $0$ | `b` vs `a` ($s_2[2]$) | 失配 `b>a` | $1$ | $-1$ | $i$ 沿 $f$ 回退到 $-1$；$k{=}1$ 仍最优 |
| 4 | `b` | $-1$ | `b` vs `a` ($s_2[1]$) | 失配 `b>a` | $1$ | $-1$ | 起点4以 `b` 开头，不可能小于起点1 |
| 5 | `a` | $-1$ | `a` vs `a` ($s_2[1]$) | 匹配 | $1$ | $0$ | 开始新一轮匹配 |
| 6 | `a` | $0$ | `a` vs `a` ($s_2[2]$) | 匹配 | $1$ | $1$ | 继承上步，已匹配 `"aa"` |
| 7 | `b` | $1$ | `b` vs `b` ($s_2[3]$) | 匹配 | $1$ | $2$ | 已匹配 `"aab"`，确认 $k{=}1$ 最优 |

最终 $k = 1$，$\text{minrot} = s[1{:}] + s[{:}1] = $ `"aabb"`。

![Booth 算法 baab 演算](images/booth_algorithm_trace.svg)

**三个关键步骤**：

1. **$j{=}1$：$k$ 更新**。$s_2[1]{=}$`a` $< s_2[0]{=}$`b`，起点 1 的旋转首位已比起点 0 小，$k$ 从 0 更新为 1。
2. **$j{=}3$：失配但 $k$ 不变**。$s_2[3]{=}$`b` $> s_2[2]{=}$`a`，起点 3 的旋转以 `b` 开头，不可能小于起点 1 的 `a` 开头，$k$ 保持 1。
3. **$j{=}5,6,7$：匹配链确认**。$s_2$ 后半段（第二个 $s$）与 $k{=}1$ 开头的旋转逐位匹配，$f$ 累积记录匹配长度 $0 \to 1 \to 2$，验证 $k{=}1$ 的旋转 `"aabb"` 完整通过。

### 4.4 分组键与计数

用哈希集合维护 `(minrot(E), minrot(O))` 的去重，最终大小即答案。键直接用规范串本身——总长受 $5 \times 10^5$ 约束，内存安全。

## 5. 正确性证明

**引理 1**（等价判定）：$s \sim t \iff |s| = |t|$ 且 $E(s), E(t)$ 互为循环旋转且 $O(s), O(t)$ 互为循环旋转。

**证明**：
- ($\Longrightarrow$) 若 $s \sim t$，则 $t$ 由 $s$ 经 $E$、$O$ 各自一次循环旋转后重组得到，故 $|s|=|t|$，且 $E(t)$ 是 $E(s)$ 的循环旋转、$O(t)$ 是 $O(s)$ 的循环旋转。循环旋转作为关系是对称的，故 $E(s), E(t)$ 互为循环旋转，$O$ 同理。 ∎
- ($\Longleftarrow$) $|s|=|t|$ 蕴含 $|E(s)|=|E(t)|$、$|O(s)|=|O(t)|$。若 $E(s), E(t)$ 互为循环旋转，则存在移位 $a$ 使 $E(s)$ 右移 $a$ 位等于 $E(t)$；同理存在 $b$ 使 $O(s)$ 右移 $b$ 位等于 $O(t)$。对 $s$ 施加该变换即得 $t$，故 $s \sim t$。 ∎

**引理 2**（最小表示刻画）：两串互为循环旋转 $\iff$ 二者的最小循环表示相同。

**证明**：循环旋转是等价关系，所有互为旋转的串构成一个等价类；该类中字典序最小者唯一（字典序为全序），作为类的规范代表。同类 $\iff$ 同代表。 ∎

**定理**：最少分组数 $= \left|\left\{\,(\text{minrot}(E(w)),\ \text{minrot}(O(w))) : w \in \text{words}\,\right\}\right|$。

**证明**：
1. **可行性**（$\le$）：由引理 1、2，键相同的两串满足 $E$、$O$ 均互为循环旋转，故 $s \sim t$。把键相同的串放入同一组，组内任意两串等价，得到一个合法分组，组数 = 不同键数。
2. **最优性**（$\ge$）：键不同的两串，$E$ 或 $O$ 不互为循环旋转，由引理 1 二者不等价，不能同组。故任何合法分组中，不同键的串必须落在不同组，组数 $\ge$ 不同键数。
3. 综合得最少分组数 = 不同键数。 ∎

## 6. 复杂度分析

- **时间复杂度**：$O\!\left(\sum |w|\right)$。每个串抽取 $E$、$O$ 为线性；Booth 算法对 $E$、$O$ 各线性；哈希插入线性。总长 $\sum |w| \le 5 \times 10^5$，整体线性。
- **空间复杂度**：$O\!\left(\sum |w|\right)$。最坏情况所有键互异，需存储全部规范串。

## 7. 参考代码

### C++

```cpp
class Solution {
  public:
    int minGroups(vector<string>& words) {
        auto leastRotation = [](const string& s) -> int {
            int n = s.size();
            if (n == 0) return 0;
            string s2 = s + s;
            vector<int> f(2 * n, -1);
            int k = 0;
            for (int j = 1; j < 2 * n; j++) {
                char sj = s2[j];
                int i = f[j - k - 1];
                while (i != -1 && sj != s2[k + i + 1]) {
                    if (sj < s2[k + i + 1]) k = j - i - 1;
                    i = f[i];
                }
                if (sj != s2[k + i + 1]) {
                    if (sj < s2[k + i + 1]) k = j;
                    f[j - k] = -1;
                } else {
                    f[j - k] = i + 1;
                }
            }
            return k;
        };
        auto canonical = [&](const string& s) -> string {
            if (s.empty()) return s;
            int k = leastRotation(s);
            return s.substr(k) + s.substr(0, k);
        };
        set<pair<string, string>> seen;
        for (const auto& w : words) {
            string E, O;
            E.reserve((w.size() + 1) / 2);
            O.reserve(w.size() / 2);
            for (int i = 0; i < (int)w.size(); i++) {
                if (i % 2 == 0) E.push_back(w[i]);
                else O.push_back(w[i]);
            }
            seen.insert({canonical(E), canonical(O)});
        }
        return (int)seen.size();
    }
};
```

```python
    def minGroups(self, words: List[str]) -> int:
        def least_rotation_start(s: str) -> int:
            n = len(s)
            if n == 0:
                return 0

            f = [-1] * (2 * n)
            k = 0
            for j in range(1, 2 * n):
                sj = s2[j]

                    if sj < s2[k + i + 1]:
                        k = j
                    f[j - k] = -1
                else:
                    f[j - k] = i + 1
            return k

        def canonical(s: str) -> str:
            if not s:
                return s
            k = least_rotation_start(s)
            return s[k:] + s[:k]

        seen = set()
        for w in words:
            seen.add((canonical(w[0::2]), canonical(w[1::2])))
        return len(seen)
```

## 8. 边界情况与易错点

1. **单字符 / 空子序列**：$|w|=1$ 时 $E=w$、$O=\varnothing$。Booth 对 $n=0$ 直接返回 0，`canonical("")=""`，无需特判。
2. **E、O 长度不必相等**：$|E|=\lceil n/2 \rceil$、$|O|=\lfloor n/2 \rfloor$。$n$ 为奇数时 $|E|=|O|+1$。等价判定只要求**同侧**长度对应（由 $|s|=|t|$ 保证），不要求 $|E|=|O|$。
3. **E、O 完全独立**：易误以为旋转会"跨奇偶位混合"，实际上偶数位永远只放回偶数位，两环互不干扰，故 $E$、$O$ 分别处理。
4. **最小表示的唯一性**：字典序最小旋转**唯一**（即便串有周期），作键安全。勿用"任意一个旋转"作键，否则同类串可能取到不同代表。
5. **Booth 的 doubling 与下标**：算法在 $s+s$ 上扫描 $2n$ 位，但返回的 $k \in [0, n)$，最终表示取 `s[k:]+s[:k]`，勿越界。
6. **复杂度边界**：单个串可达 $5 \times 10^5$，朴素 $O(n^2)$ 求最小旋转会超时（$2.5 \times 10^{11}$），必须用 Booth 的 $O(n)$。
7. **内存**：总长 $5 \times 10^5$，存储全部规范串键约同量级，安全；切勿对每对串两两比较。

**延伸思考**：

- 若变换改为「$E$、$O$ 合并后整体循环旋转」会怎样？则失去独立性，退化为整串循环旋转等价，键只需 $\text{minrot}(w)$。
