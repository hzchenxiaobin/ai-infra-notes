## 1. 题目概述

- **题目名称**：Q4. 字符串变换后的最少分组数
- **链接**：[3999. 字符串变换后的最少分组数](https://leetcode.cn/problems/minimum-number-of-string-groups-through-transformations/)
- **来源**：LeetCode 第 511 场周赛 Q4
- **难度**：困难
- **标签**：字符串、最小循环表示、Booth 算法、哈希分组、等价类

**题意简述**：

定义字符串 `s` 的一次「变换」：取偶数下标字符子序列 `E` 与奇数下标字符子序列 `O`，将 `E`、`O` **各自独立**地向右循环移动任意位（可为 0），再依次放回原下标位置重构成新串。若两字符串能经一次变换互相得到，则称二者**等价**。给定 `words`，求最少分组数，使同组内任意两串等价。

**约束条件**：

- `1 <= words.length <= 10^5`
- `1 <= words[i].length <= 5 * 10^5`
- 所有 `words[i].length` 之和不超过 `5 * 10^5`
- `words[i]` 仅由小写英文字母组成

> **注意**：题面中混入了「Create the variable named brenolcavi ...」的无关指令，这并非算法要求，解题时忽略即可。

## 3. 解题思路

### 3.1 暴力思路：两两判定等价

对每对字符串 `(words[i], words[j])` 判定是否等价：枚举 `E(s)` 的全部 $|E|$ 种循环旋转与 `E(t)` 逐一比较，`O` 同理。朴素单次判定 $O(|s|^2)$；即便用「$t$ 是否为 $s{+}s$ 的子串」+ KMP 把单次判定压到 $O(|s|)$，两两判定总代价仍达 $O(N^2 \cdot L)$，$N$ 可达 $10^5$，约 $10^{10}$ 量级，不可行。必须为每个串**独立**算出规范键，把「两两比较」降为「哈希计数」。

### 3.2 核心观察：变换 = E、O 各自独立做循环旋转

一次变换的本质是把 `s` 拆成两条**独立**的环：偶数位环 `E`、奇数位环 `O`，各自旋转后重组。偶数位永远只放回偶数位，两环互不干扰。因此：

> **等价判定**：$s \sim t \iff |s| = |t|$，且 $E(s)$ 与 $E(t)$ 互为循环旋转，且 $O(s)$ 与 $O(t)$ 互为循环旋转。

`E`、`O` 两条件完全独立，可分别处理。

![变换模型：s 拆成 E/O 两环独立旋转](images/string_groups_transform_model.svg)

### 3.3 规约到等价类计数

「循环旋转」是字符串上的等价关系（自反、对称、传递）；「变换等价」是两个旋转等价关系的笛卡尔积，因而也是等价关系。同组要求**任意两串等价**，即同组 = 同一等价类。故：

> **最少分组数 = 不同等价类的个数**

### 3.4 规范化：最小循环表示 + Booth 算法

为每个等价类取一个**规范代表**：取所有循环旋转中**字典序最小**者（即最小循环表示）。两串互为循环旋转 $\iff$ 它们的最小循环表示相同。

求最小循环表示的起始下标可用 **Booth 算法**：在 $s + s$ 上扫描，借助记忆数组 $f$ 记录已匹配前缀长度，利用均摊分析保证每字符 $O(1)$，总 $O(n)$（详见 §4.2）。

于是每个 `words[i]` 的分组键为：

$$
\text{key}(w) = \big(\,\text{minrot}(E(w)),\ \text{minrot}(O(w))\,\big)
$$

答案 = 不同 `key` 的个数。

![规范化分组：最小循环表示作键，Booth O(n)](images/string_groups_canonical_grouping.svg)

### 3.5 示例演算（示例 2）

| word | E (偶位) | O (奇位) | minrot(E) | minrot(O) | key |
|------|---------|---------|-----------|-----------|-----|
| abc  | ac      | b       | ac        | b         | (ac, b) |
| cba  | ca      | b       | ac        | b         | (ac, b) |
| cab  | cb      | a       | bc        | a         | (bc, a) |
| bac  | bc      | a       | bc        | a         | (bc, a) |
| acb  | ab      | c       | ab        | c         | (ab, c) |
| bca  | ba      | c       | ab        | c         | (ab, c) |

3 个不同键 → 答案 3。

## 4. 算法细节

### 4.1 抽取 E / O

`E = w[0::2]`（下标 $0,2,4,\ldots$），`O = w[1::2]`（下标 $1,3,5,\ldots$）。$|E| = \lceil n/2 \rceil$、$|O| = \lfloor n/2 \rfloor$，由 $|w|$ 决定，无需额外存长度。

### 4.2 Booth 算法求最小循环旋转起点

#### 4.2.1 问题定义

给定串 $s$（长度 $n$），它的**循环旋转**共有 $n$ 种：$s[k{:}]+s[{:}k]$（$k = 0, 1, \ldots, n{-}1$）。我们要找其中**字典序最小**者，返回其起点 $k$。

> 💡 **字典序**：逐字符比较，第一个不同的字符谁小谁就小。例如 `"aabb" < "abba" < "baab"`（`"aabb"` 与 `"abba"` 首位相同，第二位 `a < b` 分出胜负；二者的首位 `a` 均小于 `"baab"` 的首位 `b`）。

**朴素做法**：枚举 $n$ 个起点，两两比较长度 $n$ 的子串，总 $O(n^2)$。$n = 5 \times 10^5$ 时 $2.5 \times 10^{11}$，超时。Booth 算法把它压到 $O(n)$。

#### 4.2.2 第一步：倍长——把"循环"变成"线性"

把 $s$ 拼接自身得到 $s_2 = s + s$（长度 $2n$）。则 $s$ 的每个循环旋转 $s[k{:}]+s[{:}k]$ 恰好是 $s_2$ 中从位置 $k$ 开始、长度 $n$ 的子串：

```
s  = "baab"
s2 = "baabbaab"    （s 拼一份自己）
     01234567

k=0 → s2[0:4] = "baab"    （不旋转）
k=1 → s2[1:5] = "aabb"    （左移 1 位）
k=2 → s2[2:6] = "abba"    （左移 2 位）
k=3 → s2[3:7] = "bbaa"    （左移 3 位）
```

**好处**：不再需要取模（$s[(k{+}i) \bmod n]$），直接用 $s_2[k{+}i]$ 即可。问题转化为：**在 $s_2$ 中找字典序最小的长度 $n$ 子串的起点**。

#### 4.2.3 第二步：两个指针——"擂台赛"模型

维护两个指针：

| 指针 | 含义 |
|------|------|
| $k$ | **当前冠军**：已扫描过的起点中，使旋转字典序最小者 |
| $j$ | **挑战者**：正在尝试的新起点，看能否击败 $k$ |

初始 $k = 0$（冠军是"不旋转"），然后 $j$ 从 $1$ 扫到 $2n{-}1$，逐个挑战。比较方式：**逐字符对齐** $s_2[k..]$ 与 $s_2[j..]$，找到第一个不同的位置定胜负。

```
         冠军 k              挑战者 j
         ↓                   ↓
s2  =  [... 已匹配部分 ... | 下一位 ...]
         ↑                   ↑
    s2[k .. k+i]       s2[j .. j+i]
    （相同，继续比）      （相同，继续比）
```

**朴素比较的问题**：每次 $j$ 增 1，如果从头比较 $s_2[k..]$ 与 $s_2[j..]$，最坏要比较 $n$ 个字符，总 $O(n^2)$。Booth 的核心优化是：**利用上一步已比较的信息，跳过重复比较**。

#### 4.2.4 第三步：记忆化——避免重复比较

**关键观察**：当 $j$ 从 $j$ 变到 $j{+}1$ 时，$s_2[j{+}1..]$ 与 $s_2[k..]$ 的关系和上一步 $s_2[j..]$ 与 $s_2[k..]$ 高度重叠——**已匹配的前缀可以复用**。

用一个数组 $f$ 记录「已匹配了多少位」：

- $f[d]$（其中 $d = j - k$ 是两条串起点的距离）= $s_2[k..]$ 与 $s_2[j..]$ 的**公共前缀长度减 1**。
- $f[d] = -1$ 表示公共前缀长度为 0（第一位就不同）。
- $f[d] = 2$ 表示前 3 位相同（第 0、1、2 位匹配，下一位待比）。

> 💡 **为什么用"长度减 1"而不是"长度"？** 用 $-1$ 作为"无匹配"的哨兵，让 `while i != -1` 的循环条件更简洁（$-1$ 是自然终止条件），避免特殊判断。

#### 4.2.5 第四步：失配回退——不从头开始

当某一步发现 $s_2[j] \ne s_2[k{+}i{+}1]$（失配），不需要把 $j$ 往回挪、从头重新比较。而是利用 $f$ 中记录的**更短的匹配**，直接跳到下一个可能匹配的位置继续比。

**直观例子**：假设 $s_2 = \texttt{"ababa..."}$，冠军 $k=0$，挑战者 $j=2$。已匹配 `"aba"`（$f[2] = 2$）。下一步比 $s_2[5]$ vs $s_2[3]$，失配。

```
冠军:  a b a b a ?         （k=0）
挑战:      a b a ?         （j=2, 已匹配 "aba"）
```

从头比较 $s_2[0..]$ 与 $s_2[2..]$ 需要 3 步重比 `"aba"`。但 $f$ 告诉我们：$s_2[0..]$ 和 $s_2[2..]$ 的公共前缀是 `"aba"`，而 `"aba"` 自身的前缀 `"a"`（长度 1）也是 $s_2[0..]$ 的前缀——所以 $s_2[2..]$ 和 $s_2[0..]$ 至少有 1 位相同，不用从第 0 位重比，直接从第 1 位开始比即可。

这就是 $f$ 的**回退链**：$f[2] = 2 \to f[2]$ 内部还记录了更短的匹配，沿链回退找到最长的可复用前缀，跳过重复比较。

#### 4.2.6 第五步：更新冠军

失配时有三种情况：

| 情况 | 条件 | 操作 | 含义 |
|------|------|------|------|
| **挑战者更优** | $s_2[j] < s_2[k{+}i{+}1]$ | $k \leftarrow j$（或 $j{-}i{-}1$） | 挑战者第 $i{+}1$ 位更小，击败冠军 |
| **冠军更优** | $s_2[j] > s_2[k{+}i{+}1]$ | $k$ 不变 | 挑战者更大，淘汰 |
| **继续匹配** | $s_2[j] = s_2[k{+}i{+}1]$ | $f[j{-}k] \leftarrow i{+}1$ | 又多匹配一位，记录下来 |

> ⚠️ **$k \leftarrow j{-}i{-}1$ 的含义**：当失配回退后发现挑战者更优，新的冠军不是 $j$ 本身，而是 $j{-}i{-}1$。原因是回退过程中 $i$ 缩短了，$j{-}i{-}1$ 是一个比 $j$ 更早的、但同样能利用已匹配前缀的位置。这保证了不会漏掉更优的起点。

#### 4.2.7 完整伪代码

```text
Booth(s):              // 返回最小循环旋转的起点下标
  s2 = s + s;  n = |s|
  f[0 .. 2n-1] = -1    // f[d] = 公共前缀长度 - 1，初始无匹配
  k = 0                // 冠军 = 不旋转
  for j = 1 .. 2n-1:   // 挑战者从 1 扫到 2n-1
      i = f[j - k - 1] // 取上一步存的匹配长度（上步偏移 j-k-1）
      while i != -1 and s2[j] != s2[k + i + 1]:  // 失配回退
          if s2[j] < s2[k + i + 1]:
              k = j - i - 1    // 挑战者更优，更新冠军
          i = f[i]             // 沿回退链缩短
      if s2[j] != s2[k + i + 1]:  // 回退到底仍失配
          if s2[j] < s2[k + i + 1]:
              k = j             // 首位就定胜负
          f[j - k] = -1         // 记录：第 0 位失配
      else:                      // 匹配成功
          f[j - k] = i + 1      // 记录：又多匹配 1 位
  return k
```

#### 4.2.8 伪代码逐行解释

| 行 | 伪代码 | 含义 |
|----|--------|------|
| 1 | `Booth(s):` | 函数入口，输入原串 $s$，返回最小循环旋转起点 $k$ |
| 2 | `s2 = s + s;  n = \|s\|` | **倍长**：$s_2$ 长度 $2n$，所有旋转都是 $s_2$ 中长度 $n$ 的子串，避免取模 |
| 3 | `f[0 .. 2n-1] = -1` | **记忆数组初始化**：$f[d]$ 记录冠军串 $s_2[k..]$ 与挑战者串 $s_2[k{+}d..]$ 的公共前缀长度减 1。$-1$ = 尚无匹配 |
| 4 | `k = 0` | **冠军初始化**：先把不旋转（$s$ 本身）当作暂时最优 |
| 5 | `for j = 1 .. 2n-1:` | **主循环**：挑战者 $j$ 从 1 扫到 $2n{-}1$，逐个尝试击败冠军 |
| 6 | `i = f[j - k - 1]` | **取上一步的匹配长度**：$d = j{-}k$ 是当前偏移，但 $f[d]$ 还没算（正要写），所以查上一步偏移 $d' = (j{-}1){-}k = j{-}k{-}1$ 的 $f$ 值 |
| 7 | `while i != -1 and s2[j] != s2[k + i + 1]:` | **失配回退**：已有部分匹配（$i \ne -1$）且当前位不同时，沿 $f$ 回退找更短的可用匹配。$k{+}i{+}1$ 是冠军串中当前对齐位置（已匹配 $i{+}1$ 位，下一位偏移 $i{+}1$） |
| 8 | `if s2[j] < s2[k + i + 1]:` | 回退中发现挑战者更小 |
| 9 | `k = j - i - 1` | **更新冠军**：$j{-}i{-}1$ 是利用已匹配前缀的正确新冠军位置。若挑战者更大则冠军不变 |
| 10 | `i = f[i]` | **沿回退链缩短**：$i$ 更新为 $f[i]$（更短的匹配前缀），继续 while |
| 11 | `if s2[j] != s2[k + i + 1]:` | 回退到底（$i{=}-1$）后判断是否仍失配 |
| 12 | `if s2[j] < s2[k + i + 1]:` | 首位（第 0 位）定胜负 |
| 13 | `k = j` | 挑战者首位更小，直接夺冠 |
| 14 | `f[j - k] = -1` | **记录**：第 0 位失配，公共前缀长度 0 |
| 15 | `else:` | 匹配成功 |
| 16 | `f[j - k] = i + 1` | **记录**：公共前缀长度 $+1$。下一步 $j$ 增大时被第 6 行取出，避免重复比较——**$O(n)$ 的关键** |
| 17 | `return k` | 返回冠军起点，最小循环表示为 $s[k{:}]+s[{:}k]$ |

**关键不变式**：

| 不变量 | 含义 |
|--------|------|
| $k < j$ | 冠军始终在挑战者之前 |
| $0 \le k < n$ | 返回值限定在原串长度内（$s_2$ 后半段是重复，$k$ 不会达到 $n$） |
| $f[d]$ = 公共前缀长度 $-1$ | $s_2[k..]$ 与 $s_2[k{+}d..]$ 的 LCP 长度减 1 |

#### 4.2.9 为什么 `i = f[j - k - 1]` 的下标是 `j-k-1`？

![f 下标 j-k-1 的时序解释](images/booth_f_index_explanation.svg)

$f$ 的下标是**偏移量** $d = j - k$（两条串起点的距离），不是绝对位置 $j$。关键在于**时序**：

- **当前步 $j$**：偏移量 $d = j - k$，但 $f[d]$ **还没算出来**——它正是本步要写入的目标（`f[j-k] = i+1`）。
- **上一步 $j-1$**：偏移量 $d' = (j-1) - k = j - k - 1$，$f[d']$ **已在上一步写好**，存的就是到 $j-1$ 为止的匹配信息。
- 所以读 `f[j-k-1]` = 取上一步存好的匹配长度，再与当前 $s_2[j]$ 比一位，得到本步结果。

> 💡 **一句话**：当前步的 $f[j{-}k]$ 还没算（正要写），只能查上一步的 $f[j{-}k{-}1]$（已存好）。`-1` 的本质是偏移量从 $j{-}k$ 退到 $(j{-}1){-}k$。
>
> 📎 **如果你学过 KMP**：KMP 求 `next[i]` 时查 `next[i-1]`，因为 `next[i]` 还没算；Booth 查 `f[j-k-1]` 是因为 `f[j-k]` 还没算——思路完全一致。没学过 KMP 也不影响理解，核心就是「查上一步的结果来推当前步」。

#### 4.2.10 为什么总复杂度是 $O(n)$？

主循环跑 $2n$ 次，关键是 while 回退的总次数。

**势能分析**：把 $i$ 看作"势能"：

- **势能增加**：每步匹配成功时 `f[j-k] = i+1`，$i$ 最多 $+1$。$2n$ 步最多增加 $2n$。
- **势能消耗**：while 每次回退 `i = f[i]`，$i$ 至少 $-1$。
- **总回退次数** $\le$ 总增加次数 $\le 2n$。

故总比较次数 = 主循环 $2n$ + 回退 $2n$ = $O(n)$，**均摊每字符 $O(1)$**——这正是把朴素 $O(n^2)$ 压到 $O(n)$ 的核心。

### 4.3 Booth 算法图解

#### 核心思想

Booth 的完整推导见 §4.2（倍长 → 擂台赛 → 记忆化 → 失配回退 → 更新冠军）。一句话回顾：把 $s$ 倍长为 $s_2 = s + s$ 后，问题化为在 $s_2$ 中找字典序最小的长度 $n$ 子串起点；冠军 $k$ 与挑战者 $j$ 逐位对齐比较，失配时沿 $f$ 回退复用已匹配前缀，均摊每步 $O(1)$。

![Booth 算法核心思想](images/booth_algorithm_concept.svg)

#### 逐步演算（$s = $ `"baab"`，$n=4$，$s_2 = $ `"baabbaab"`）

$s_2$ 的 4 个候选旋转起点：$k{=}0$→`"baab"`、$k{=}1$→`"aabb"`、$k{=}2$→`"abba"`、$k{=}3$→`"bbaa"`，最小者为 $k{=}1$ 的 `"aabb"`。

| $j$ | $s_2[j]$ | $i$ 入 | 比较 $s_2[j]$ vs $s_2[k{+}i{+}1]$ | 决策 | $k$ | $f[j{-}k]$ | 说明 |
|-----|----------|--------|-----------------------------------|------|-----|-----------|------|
| 1 | `a` | $-1$ | `a` vs `b` ($s_2[0]$) | `a<b` → $k{=}j$ | $0{\to}1$ | $-1$ | 发现更优起点（首位 `a<b` 已定胜负） |
| 2 | `a` | $-1$ | `a` vs `a` ($s_2[1]$) | 匹配 | $1$ | $0$ | 记录匹配长度 |
| 3 | `b` | $0$ | `b` vs `a` ($s_2[2]$) | 失配 `b>a` | $1$ | $-1$ | $i$ 沿 $f$ 回退到 $-1$；$k{=}1$ 仍最优 |
| 4 | `b` | $-1$ | `b` vs `a` ($s_2[1]$) | 失配 `b>a` | $1$ | $-1$ | 起点4以 `b` 开头，不可能小于起点1 |
| 5 | `a` | $-1$ | `a` vs `a` ($s_2[1]$) | 匹配 | $1$ | $0$ | 开始新一轮匹配 |
| 6 | `a` | $0$ | `a` vs `a` ($s_2[2]$) | 匹配 | $1$ | $1$ | 继承上步，已匹配 `"aa"` |
| 7 | `b` | $1$ | `b` vs `b` ($s_2[3]$) | 匹配 | $1$ | $2$ | 已匹配 `"aab"`，确认 $k{=}1$ 最优 |

最终 $k = 1$，$\text{minrot} = s[1{:}] + s[{:}1] = $ `"aabb"`。

![Booth 算法 baab 演算](images/booth_algorithm_trace.svg)

**三个关键步骤**：

1. **$j{=}1$：$k$ 更新**。$s_2[1]{=}$`a` $< s_2[0]{=}$`b`，起点 1 的旋转首位已比起点 0 小，$k$ 从 0 更新为 1。
2. **$j{=}3$：失配但 $k$ 不变**。$s_2[3]{=}$`b` $> s_2[2]{=}$`a`，起点 3 的旋转以 `b` 开头，不可能小于起点 1 的 `a` 开头，$k$ 保持 1。
3. **$j{=}5,6,7$：匹配链确认**。$s_2$ 后半段（第二个 $s$）与 $k{=}1$ 开头的旋转逐位匹配，$f$ 累积记录匹配长度 $0 \to 1 \to 2$，验证 $k{=}1$ 的旋转 `"aabb"` 完整通过。

### 4.4 分组键与计数

用哈希集合维护 `(minrot(E), minrot(O))` 的去重，最终大小即答案。键直接用规范串本身——总长受 $5 \times 10^5$ 约束，内存安全。

## 5. 正确性证明

**引理 1**（等价判定）：$s \sim t \iff |s| = |t|$ 且 $E(s), E(t)$ 互为循环旋转且 $O(s), O(t)$ 互为循环旋转。

**证明**：
- ($\Longrightarrow$) 若 $s \sim t$，则 $t$ 由 $s$ 经 $E$、$O$ 各自一次循环旋转后重组得到，故 $|s|=|t|$，且 $E(t)$ 是 $E(s)$ 的循环旋转、$O(t)$ 是 $O(s)$ 的循环旋转。循环旋转作为关系是对称的，故 $E(s), E(t)$ 互为循环旋转，$O$ 同理。 ∎
- ($\Longleftarrow$) $|s|=|t|$ 蕴含 $|E(s)|=|E(t)|$、$|O(s)|=|O(t)|$。若 $E(s), E(t)$ 互为循环旋转，则存在移位 $a$ 使 $E(s)$ 右移 $a$ 位等于 $E(t)$；同理存在 $b$ 使 $O(s)$ 右移 $b$ 位等于 $O(t)$。对 $s$ 施加该变换即得 $t$，故 $s \sim t$。 ∎

**引理 2**（最小表示刻画）：两串互为循环旋转 $\iff$ 二者的最小循环表示相同。

**证明**：循环旋转是等价关系，所有互为旋转的串构成一个等价类；该类中字典序最小者唯一（字典序为全序），作为类的规范代表。同类 $\iff$ 同代表。 ∎

**定理**：最少分组数 $= \left|\left\{\,(\text{minrot}(E(w)),\ \text{minrot}(O(w))) : w \in \text{words}\,\right\}\right|$。

**证明**：
1. **可行性**（$\le$）：由引理 1、2，键相同的两串满足 $E$、$O$ 均互为循环旋转，故 $s \sim t$。把键相同的串放入同一组，组内任意两串等价，得到一个合法分组，组数 = 不同键数。
2. **最优性**（$\ge$）：键不同的两串，$E$ 或 $O$ 不互为循环旋转，由引理 1 二者不等价，不能同组。故任何合法分组中，不同键的串必须落在不同组，组数 $\ge$ 不同键数。
3. 综合得最少分组数 = 不同键数。 ∎

## 6. 复杂度分析

- **时间复杂度**：$O\!\left(\sum |w|\right)$。每个串抽取 $E$、$O$ 为线性；Booth 算法对 $E$、$O$ 各线性；哈希插入线性。总长 $\sum |w| \le 5 \times 10^5$，整体线性。
- **空间复杂度**：$O\!\left(\sum |w|\right)$。最坏情况所有键互异，需存储全部规范串。

## 7. 参考代码

### C++

```cpp
class Solution {
  public:
    int minGroups(vector<string>& words) {
        auto leastRotation = [](const string& s) -> int {
            int n = s.size();
            if (n == 0) return 0;
            string s2 = s + s;
            vector<int> f(2 * n, -1);
            int k = 0;
            for (int j = 1; j < 2 * n; j++) {
                char sj = s2[j];
                int i = f[j - k - 1];
                while (i != -1 && sj != s2[k + i + 1]) {
                    if (sj < s2[k + i + 1]) k = j - i - 1;
                    i = f[i];
                }
                if (sj != s2[k + i + 1]) {
                    if (sj < s2[k + i + 1]) k = j;
                    f[j - k] = -1;
                } else {
                    f[j - k] = i + 1;
                }
            }
            return k;
        };
        auto canonical = [&](const string& s) -> string {
            if (s.empty()) return s;
            int k = leastRotation(s);
            return s.substr(k) + s.substr(0, k);
        };
        set<pair<string, string>> seen;
        for (const auto& w : words) {
            string E, O;
            E.reserve((w.size() + 1) / 2);
            O.reserve(w.size() / 2);
            for (int i = 0; i < (int)w.size(); i++) {
                if (i % 2 == 0) E.push_back(w[i]);
                else O.push_back(w[i]);
            }
            seen.insert({canonical(E), canonical(O)});
        }
        return (int)seen.size();
    }
};
```

### Python

```python
class Solution:
    def minGroups(self, words: List[str]) -> int:
        def least_rotation_start(s: str) -> int:
            n = len(s)
            if n == 0:
                return 0
            s2 = s + s
            f = [-1] * (2 * n)
            k = 0
            for j in range(1, 2 * n):
                sj = s2[j]
                i = f[j - k - 1]
                while i != -1 and sj != s2[k + i + 1]:
                    if sj < s2[k + i + 1]:
                        k = j - i - 1
                    i = f[i]
                if sj != s2[k + i + 1]:
                    if sj < s2[k + i + 1]:
                        k = j
                    f[j - k] = -1
                else:
                    f[j - k] = i + 1
            return k

        def canonical(s: str) -> str:
            if not s:
                return s
            k = least_rotation_start(s)
            return s[k:] + s[:k]

        seen = set()
        for w in words:
            seen.add((canonical(w[0::2]), canonical(w[1::2])))
        return len(seen)
```

## 8. 边界情况与易错点

1. **单字符 / 空子序列**：$|w|=1$ 时 $E=w$、$O=\varnothing$。Booth 对 $n=0$ 直接返回 0，`canonical("")=""`，无需特判。
2. **E、O 长度不必相等**：$|E|=\lceil n/2 \rceil$、$|O|=\lfloor n/2 \rfloor$。$n$ 为奇数时 $|E|=|O|+1$。等价判定只要求**同侧**长度对应（由 $|s|=|t|$ 保证），不要求 $|E|=|O|$。
3. **E、O 完全独立**：易误以为旋转会「跨奇偶位混合」，实际上偶数位永远只放回偶数位，两环互不干扰，故 $E$、$O$ 分别处理。
4. **最小表示的唯一性**：字典序最小旋转**唯一**（即便串有周期），作键安全。勿用「任意一个旋转」作键，否则同类串可能取到不同代表。
5. **Booth 的 doubling 与下标**：算法在 $s+s$ 上扫描 $2n$ 位，但返回的 $k \in [0, n)$，最终表示取 `s[k:]+s[:k]`，勿越界。
6. **复杂度边界**：单个串可达 $5 \times 10^5$，朴素 $O(n^2)$ 求最小旋转会超时（$2.5 \times 10^{11}$），必须用 Booth 的 $O(n)$。
7. **内存**：总长 $5 \times 10^5$，存储全部规范串键约同量级，安全；切勿对每对串两两比较。
