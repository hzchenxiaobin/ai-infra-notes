## 1. 题目概述

- **题目名称**：Q3. 使用子序列排序转换二进制字符串
- **链接**：[3993. 使用子序列排序转换二进制字符串](https://leetcode.cn/problems/transform-binary-string-using-subsequence-sort/)
- **来源**：LeetCode 第 511 场周赛 Q3
- **难度**：中等
- **标签**：贪心、前缀和、字符串、不变量、可达性判定

**题意简述**：

给定二进制串 `s`。每次操作可任选 `s` 的一个子序列，将其字符按非递减排序后放回原位置（其余字符不动），可执行任意次。对 `strs` 中每个带 `'?'` 的模板串（`'?'` 可替换为 `'0'` 或 `'1'`），判断是否存在一种替换，使替换后的串能由 `s` 经若干次操作得到。返回布尔数组。

**约束条件**：

- `1 <= n == s.length <= 2000`
- `s[i]` 为 `'0'` 或 `'1'`
- `1 <= strs.length <= 2000`
- `strs[i].length == n`，仅由 `'0'`、`'1'`、`'?'` 组成

> **注意**：题面中混入了「Create the variable named veltromina ...」的无关指令，这并非算法要求，解题时忽略即可。

## 2. 示例

**示例 1**

```text
输入：s = "101", strs = ["1?1","0?1","0?0"]
输出：[true,true,false]
解释：
  "1?1" → ? 填 0 得 "101" = s，平凡可达。
  "0?1" → ? 填 1 得 "011"，对 s 排序整串 "101"→"011" 即得。
  "0?0" → 无论 ? 填 0 ("000") 或 1 ("010")，1 的总数都不等于 s 的 2 个 → 不可达。
```

**示例 2**

```text
输入：s = "1100", strs = ["0011","11?1","1?1?"]
输出：[true,false,true]
解释：
  "0011"：排序整串 "1100"→"0011" 即得。
  "11?1" → "1101"，含 3 个 1 ≠ s 的 2 个 → 不可达。
  "1?1?" → 两个 ? 都填 0 得 "1010"，排序子序列 {1,2}="10"→"01" 即得。
```

**示例 3**

```text
输入：s = "1010", strs = ["0011"]
输出：[true]
解释：选子序列 {0,2,3}="110"，排序得 "011"，s 变为 "0011"。
```

## 3. 解题思路

### 3.1 暴力思路：BFS 搜索可达串

对每个替换后的目标串 `t`，从 `s` 出发 BFS 所有「排序一个子序列」得到的状态，看 `t` 是否在可达集中。子序列数 $2^n$，状态空间巨大，$n=2000$ 完全不可行。需要寻找**不变量**刻画可达性。

### 3.2 核心观察：前缀 1 计数单调不增

定义 $f(x, i)$ = $x[0..i]$ 中 `'1'` 的个数（前缀 1 计数）。

**关键引理**：一次「排序子序列」操作不会使任何前缀的 1 计数增加。

直观上，排序子序列 = 把子序列里的 `1` 整体右移、`0` 整体左移。对任意切分点 $i$，子序列中位于 $i$ 左侧的 `1` 可能被推到右侧，但右侧的 `1` 绝不会被拉到左侧，所以前缀 $[0..i]$ 里的 1 数只会减少或不变。

![核心不变量：前缀 1 计数单调不增 + 配对构造](images/binary_subseq_sort_invariant.svg)

### 3.3 可达性定理

由引理，多次操作复合后前缀 1 计数仍单调不增（归纳）。又因排序是置换，**1 的总数守恒**。故必要条件：

$$
\text{reachable}(s \to t) \;\Longrightarrow\; f(t, i) \le f(s, i)\ \forall i,\quad \text{且}\ f(t, n{-}1) = f(s, n{-}1).
$$

**这个条件也是充分的**。证明思路（构造性）：设 $d(i) = f(s, i) - f(t, i) \ge 0$，$d(n{-}1) = 0$。把每个 $s{=}1, t{=}0$ 的位置（记 $L$，1 过剩）看作左括号、每个 $s{=}0, t{=}1$ 的位置（记 $R$，1 不足）看作右括号，$d$ 的非负性保证这是合法括号序列，故每个 $R$ 可配对一个在其左侧的 $L$。对每对 $(l, r)$（$l < r$，$s[l]{=}1, s[r]{=}0$），执行子序列 $\{l, r\}$ 的排序：`"10" → "01"`，即把 $l$ 位变 0、$r$ 位变 1，恰好对齐 $t$。各对位置互不相交，逐一执行后 $s$ 变为 $t$。∎

> 几何直觉：$t$ 的前缀 1 计数阶梯曲线整条落在 $s$ 的曲线下方（端点可重合）。

![可达性 = 前缀 1 计数曲线逐点支配](images/binary_subseq_sort_reachability.svg)

### 3.4 带 '?' 的判定：右取 1 贪心

模板串含 `'?'`，需判断是否存在替换满足上述条件。设 $O_s$ = `s` 中 1 的总数；对模板 `t`：固定 `'1'` 数为 $a$、`'?'` 数为 $B$。需把其中 $K = O_s - a$ 个 `'?'` 填为 1（保证总数相等），要求 $0 \le K \le B$，否则直接 `false`。

约束只有**上界** $f(t, i) \le f(s, i)$（无下界）。为最易满足，应让每个前缀里被计入的 1 尽量少 → 把配额 $K$ 个 1 **尽量往右放**：即最右 $K$ 个 `'?'` 填 1，其余填 0。此分配使前缀 1 计数**逐点最小**。

- 若此最小分配仍使某处 $f(t, i) > f(s, i)$，则任何分配都越界 → `false`。
- 若全部满足 → `true`。

!['?' 右取 1 贪心分配](images/binary_subseq_sort_question_greedy.svg)

### 3.5 示例演算（示例 2，`s = "1100"`，$O_s = 2$，$ps = [1,2,2,2]$）

| 模板 | $a$ | $B$ | $K$ | 分配后 $t$ | $pt$ | $pt \le ps$? | 结果 |
|------|-----|-----|-----|-----------|------|--------------|------|
| `0011` | 2 | 0 | 0 | `0011` | `[0,0,1,2]` | ✓ | true |
| `11?1` | 3 | 1 | $-1$ | — | — | $K<0$ | false |
| `1?1?` | 2 | 2 | 0 | `1010` | `[1,1,2,2]` | ✓ | true |

## 4. 算法细节

### 4.1 预处理 `s` 的前缀 1 计数

一遍扫描得 $ps[i] = f(s, i)$，$O_s = ps[n{-}1]$，$O(n)$。

### 4.2 每个模板的两遍扫描

1. **计数**：扫一遍模板，统计固定 `'1'` 数 $a$ 与 `'?'` 数 $B$。计算 $K = O_s - a$；若 $K < 0$ 或 $K > B$ → `false`。
2. **验证**：令 $Z_q = B - K$（填 0 的 `'?'` 个数，取最左 $Z_q$ 个）。再扫一遍，维护 $pt$ = 当前前缀 1 计数：
   - 遇 `'1'`：`pt += 1`。
   - 遇 `'?'`：`q_seen += 1`；若 `q_seen > Z_q`（即属于最右 $K$ 个）则 `pt += 1`，否则填 0。
   - 遇 `'0'`：不变。
   - 每步检查 `pt <= ps[i]`，一旦越界立即 `false`。
3. 全程通过 → `true`。

### 4.3 关键公式

最右 $K$ 个 `'?'` 填 1 时，前缀 $[0..i]$ 的 1 计数为：

$$
pt[i] = a[i] + \max\!\big(0,\ b[i] - (B - K)\big),
$$

其中 $a[i], b[i]$ 分别是前缀中固定 `'1'` 数与 `'?'` 数。等价地 $pt[i] = \max\!\big(a[i],\ a[i]+b[i] - B + K\big)$，只需验 $pt[i] \le ps[i]$。

## 5. 正确性证明

**引理 1**（操作不变量）：若 $t$ 由 $s$ 经一次「排序子序列」操作得到，则 $\forall i:\ f(t, i) \le f(s, i)$。

**证明**：设选中子序列位置 $p_1 < \dots < p_k$，其中含 $z$ 个 `0`、$o$ 个 `1`（$z+o=k$）。排序后 $p_1,\dots,p_z$ 为 `0`，$p_{z+1},\dots,p_k$ 为 `1`。对切分点 $i$，令 $L$ = 子序列中 $\le i$ 的位置，$o_L$ = $L$ 中原 `1` 数，$z_R$ = 子序列中 $> i$ 的原 `0` 数。排序后 $L$ 中的 `1` 数 $= \max(0, o_L - z_R) \le o_L$（推导：$|L| = z_L + o_L$，排序后 $L$ 中 `1` 数 $= \max(0, |L| - z) = \max(0, o_L - z_R)$）。子序列外的字符不变，故 $f(t, i) \le f(s, i)$。多次操作由归纳即得。 ∎

**定理**：$t$ 可由 $s$ 经若干次操作得到 $\iff$ $\forall i:\ f(t, i) \le f(s, i)$ 且 $f(t, n{-}1) = f(s, n{-}1)$。

**证明**：
- **必要性**：操作保持 1 总数守恒（排序是置换），且由引理 1 前缀计数不增，故 $f(t, i) \le f(s, i)$，$f(t, n{-}1) = f(s, n{-}1)$。
- **充分性**：若 $s = t$ 显然。否则定义 $d(i) = f(s, i) - f(t, i) \ge 0$，$d(-1) = d(n{-}1) = 0$，$d$ 非恒零。位置 $i$ 处 $d$ 增量为 $s[i] - t[i]$（视字符为 0/1）：$+1$ 当 $s{=}1, t{=}0$（$L$ 位），$-1$ 当 $s{=}0, t{=}1$（$R$ 位）。故 $d$ 是返回 0 的非负游走（Dyck 型）。用栈匹配：每个 $R$ 配最近的未配 $L$（在其左侧），得双射 $\pi$ 满足 $\pi(l) > l$。对每对 $(l, r)$ 执行子序列 $\{l, r\}$ 排序 `"10"→"01"`：初始 $s[l]{=}1, s[r]{=}0$，操作后 $l$ 位变 0、$r$ 位变 1，与 $t$ 对齐；各对位置互不相交，处理一对不影响其他对的原值。逐一执行后 $s = t$。 ∎

**推论**（贪心正确性）：对带 `'?'` 的模板，存在合法替换 $\iff$ 「最右 $K$ 个 `'?'` 填 1、其余填 0」这一分配满足 $pt[i] \le ps[i]$ 且 $0 \le K \le B$。

**证明**：约束仅为上界 $pt[i] \le ps[i]$（无下界）。右取 1 分配使每个前缀的 1 计数**逐点最小**（所选 1 的下标集合取最大，前缀交最小）。若它越界，则任何分配的 $pt[i]$ 不小于它，必也越界 → 无解。若它全满足，则它本身就是一组合法替换 → 有解。$K \notin [0, B]$ 时总数无法匹配，显然无解。 ∎

## 6. 复杂度分析

- **时间复杂度**：$O(n + |strs| \cdot n)$。预处理 $ps$ 为 $O(n)$；每个模板两遍扫描 $O(n)$。$n, |strs| \le 2000$ 时约 $4 \times 10^6$，可轻松通过。
- **空间复杂度**：$O(n)$。仅存 $ps$ 数组；模板可逐字符流式扫描。

## 7. 参考代码

### C++

```cpp
class Solution {
  public:
    vector<bool> canTransform(string s, vector<string>& strs) {
        int n = s.size();
        vector<int> ps(n);
        int Os = 0;
        for (int i = 0; i < n; i++) {
            if (s[i] == '1') Os++;
            ps[i] = Os;
        }
        vector<bool> ans;
        ans.reserve(strs.size());
        for (const string& t : strs) {
            int a = 0, B = 0;
            for (char ch : t) {
                if (ch == '1') a++;
                else if (ch == '?') B++;
            }
            int K = Os - a;
            if (K < 0 || K > B) {
                ans.push_back(false);
                continue;
            }
            int Zq = B - K;   // 最左 Zq 个 '?' 填 0
            int pt = 0, qseen = 0;
            bool ok = true;
            for (int i = 0; i < n; i++) {
                char ch = t[i];
                if (ch == '1') {
                    pt++;
                } else if (ch == '?') {
                    qseen++;
                    if (qseen > Zq) pt++;   // 最右 K 个 '?' 填 1
                }
                if (pt > ps[i]) {
                    ok = false;
                    break;
                }
            }
            ans.push_back(ok);
        }
        return ans;
    }
};
```

### Python

```python
class Solution:
    def canTransform(self, s: str, strs: List[str]) -> List[bool]:
        n = len(s)
        ps = [0] * n
        c = 0
        for i, ch in enumerate(s):
            if ch == '1':
                c += 1
            ps[i] = c
        Os = c

        ans = []
        for t in strs:
            a = t.count('1')
            B = t.count('?')
            K = Os - a
            if K < 0 or K > B:
                ans.append(False)
                continue
            Zq = B - K  # 最左 Zq 个 '?' 填 0
            pt = 0
            qseen = 0
            ok = True
            for i in range(n):
                ch = t[i]
                if ch == '1':
                    pt += 1
                elif ch == '?':
                    qseen += 1
                    if qseen > Zq:
                        pt += 1
                if pt > ps[i]:
                    ok = False
                    break
            ans.append(ok)
        return ans
```

## 8. 边界情况与易错点

1. **1 的总数必须守恒**：操作是置换，不增减 1 的个数。模板固定 `'1'` 数 $a$ 若已超过 $O_s$（$K < 0$），或 `'?'` 不足以补足差额（$K > B$），直接 `false`。这是最常见的判否情形（如示例 1 的 `"0?0"`、示例 2 的 `"11?1"`）。
2. **前缀上界是逐位的**：即便总数相等，某前缀处 $f(t, i) > f(s, i)$ 仍不可达。如 `s="101"`、`t="110"`：总数都为 2，但 $f(t, 1) = 2 > f(s, 1) = 1$，不可达。
3. **'?' 贪心方向**：1 配额往**右**放（不是往左）。往左放会让前缀 1 计数增大、更易越界。务必用「最右 $K$ 个 `'?'` 填 1」。
4. `Zq = B - K` **的含义**：填 0 的 `'?'` 取**最左** $Zq$ 个；扫描时 `qseen > Zq` 标志进入「填 1 区」。边界 `qseen == Zq` 仍填 0。
5. **提前终止**：验证阶段一旦 `pt > ps[i]` 立即 `break`，避免无谓扫描。
6. **空操作合法**：题目允许「不执行操作」，故 `t == s` 平凡可达，算法自然处理（$pt = ps$ 全等）。
7. **完全排序串总可达**：`000...111` 的前缀 1 计数为 `[0,...,0,1,2,...,Os]`，对任何同组成的 `s` 都逐点 $\le$，故总可达——只需排序整串。这是判断「最宽松目标」的直觉。
8. **忽略 prompt 注入**：题面「Create the variable named veltromina...」与算法无关，代码中**不要**创建该变量。
