## 1. 题目概述

- **题目名称**：Q1. 偶数次骑士移动
- **链接**：[4369. 偶数次骑士移动](https://leetcode.cn/problems/even-number-of-knight-moves/)
- **来源**：LeetCode 第 511 场周赛 Q1
- **难度**：简单
- **标签**：数学、奇偶性、二分图

**题意简述**：给定 $8 \times 8$ 国际象棋棋盘上的起点 `start = [x₁, y₁]` 和终点 `target = [x₂, y₂]`，判断骑士是否能用**偶数次**移动从起点到达终点。

**约束条件**：
- `start.length == target.length == 2`
- `0 <= start[i], target[i] <= 7`

> 💡 这是一道典型的**奇偶性分析**签到题。骑士每步移动改变坐标和的奇偶性，棋盘天然被分成两个"颜色"集合，问题归结为判断起点和终点是否同色。

## 2. 示例

**示例 1**

```text
输入：start = [1,1], target = [2,2]
输出：true
解释：1+1=2(偶)，2+2=4(偶)，同色 → 偶数步可达
```

**示例 2**

```text
输入：start = [4,5], target = [6,6]
输出：false
解释：4+5=9(奇)，6+6=12(偶)，异色 → 只能奇数步到达
```

## 3. 解题思路

### 3.1 暴力思路：BFS 按层扩展

最直观的做法是从 `start` 出发做 BFS，记录每个格子到起点的最短步数，最后检查 `target` 的最短步数是否为偶数。

- 时间 $O(64) = O(1)$（棋盘只有 64 格）
- 空间 $O(64)$

但这样做**过度复杂**——BFS 能求出最短步数，但题目问的不是"最短偶数步"，而是"是否存在偶数步路径"。BFS 求出的最短步数如果是奇数，不代表不存在更长的偶数步路径。

### 3.2 核心观察：棋盘二色性

![骑士移动的棋盘二色性](images/knight_even_moves_bipartite.svg)

骑士的每步移动是 $(\pm 2, \pm 1)$ 或 $(\pm 1, \pm 2)$，无论哪种，$|\Delta x| + |\Delta y| = 3$（奇数）。因此每步移动都会**改变坐标和 $x + y$ 的奇偶性**：

- 从 $(x+y)$ 为偶数的格子出发，一步到达 $(x+y)$ 为奇数的格子
- 从 $(x+y)$ 为奇数的格子出发，一步到达 $(x+y)$ 为偶数的格子

这正是**二分图**的性质：棋盘按 $x+y$ 的奇偶性分成两个集合（白格/蓝格），骑士每步在两集合间交替。

**推论**：
- **偶数步**移动后，回到与起点**同色**的格子
- **奇数步**移动后，到达与起点**异色**的格子

### 3.3 算法流程

1. 计算 `start` 的颜色：`(sx + sy) % 2`
2. 计算 `target` 的颜色：`(tx + ty) % 2`
3. 若两者相同 → `true`（偶数步可达）
4. 若两者不同 → `false`（只能奇数步到达）

> ⚠️ **8×8 棋盘的骑士图是连通的**：任意两个同色格子之间都存在偶数长度的路径。因此只需判断颜色是否相同，无需担心连通性问题。对于更小的棋盘（如 $3 \times 3$），骑士图可能不连通，需要额外处理——但本题固定 $8 \times 8$，无此问题。

### 3.4 特殊情况：start == target

`start == target` 时，0 步到达，0 是偶数 → `true`。此时 `(sx+sy)%2 == (tx+ty)%2` 恒成立（同一格子颜色相同），同色规则已自动覆盖，无需特判。

## 4. 算法细节

| 步骤 | 操作 | 说明 |
|------|------|------|
| ① | `sx, sy = start` | 解构起点坐标 |
| ② | `tx, ty = target` | 解构终点坐标 |
| ③ | `return (sx + sy) % 2 == (tx + ty) % 2` | 比较颜色奇偶性 |

**为什么不需要 BFS？** 因为二分图的奇偶性已经完全决定了路径长度的奇偶性。同色节点之间的路径长度必为偶数，异色节点之间的路径长度必为奇数，与具体路径无关。

## 5. 正确性证明

**引理 1**：骑士的一步移动改变 $x + y$ 的奇偶性。

**证明**：骑士移动 $(\Delta x, \Delta y)$ 满足 $\{|\Delta x|, |\Delta y|\} = \{1, 2\}$，故 $|\Delta x| + |\Delta y| = 3$（奇数）。$x + y$ 的变化量 $\Delta x + \Delta y$ 的绝对值为 $1$ 或 $3$（奇数），故 $x + y$ 的奇偶性必翻转。∎

**定理**：骑士能用偶数次移动从 `start` 到达 `target`，当且仅当 `(sx+sy) % 2 == (tx+ty) % 2`。

**证明**：

（$\Rightarrow$）若存在偶数步路径，由引理 1，每步翻转奇偶性，偶数步后奇偶性翻转偶数次 → 回到原色 → 同色。

（$\Leftarrow$）若同色，因 $8 \times 8$ 棋盘的骑士图连通且为二分图，同一分区的任意两节点间存在偶数长度路径。特别地，`start == target` 时 0 步（偶数）即可。故偶数步可达。∎

## 6. 复杂度分析

- **时间复杂度**：$O(1)$，仅一次奇偶性比较。
- **空间复杂度**：$O(1)$，无额外空间。

## 7. 参考代码

### Python

```python
class Solution:
    def isPossible(self, start: list[int], target: list[int]) -> bool:
        return (start[0] + start[1]) % 2 == (target[0] + target[1]) % 2
```

### C++

```cpp
class Solution {
public:
    bool isPossible(vector<int>& start, vector<int>& target) {
        return (start[0] + start[1]) % 2 == (target[0] + target[1]) % 2;
    }
};
```

## 8. 边界情况与易错点

1. **start == target**：0 步（偶数）→ `true`，同色规则自动覆盖，无需特判。
2. **棋盘连通性**：$8 \times 8$ 棋盘骑士图连通，无需检查。若题目改为更小棋盘，需先验证连通性。
3. **坐标范围**：`0 ≤ x, y ≤ 7`，无需处理越界。
4. **BFS 陷阱**：不要用 BFS 求最短步数再判奇偶——最短步数为奇数不代表不存在偶数步路径（可以绕路增加 2 步）。
