## 1. 题目概述

- **题目名称**：Q3. 购买最多物品数目 II
- **链接**：[3947. 购买最多物品数目 II](https://leetcode.cn/problems/maximum-number-of-items-from-sale-ii/)
- **来源**：LeetCode 第 504 场周赛 Q3
- **难度**：中等
- **标签**：贪心、数组、排序、堆（优先队列）

**题意简述**：

给定 `n` 种物品，`items[i] = [factor_i, price_i]`，每种**无限供应**，购买总花费不超过 `budget`。免费规则（**与 I 版不同**）：

1. 购买的**每一份**类型 `i` 的物品，至多让你免费获得一份类型 `j`（`j ≠ i` 且 `factor_i` 整除 `factor_j`）；
2. 对每个**有序对** `(i, j)`，无论购买多少件 `i`，至多通过 `i` 免费获得**一次** `j`；
3. 同一个 `j` 可以因**不同的来源** `i` 被免费获得多次。

求能获得的物品最大总数（**购买 + 免费**）。

**约束条件**：

- `1 <= items.length <= 10^5`
- `1 <= factor_i <= items.length`
- `1 <= price_i <= 10^9`
- `1 <= budget <= 10^9`

## 2. 示例

**示例 1**

```text
输入：items = [[1,6],[2,4],[3,5]], budget = 19
输出：5
解释：买 2 件物品 0 和 1 件物品 1，花费 2×6 + 4 = 16 <= 19。
     物品 0 的 factor = 1 可整除所有 factor，两件物品 0 分别免费送物品 1、物品 2 各一件。
     总数 = 3（购买）+ 2（免费）= 5。
```

**示例 2**

```text
输入：items = [[2,8],[1,10],[6,6],[4,12],[5,20],[5,17]], budget = 35
输出：7
解释：买物品 0 两件、物品 1 一件、物品 2 一件，花费 16 + 10 + 6 = 32 <= 35。
     两件物品 0（factor=2）各送一件 factor 为 2 倍数的物品，物品 1（factor=1）再送一件，免费共 3 件。
     总数 = 4（购买）+ 3（免费）= 7。
```

## 3. 解题思路

### 3.1 暴力思路：背包 DP 不可行

先定义与 I 版相同的量：**c_i = 满足 j ≠ i 且 factor_i | factor_j 的物品数**。由 3.2 的观察可知，买 `x_i` 件类型 `i` 的总收益为 `x_i + min(x_i, c_i)`（前 `c_i` 件每件「一拖一」得 2 件，之后每件只得 1 件）。

于是暴力做法是**分组背包**：`dp[b] = 花费不超过 b 时的最大物品数`，对每种类型枚举买 `0, 1, ..., c_i, c_i+1, ...` 件进行转移，复杂度 `O(n · budget)`（甚至更高）。但本题 `budget <= 10^9`、`n <= 10^5`，`O(n·budget)` 高达 `10^14`，完全不可行。**必须利用收益结构找到贪心做法**。

### 3.2 核心观察：每件至多送一份 → 免费数 = min(x_i, c_i)

I 版的规则是「**买过**类型 `i` 即解锁全部 `c_i` 件免费」；II 版改为「**每一份**购买的 `i` 至多送一份免费，且每个目标 `j` 经 `i` 至多送一次」。两个上界同时作用：

- 每件至多送 1 份 ⟹ 免费数 `<= x_i`；
- 有序对 `(i, j)` 至多一次，可作为目标的 `j` 共 `c_i` 个 ⟹ 免费数 `<= c_i`。

而任选 `min(x_i, c_i)` 个可整除目标与购买的物品一一配对即可达到上界，故：

> **买 x_i 件类型 i 获得的免费物品数 = min(x_i, c_i)**。

![I 版与 II 版免费规则对比](images/sale2_rule_vs_i.svg)

### 3.3 槽位分解与阈值贪心

把 `x_i` 拆成两部分：`y_i = min(x_i, c_i)`（**双倍件**，每件都带一份免费）、`z_i = x_i − y_i`（**单倍件**）。总物品数：

```text
总数 = Σ (x_i + min(x_i, c_i)) = Σ (2·y_i + z_i)
```

等价地，每种类型 `i` 提供两类**槽位**：

- **双倍槽**：共 `c_i` 个，花 `p_i` 得 **2** 件；
- **单倍槽**：无限个，花 `p_i` 得 **1** 件。

两个贪心决策：

1. **单倍件只买最便宜类型**（最低价格记为 `p1`）：单倍件收益都是 1 件，当然选单价最低的。
2. **双倍槽按价格升序，当且仅当** `p <= 2·p1` **时拿**：拿一个价格为 `p` 的双倍槽（+2 件），等价于放弃用同样预算能买的 `⌈p/p1⌉` 个单倍件。
   - `p <= 2·p1` 时 `⌈p/p1⌉ <= 2`，件数变化 `Δ = 2 − ⌈p/p1⌉ >= 0`，**不亏**（`p = p1` 时白赚 1 件）；
   - `p > 2·p1` 时拿双倍还不如买单倍（最多持平），**放弃**。

![槽位分解与阈值 2·p1 贪心（槽位取自示例 2）](images/sale2_slots_greedy.svg)

### 3.4 示例演算

示例 2：`items = [[2,8],[1,10],[6,6],[4,12],[5,20],[5,17]]`，`budget = 35`。

先求各类型 `c_i`（factor 为 `factor_i` 倍数的物品总数 − 1）：`c = [2, 5, 0, 0, 1, 1]`。最低价 `p1 = 6`（类型 2），阈值 `2·p1 = 12`。双倍槽按价格升序扫描：

| 步骤 | 类型 (price, c_i) | 决策 | 收获 | 剩余 rem |
|------|-------------------|------|------|----------|
| 1 | 类型 2 (6, 0) | c=0，无槽 | — | 35 |
| 2 | 类型 0 (8, 2) | 8 ≤ 12，拿 t=min(2, ⌊35/8⌋)=2 | +4 | 19 |
| 3 | 类型 1 (10, 5) | 10 ≤ 12，拿 t=min(5, ⌊19/10⌋)=1 | +2 | 9 |
| 4 | 类型 3 (12, 0) | c=0，无槽 | — | 9 |
| 5 | 类型 5 (17, 1) | 17 > 12，**break** | — | 9 |
| 6 | 类型 4 (20, 1) | 不再考虑 | — | 9 |

余钱买单倍：`⌊9 / 6⌋ = 1` 件。总数 = `4 + 2 + 1 = 7` ✓。

## 4. 算法细节

### 4.1 c_i 的计算：freq + 倍数枚举

`factor_i <= n`，开 `freq[v]` 统计每个 factor 的出现次数；再对每个 `v` 枚举其倍数 `m = v, 2v, 3v, ...`，累加出 `mult[v] = Σ freq[m]`（factor 为 `v` 的倍数的物品总数），则：

```text
c_i = mult[factor_i] − 1   （去掉自身）
```

枚举量为 `F/1 + F/2 + ... + F/F = O(F log F)`（调和级数），`F = max factor <= n = 10^5`。

### 4.2 槽位收集与排序

收集所有 `c_i > 0` 的 `(p_i, c_i)` 作为双倍槽，按价格升序排序（`O(n log n)`）。本题槽位静态已知，排序即可；标签中的「堆」对应在线处理的写法（用优先队列按价格维护槽位），与排序在此等价。

### 4.3 扫描与结算

```text
rem = budget, ans = 0
对排序后的每个槽 (p, c)：
    若 p > 2·p1：break             // 后面只会更贵
    t = min(c, rem / p)           // 受槽数与预算双重限制
    ans += 2·t, rem -= t·p
ans += rem / p1                   // 余钱全买最便宜类型的单倍件
```

`p = 2·p1` 恰为持平点（`⌈p/p1⌉ = 2`，拿不拿件数相同），代码取「拿」（条件写成 `p <= 2·p1`），对答案无影响。

## 5. 正确性证明

**引理 1（免费规则定量）**：买 `x_i` 件类型 `i` 获得的免费物品数 `= min(x_i, c_i)`。

**证明**：
- **上界**：规则①「每一份购买的物品至多带来一份免费物品」给出免费数 `<= x_i`；规则②「每个有序对 `(i, j)` 至多免费一次」且满足 `j ≠ i`、`factor_i | factor_j` 的目标 `j` 恰有 `c_i` 个，给出免费数 `<= c_i`。故免费数 `<= min(x_i, c_i)`。
- **可达性**：从 `c_i` 个可送目标中任选 `min(x_i, c_i)` 个，与前 `min(x_i, c_i)` 件购买的物品一一配对赠送，不违反任何规则，达到上界。∎

**引理 2（分解与下界可达）**：任意购买方案对应 `(Y, Z)`（`Y = Σy_i` 为双倍件总数，`Z = Σz_i` 为单倍件总数），其总物品数 `= 2Y + Z`，且花费 `>= cost1(Y) + p1·Z`，其中 `cost1(Y)` 为全部双倍槽中**最便宜的 Y 个**的价格之和，`p1` 为全局最低价格。该下界是可达的。

**证明**：由引理 1，总物品数 `= Σ(x_i + min(x_i, c_i)) = Σ(2·y_i + z_i) = 2Y + Z`。
- **花费下界**：双倍槽的收益恒为 2 件、仅受各类型槽数上限 `c_i` 约束，故购买 `Y` 个双倍槽的最省方式是取**最便宜的 Y 个**（任一方案的双倍槽集合若漏掉某个更便宜的可用槽，替换之花费不增），花费 `>= cost1(Y)`；单倍件每件至少花 `p1`，花费 `>= p1·Z`。
- **可达性**：直接购买最便宜的 `Y` 个双倍槽对应的物品（同一类型买多件时，前 `c_i` 件都是双倍件），再买 `Z` 件最便宜类型（超出其 `c_i` 的部分恰为单倍件）。由引理 1 免费配额全部兑现，总物品数恰为 `2Y + Z`，花费恰为 `cost1(Y) + p1·Z`。∎

**引理 3（交换论证）**：设双倍槽按价格升序考虑，`p1` 为全局最低价。
- 若 `p <= 2·p1`：把方案中用于购买 `⌈p/p1⌉` 个 `p1` 价单倍件的预算（`p1·⌈p/p1⌉ >= p`）改为购买一个 `p` 价双倍槽：花费不增，件数变化 `Δ = 2 − ⌈p/p1⌉ >= 0`（因 `p <= 2·p1` ⟹ `⌈p/p1⌉ <= 2`），故**拿双倍槽不劣**。
- 若 `p > 2·p1`：把方案中的一个 `p` 价双倍槽改回单倍件：释放的 `p` 元可买 `⌊p/p1⌋` 个单倍件，花费不增，件数变化 `⌊p/p1⌋ − 2 >= 0`（因 `p > 2·p1` ⟹ `⌊p/p1⌋ >= 2`），故**不拿不劣**。∎

**定理**：贪心算法（拿光所有 `p <= 2·p1` 且买得起的双倍槽，余钱全买 `p1` 价单倍件）达到最大物品数。

**证明**：由引理 2，任意方案的价值完全由 `(Y, Z)` 决定，且花费下界 `cost1(Y) + p1·Z` 可达，故问题归结为**按价格升序逐个决策每个双倍槽是否拿**。由引理 3，价格为 `p` 的双倍槽当且仅当 `p <= 2·p1` 时值得拿（同价槽内部顺序无关）；槽已按价升序，值得拿的槽恰为一段前缀，扫描时数量受 `c_i` 与剩余预算 `rem/p` 双重限制，取 `min` 即为该类型的最优拿法。剩余预算全部购买单倍件时，选最低价 `p1` 显然最优（单件收益相同、单价最低）。故贪心方案不劣于任意方案，达到最大物品数。∎

## 6. 复杂度分析

- **时间复杂度**：`O(F log F + n log n)`。
  - `freq` 统计 `O(n)`；倍数枚举求 `mult` 为调和级数 `O(F log F)`，`F = max factor <= n`；
  - 槽位排序 `O(n log n)`；扫描结算 `O(n)`。
  - `n <= 10^5`，总量约 `10^6` 级别，轻松通过。
- **空间复杂度**：`O(F)`。`freq` 与 `mult` 数组各开 `F + 1`；槽位列表 `O(n)`。

注意中间计算用 64 位整数：`price, budget <= 10^9`，每件购买至多贡献 2 件物品，答案上限约 `2 × 10^9`，已贴近 32 位有符号整数上限，`2 * p1`、`t * p` 与答案累加都必须用 `long long`。

## 7. 参考代码

### C++

```cpp
class Solution {
public:
    int maximumSaleItems(vector<vector<int>>& items, int budget) {
        int maxf = 0;
        long long p1 = LLONG_MAX;
        for (auto &it : items) {
            maxf = max(maxf, it[0]);
            p1 = min(p1, (long long)it[1]);
        }
        vector<int> freq(maxf + 1, 0), mult(maxf + 1, 0);
        for (auto &it : items) freq[it[0]]++;
        for (int v = 1; v <= maxf; v++)
            for (int m = v; m <= maxf; m += v)
                mult[v] += freq[m];
        vector<pair<long long, int>> slots;
        for (auto &it : items) {
            int ci = mult[it[0]] - 1;
            if (ci > 0) slots.push_back({it[1], ci});
        }
        sort(slots.begin(), slots.end());
        long long rem = budget, ans = 0;
        for (auto &[p, ci] : slots) {
            if (p > 2 * p1) break;
            long long t = min((long long)ci, rem / p);
            ans += 2 * t;
            rem -= t * p;
        }
        ans += rem / p1;
        return (int)ans;
    }
};
```

### Python

```python
class Solution:
    def maximumSaleItems(self, items: List[List[int]], budget: int) -> int:
        maxf = max(f for f, _ in items)
        freq = [0] * (maxf + 1)
        for f, _ in items:
            freq[f] += 1
        mult = [0] * (maxf + 1)
        for v in range(1, maxf + 1):
            for m in range(v, maxf + 1, v):
                mult[v] += freq[m]
        p1 = min(p for _, p in items)
        slots = []
        for f, p in items:
            ci = mult[f] - 1
            if ci > 0:
                slots.append((p, ci))
        slots.sort()
        rem, ans = budget, 0
        for p, ci in slots:
            if p > 2 * p1:
                break
            t = min(ci, rem // p)
            ans += 2 * t
            rem -= t * p
        ans += rem // p1
        return ans
```

## 8. 边界情况与易错点

1. `budget < p1`**（一件都买不起）**：此时所有槽 `t = min(c, rem/p) = 0`，最后 `rem / p1 = 0`，自然返回 `0`，无需特判，但要确认逻辑覆盖。

2. `c_i = 0` **的类型**：只提供单倍槽，即只有「最便宜类型」的身份有意义（如示例 2 的类型 2）；不要误以为它能送免费物品。

3. `p = 2·p1` **是持平点**：`⌈p/p1⌉ = 2`，拿与不拿件数相同。代码用 `p <= 2·p1` 取「拿」，对答案无影响；真正关键的是 `p > 2·p1` 时必须不拿。

4. **遇** `p > 2·p1` **应** `break` **而非** `continue`：槽位已按价格升序，后面只会更贵，直接退出；若未排序则必须逐个判断，不能提前退出。

5. `int` **溢出**：`price, budget <= 10^9`，答案上限约 `2 × 10^9`，`2 * p1`、`t * p` 与 `ans` 累加都必须用 `long long`（Python 无此问题）。

6. **与 I 版规则的区别**：I 版是「**买过**类型 `i` 即送全套 `c_i` 件」（免费数与购买件数解耦，背包可解）；II 版是「**每件**至多送一份」（免费数 `= min(x_i, c_i)`，槽位贪心）。两题模型完全不同，不要混用。

7. `factor_i <= n`：`freq` / `mult` 数组开 `n + 1` 足够；`c_i = mult[factor_i] − 1` 中的 `−1` 是去掉自身（`j ≠ i`），不要忘记。

## 9. 相关题目与扩展

- 本场周赛 Q2《购买最多物品数目 I》：背包版对照。I 版规则为「买过即送全套」，免费数与购买件数解耦，可用背包 DP；II 版则必须槽位分解 + 阈值贪心。对比两题能深刻理解**规则差异如何决定算法形态**。
- [630. 课程表 III](https://leetcode.cn/problems/course-schedule-iii/)：贪心 + 堆的经典——先全拿，冲突时再用大根堆反悔替换。本题的「按价排序 + 阈值取舍」是同类「先排再选」思想。
- [871. 最低加油次数](https://leetcode.cn/problems/minimum-number-of-refueling-stops/)：贪心补能，顺序扫描 + 大根堆选取最优补给点。

**延伸思考**：

- 若双倍槽收益改为「花 `p` 得 `k` 件」（每买一件类型 `i` 最多免费带 `k − 1` 件），阈值如何变化？拿一个 `p` 价槽放弃 `⌈p/p1⌉` 个单倍件，件数变化 `Δ = k − ⌈p/p1⌉ >= 0` ⟺ `p <= k·p1`，故**阈值变为** `k·p1`。本题即 `k = 2` 的特例（阈值 `2·p1`）。
- 若同一有序对 `(i, j)` 的赠送次数上限从 1 改为 `r`，则免费数变为 `min(x_i, r·c_i)`，等价于每个类型有 `r·c_i` 个双倍槽，算法框架不变。
