## 1. 题目概述

- **题目名称**：Q1. 计算数字频率得分
- **链接**：[3945. 计算数字频率得分](https://leetcode.cn/problems/digit-frequency-score/)
- **来源**：LeetCode 第 504 场周赛 Q1
- **难度**：简单
- **标签**：哈希表、数学

**题意简述**：

给定整数 `n`。对其十进制表示中每个**不同数字** `d`，统计 `d` 出现的次数 `freq(d)`；`n` 的**得分**定义为所有不同数字的 `d * freq(d)` 之和。返回该得分。

**约束条件**：

- `1 <= n <= 10^9`（十进制至多 10 位）

---

## 2. 示例

**示例 1**

```text
输入：n = 122
输出：5
解释：数字 1 出现 1 次，贡献 1 × 1 = 1；数字 2 出现 2 次，贡献 2 × 2 = 4。总得分 1 + 4 = 5。
```

**示例 2**

```text
输入：n = 101
输出：2
解释：数字 0 出现 1 次，贡献 0 × 1 = 0；数字 1 出现 2 次，贡献 1 × 2 = 2。总得分 0 + 2 = 2。
```

**示例 3（补充）**

```text
输入：n = 1000000000
输出：1
解释：数字 1 出现 1 次，贡献 1；数字 0 出现 9 次，贡献恒为 0。总得分 1。
```

## 3. 解题思路

### 3.1 快速切入（签到题的最短路径）

题面出现「对每个**不同数字**统计出现次数」，立刻想到**频率计数**模型；而数字只有 `0 ~ 9` 共 10 种，键空间有限，用**长度为 10 的计数数组**即可，甚至不需要哈希表。

剩下的唯一工作是把 `n` 拆成十进制位：经典的 `n % 10` 取末位、`n /= 10` 右移一位，循环到 `n == 0` 为止。从读题到写完代码就是一条直线。

### 3.2 核心观察：计数数组 + 加权求和

![数字频率得分的三步计算流程](images/digit_freq_score_concept.svg)

如上图，整个算法只有三步：

1. **数位分解**：把 `n` 逐位拆开；
2. **频率统计**：`cnt[d]` 记录数字 `d` 出现的次数；
3. **加权求和**：答案 `= Σ d · cnt[d]`（`d` 取遍 `0 ~ 9`）。

**上界观察**：`n ≤ 10^9` 至多 10 位，每位贡献至多 `9`，答案 `≤ 9 × 10 = 90`，`int` 绝不溢出。

### 3.3 示例演算：n = 101

![示例演算：n = 101 的频率统计与贡献](images/digit_freq_score_example101.svg)

- `cnt[0] = 1`，贡献 `0 × 1 = 0`（0 乘任何数都是 0，**无需特判**）；
- `cnt[1] = 2`，贡献 `1 × 2 = 2`；
- 得分 `= 0 + 2 = 2`。

## 4. 算法细节

1. **计数数组**：`int cnt[10] = {0};`，下标即数字 `d`，值即频率。题目标签虽含「哈希表」，但键空间只有 10，定长数组比 `unordered_map` 更快也更简单。

2. **数位分解**：

```text
while n > 0:
    cnt[n % 10] += 1   // 取出末位并计数
    n /= 10            // 右移一位
```

约束保证 `n ≥ 1`，循环至少执行一次，天然安全。**若题目允许** `n = 0`，此循环一次都不执行，会漏掉 `cnt[0] = 1`，需要特判——本题不涉及。

3. **加权求和**：`ans = Σ_{d=0}^{9} d * cnt[d]`。`d = 0` 的项恒为 0，照常累加即可，不必跳过、不必特判。

4. **溢出分析**：`ans ≤ 9 × 10 = 90`，`int` 绰绰有余，无需 `long long`。

---

## 5. 正确性证明

**引理 1**：数位分解循环结束后，对任意 `d ∈ [0, 9]`，`cnt[d]` 恰等于数字 `d` 在 `n` 的十进制表示中出现的次数。

**证明**：对循环次数归纳。循环不变式：设已处理 `n` 的最低 `k` 位，则对每个 `d`，`cnt[d]` 等于 `d` 在这最低 `k` 位中的出现次数。

- **初始化**：`k = 0`，尚未处理任何位，`cnt` 全为 0，不变式成立。
- **保持**：第 `k + 1` 次迭代取出末位 `d* = n % 10`（即原数从低位起第 `k + 1` 位），执行 `cnt[d*] += 1`，恰好把这一位计入；随后 `n /= 10`，已处理位数变为 `k + 1`，不变式保持。
- **终止**：`n = 0` 时所有位均已处理，`cnt[d]` 即 `d` 在整个十进制表示中的出现次数。∎

**定理**：算法返回的值等于题目定义的得分。

**证明**：由引理 1，返回时 `cnt[d] = freq(d)`。算法返回 `Σ_{d=0}^{9} d · cnt[d] = Σ_{d=0}^{9} d · freq(d)`；对未出现的数字 `freq(d) = 0`，对应项为 0，不影响求和。故该和式正是题目定义的「所有不同数字 `d` 的 `d · freq(d)` 之和」。∎

---

## 6. 复杂度分析

- **时间复杂度**：`O(L)`，`L` 为 `n` 的十进制位数。数位分解循环 `L` 次，加权求和固定 10 次；由 `L ≤ 10`（`n ≤ 10^9`），实际运行时间可视为 `O(1)`。
- **空间复杂度**：`O(1)`。仅使用定长 10 的计数数组，与输入规模无关。

## 7. 参考代码

### C++

```cpp
class Solution {
public:
    int digitFrequencyScore(int n) {
        int cnt[10] = {0};
        while (n > 0) { cnt[n % 10]++; n /= 10; }
        int ans = 0;
        for (int d = 0; d < 10; d++) ans += d * cnt[d];
        return ans;
    }
};
```

### Python

```python
class Solution:
    def digitFrequencyScore(self, n: int) -> int:
        cnt = [0] * 10
        while n > 0:
            cnt[n % 10] += 1
            n //= 10
        return sum(d * cnt[d] for d in range(10))
```

## 8. 边界情况与易错点

1. `n = 10^9 = 1000000000`：含 9 个 `0`。`0` 的贡献恒为 0，答案仅由数字 `1` 贡献，结果为 1。可用来检验代码是否正确处理大量 `0`。
2. **单位数** `n ∈ [1, 9]`：循环只执行一次，`cnt[n] = 1`，答案恰为 `n` 本身。
3. `d = 0` **不要画蛇添足地特判**：`0 × freq(0) = 0`，统一累加即可；多写一个「跳过 0」的分支虽然结果相同，但属于多余逻辑。
4. `n = 0` **的隐患**：本题约束 `n ≥ 1`，`while n > 0` 安全。若换一道题允许 `n = 0`，循环一次都不执行会得到 `cnt` 全 0，而正确答案应含 `cnt[0] = 1`，必须特判。
5. **不要过度设计**：无需转字符串、无需排序、无需 `unordered_map`；「键空间有限 → 定长数组」就是最短路径。
6. **不会溢出**：答案上界 90，`int` 足够（用 `long long` 写也不错，但并非必需）。
