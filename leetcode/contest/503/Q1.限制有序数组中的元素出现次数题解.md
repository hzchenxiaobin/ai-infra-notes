# 限制有序数组中的元素出现次数

## 1. 题目概述

- **题目名称**：Q1. 限制有序数组中的元素出现次数
- **链接**：[3940. 限制有序数组中的元素出现次数](https://leetcode.cn/problems/limit-occurrences-in-sorted-array/)
- **来源**：LeetCode 第 503 场周赛 Q1
- **难度**：简单
- **标签**：数组、双指针

**题意简述**：

给定一个**非递减排序**的整数数组 `nums` 和一个整数 `k`。要求返回一个新数组，使其中每个**不同元素**最多出现 `k` 次，同时保持元素的**相对顺序**不变。等价于：对每个值，只保留它在前 `k` 次出现的位置，跳过第 `k+1` 次及之后的重复。

**约束条件**：

- `1 <= nums.length <= 100`
- `1 <= nums[i] <= 100`
- `nums` 已按**非递减**顺序排序
- `1 <= k <= nums.length`

> 💡 **进阶要求**：用 `O(1)` 额外空间**原地**完成（输出复用输入数组）。

## 2. 示例

**示例 1**

```text
输入：nums = [1,1,1,2,2,3], k = 2
输出：[1,1,2,2,3]
解释：1 出现 3 次，保留前 2 次；2、3 出现均不超过 2 次，全部保留。
```

**示例 2**

```text
输入：nums = [1,1,1,1], k = 2
输出：[1,1]
解释：1 出现 4 次，仅保留前 2 次。
```

**示例 3**

```text
输入：nums = [1,2,3,4], k = 1
输出：[1,2,3,4]
解释：所有元素互不相同，k=1 时每个元素恰出现 1 次，全部保留。
```

**示例 4**

```text
输入：nums = [1,1,1,2,2,2,3,3,3], k = 1
输出：[1,2,3]
解释：k=1 时每个值只保留首次出现。
```

## 3. 解题思路

### 3.1 暴力思路

用哈希表统计每个值已保留的次数，顺序遍历 `nums`：若当前值的已保留次数 `< k`，则追加到答案并计数 `+1`；否则跳过。

- 时间 `O(n)`，空间 `O(n)`（答案数组 + 哈希表）。
- 能通过，但**未利用「数组已排序」**这一关键性质，且空间非 `O(1)`，不满足进阶。

### 3.2 核心观察：排序 + 双指针「回看 k 位」

![双指针读写扫描与逐步演算](images/limit_occurrences_two_pointers.svg)

由于数组**已排序**，相同元素必然**连续**出现。判断「当前值是否已写满 k 次」时，无需哈希表，只需**回看写入指针** `w` **前** `k` **位**：

```text
若 w < k                 → 写入区不足 k 个，直接写
若 nums[r] != nums[w-k]  → 写入区末尾 k 个不全是 nums[r]，写入不会超额
若 nums[r] == nums[w-k]  → 写入区末尾 k 个全是 nums[r]（已写满 k 次），跳过
```

> 💡 **关键观察**：因为输入升序，`nums[w-k]` 与 `nums[r]` 相等当且仅当写入区末尾 `k` 个全部等于 `nums[r]`——这正是「该值已写满 k 次」的充要条件。无需显式维护计数。

### 3.3 算法流程

1. 设写入指针 `w = 0`，读指针 `r` 从 `0` 到 `n-1` 扫描。
2. 对每个 `nums[r]`：
   - 若 `w < k` 或 `nums[r] != nums[w-k]`：`nums[w] = nums[r]`，`w++`。
   - 否则跳过（已写满 k 次）。
3. 截断 `nums` 到长度 `w`，返回。

### 3.4 示例演算

`nums = [1,1,1,2,2,3]`, `k = 2`：

| r | nums[r] | w | 判定 (`w<k` 或 `nums[r]!=nums[w-k]`) | 动作 | ans |
|---|---------|---|---------------------------------------|------|-----|
| 0 | 1 | 0 | `w<k` (0<2) ✓ | 写入 | [1] |
| 1 | 1 | 1 | `w<k` (1<2) ✓ | 写入 | [1,1] |
| 2 | 1 | 2 | `nums[0]==1` ✗ | 跳过 | [1,1] |
| 3 | 2 | 2 | `nums[0]==1 != 2` ✓ | 写入 | [1,1,2] |
| 4 | 2 | 3 | `nums[1]==1 != 2` ✓ | 写入 | [1,1,2,2] |
| 5 | 3 | 4 | `nums[2]==2 != 3` ✓ | 写入 | [1,1,2,2,3] |

最终 `w = 5`，返回 `[1,1,2,2,3]` ✓

## 4. 算法细节

### 4.1 为什么「回看 k 位」等价于「计数」

设写入区 `nums[0..w-1]` 始终满足「每个不同值出现 `<= k` 次」这一不变式。考虑是否写入 `nums[r]`：

- 写入区中与 `nums[r]` 相同的值必然**集中在末尾**（输入升序 → 写入区也升序 → 同值连续）。
- 末尾连续段长度恰为「`nums[r]` 已写入次数」。该长度 `>= k` ⇔ `nums[w-k] == nums[r]`（末尾 k 个都等于它）。
- 因此 `nums[r] != nums[w-k]` ⇔ 已写入次数 `< k` ⇔ 可继续写入。

### 4.2 为什么可以原地（in-place）安全

读指针 `r` 始终 `>=` 写指针 `w`（每次写入 `w` 最多 `+1`，而 `r` 每次 `+1`，初值均 0）。因此 `nums[w] = nums[r]` 永远不会覆盖尚未读取的数据。原地复用输入数组即可。

### 4.3 两种等价写法

- **回看 k 位**（上面主解）：无需额外状态，最简洁。
- **显式计数**：维护 `cur`（当前值）与 `count`（已写入次数），遇到新值则重置。逻辑等价，但状态更多、易写错。

## 5. 正确性证明

**引理 1（写入区不变式）**：在算法任意时刻，写入区 `nums[0..w-1]` 中每个不同值出现次数 `<= k`，且为「输入按序保留前 k 次出现」的前缀。

**证明**：对 `r` 归纳。
- 初始 `w=0`，写入区空，平凡成立。
- 设处理 `nums[r]` 前不变式成立。若跳过（`nums[r] == nums[w-k]`），由 §4.1 末尾已有 `k` 个 `nums[r]`，跳过后写入区不变，仍满足「保留前 k 次」（第 `k+1` 次被丢弃）。若写入，则写入前 `nums[r]` 出现次数 `< k`，写入后 `<= k`，且这次是按序的下一处保留。故不变式保持。 ∎

**引理 2（顺序保持）**：写入区元素的相对顺序与输入一致。

**证明**：`r` 单调递增，写入顺序即读取顺序，故相对顺序不变。 ∎

**定理**：算法返回的数组满足「每个不同元素最多出现 k 次，且保持相对顺序」。

**证明**：由引理 1，最终写入区每个值出现 `<= k` 次，且恰为「保留各值前 k 次出现」的结果（多余重复被丢弃）；由引理 2，顺序保持。截断到 `w` 即得答案。 ∎

**完备性**：对每个值，其前 `k` 次出现必然全部被写入（前 k 次中每次判定时末尾同值段长度 `< k`），第 `k+1` 次起必然被跳过。故结果唯一且为「最长合法子序列」。 ∎

---

## 6. 复杂度分析

- **时间复杂度**：`O(n)`。读指针 `r` 单趟扫描，每次 `O(1)` 判定。
- **空间复杂度**：`O(1)` 额外空间（原地复用 `nums`，仅用 `w`、`r` 两个指针变量）。返回时不计输出所占空间。

## 7. 参考代码

### C++

```cpp
class Solution {
public:
    vector<int> limitOccurrencesInSortedArray(vector<int>& nums, int k) {
        int w = 0;
        for (int r = 0; r < (int)nums.size(); r++) {
            // w < k：写入区不足 k 个，必然可写
            // nums[r] != nums[w-k]：末尾 k 个不全是 nums[r]，写入不超额
            if (w < k || nums[r] != nums[w - k]) {
                nums[w++] = nums[r];
            }
        }
        nums.resize(w);
        return nums;
    }
};
```

### Python

```python
class Solution:
    def limitOccurrencesInSortedArray(self, nums: List[int], k: int) -> List[int]:
        w = 0
        for r in range(len(nums)):
            if w < k or nums[r] != nums[w - k]:
                nums[w] = nums[r]
                w += 1
        return nums[:w]
```

> 💡 **显式计数写法（等价，便于理解）**：维护 `cur` 与 `count`，遇新值重置 `count=0`；`count < k` 则写入并 `count++`，否则跳过。两法等价，回看 k 位更简洁。

## 8. 边界情况与易错点

1. `k = 1`：退化为「去除所有重复元素」，等价于 [26. 删除有序数组中的重复项]。条件 `w < k` 即 `w < 1` 即 `w == 0`（首元素必写），后续靠 `nums[r] != nums[w-1]`。
2. `k >= n`：任何值都不会超额，所有元素保留，返回原数组。`w < k` 恒真（直到写满 n），全部写入。
3. **全部相同**：如 `[5,5,5,5]`, `k=2` → `[5,5]`。前两个因 `w<k` 写入，后两个因 `nums[r]==nums[w-k]` 跳过。
4. **全部不同**：如 `[1,2,3]`，任意 `k` 都原样返回。
5. **回看下标** `w-k` **的合法性**：仅在 `w >= k` 时才访问 `nums[w-k]`，因 `w < k` 已短路，不会越界。切勿写成先取 `nums[w-k]` 再判断。
6. **原地写入不可提前 return**：必须扫完整段，因为「可写」条件依赖 `w`，而 `w` 只随写入增长。
7. **返回新数组 vs 原地**：本题接口返回 `vector<int>`，`resize(w)` 后返回引用即可；若要求不修改输入，则需另开 `O(n)` 空间。

**延伸思考**：

- 若数组**未排序**，要求每个值最多保留 k 次且保持相对顺序？则失去「同值连续」性质，必须用哈希表计数，`O(n)` 时间 / `O(distinct)` 空间。
- 若改为「保留每个值的**最后** k 次出现」？可逆向扫描 + 回看，或正向扫描后从尾部计数保留。
- 若 `k` 对每个值不同（给出 `limit[v]`）？需哈希表存各值上限，丧失 `O(1)` 空间优势。
