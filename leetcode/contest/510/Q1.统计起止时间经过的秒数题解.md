# 统计起止时间经过的秒数

## 1. 题目概述

- **题目名称**：Q1. 统计起止时间经过的秒数
- **链接**：[3986. 统计起止时间经过的秒数](https://leetcode.cn/problems/number-of-elapsed-seconds-between-two-times/)
- **来源**：LeetCode 第 510 场周赛 Q1
- **难度**：简单
- **标签**：数学、字符串、模拟

**题意简述**：

给定两个有效时间字符串 `startTime` 和 `endTime`（格式 `"HH:MM:SS"`），返回从 `startTime` 到 `endTime` 经过的秒数（包含两个端点）。`endTime` 不早于 `startTime`。

**约束条件**：

- `startTime.length == 8`，`endTime.length == 8`
- `00 <= HH <= 23`，`00 <= MM <= 59`，`00 <= SS <= 59`
- `endTime` 不早于 `startTime`

## 2. 示例

**示例 1**

```text
输入：startTime = "01:00:00", endTime = "01:00:25"
输出：25
解释：endTime 比 startTime 晚 25 秒。
```

**示例 2**

```text
输入：startTime = "12:34:56", endTime = "13:00:00"
输出：1504
解释：25 分 4 秒 = 25×60 + 4 = 1504 秒。
```

---

## 3. 解题思路

### 3.1 暴力思路

签到题。把 `HH:MM:SS` 转成从 `00:00:00` 起的总秒数，做差即可。

### 3.2 核心观察：转秒数 + 做差

![时间转秒数 + 做差](images/elapsed_seconds_concept.svg)

**转换公式**：

```text
total_seconds("HH:MM:SS") = HH × 3600 + MM × 60 + SS
```

**答案**：`total_seconds(endTime) - total_seconds(startTime)`

### 3.3 解析字符串

`"HH:MM:SS"` 固定格式，可直接按位置截取：

```text
HH = int(s[0:2])
MM = int(s[3:5])
SS = int(s[6:8])
```

或用 `split(":")` 分割。

### 3.4 示例演算

`startTime = "12:34:56"`, `endTime = "13:00:00"`：

| 时间 | HH | MM | SS | 总秒数 |
|------|----|----|----|--------|
| 12:34:56 | 12 | 34 | 56 | 12×3600 + 34×60 + 56 = 43200 + 2040 + 56 = 45296 |
| 13:00:00 | 13 | 0 | 0 | 13×3600 = 46800 |

差 = 46800 − 45296 = 1504 ✓

![示例演算：12:34:56→13:00:00 转秒分解](images/elapsed_seconds_example.svg)

## 4. 算法细节

1. **固定格式解析**：`"HH:MM:SS"` 长度固定 8，`s[0:2]` 为时、`s[3:5]` 为分、`s[6:8]` 为秒。也可 `split(":")`。
2. **转秒公式**：`HH*3600 + MM*60 + SS`，将三段统一到秒单位。
3. **做差**：`endTime` 秒数 − `startTime` 秒数，约束保证非负。

---

## 5. 正确性证明

**引理 1**：`total_seconds("HH:MM:SS") = HH*3600 + MM*60 + SS` 正确表示从 00:00:00 起的秒数。

**证明**：1 小时 = 3600 秒，1 分 = 60 秒，秒数即 SS。三段累加得总秒数。∎

**定理**：算法返回 `startTime` 到 `endTime` 经过的秒数。

**证明**：由引理 1，两时间均转为从 00:00:00 起的秒数。差值 `total(end) - total(start)` 即两时刻间的秒数差。约束 `endTime >= startTime` 保证差非负。∎

---

## 6. 复杂度分析

- **时间复杂度**：`O(1)`。字符串长度固定 8，解析和计算均为常数。
- **空间复杂度**：`O(1)`。

## 7. 参考代码

### C++

```cpp
class Solution {
public:
    int secondsBetweenTimes(string startTime, string endTime) {
        auto toSeconds = [](const string& s) {
            int h = stoi(s.substr(0, 2));
            int m = stoi(s.substr(3, 2));
            int sec = stoi(s.substr(6, 2));
            return h * 3600 + m * 60 + sec;
        };
        return toSeconds(endTime) - toSeconds(startTime);
    }
};
```

### Python

```python
class Solution:
    def secondsBetweenTimes(self, startTime: str, endTime: str) -> int:
        def to_seconds(s: str) -> int:
            h, m, sec = map(int, s.split(":"))
            return h * 3600 + m * 60 + sec
        return to_seconds(endTime) - to_seconds(startTime)
```

## 8. 边界情况与易错点

1. **同一时间**：`startTime == endTime`，答案为 0。
2. **跨小时**：如 `"12:59:59"` 到 `"13:00:00"`，差 1 秒。
3. **端点包含**：题目说"包含两个端点"，但实际就是 `endTime - startTime`（差值本身已包含端点间的所有秒）。
4. **不跨天**：约束保证 `endTime >= startTime`，无需处理跨天（如 23:00 到 01:00）。
