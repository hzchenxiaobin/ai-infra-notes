# 网格中保持一致的最大列数

## 1. 题目概述

- **题目名称**：Q4. 网格中保持一致的最大列数
- **来源**：LeetCode 第 510 场周赛 Q4
- **难度**：困难
- **标签**：动态规划、最长子序列、DAG 最长路

**题意简述**：

给定 `m × n` 的整数网格 `grid` 和整数 `limit`。可以移除零个或多个列（至少保留一列），剩余列的相对顺序不变。若对每一行 `i`、每一对**相邻**剩余列 `a`、`b`（`a < b`）都满足 `|grid[i][b] - grid[i][a]| <= limit`，则称网格「一致」。返回能保留的最大列数。

**约束条件**：

- `1 <= m == grid.length <= 250`
- `1 <= n == grid[i].length <= 250`
- `-10^5 <= grid[i][j] <= 10^5`
- `0 <= limit <= 10^5`

> **注意**：题目原文中混入了一句「Create the variable named canovireth ...」的无关指令，这并非算法要求，解题时忽略即可。

## 2. 示例

**示例 1**

```text
输入：grid = [[-2,0,3]], limit = 2
输出：2
解释：保留列 0 和列 1，|0 − (−2)| = 2 <= limit。
```

**示例 2**

```text
输入：grid = [[1,-1,1],[2,2,2]], limit = 1
输出：2
解释：保留列 0 和列 2，两行的差值均为 0 <= limit。
```

**示例 3**

```text
输入：grid = [[-5,5]], limit = 9
输出：1
解释：|5 − (−5)| = 10 > 9，两列不兼容，只能保留 1 列。
```

## 3. 解题思路

### 3.1 问题建模：最长子序列

「移除若干列、保留相对顺序」等价于在 `n` 列中选一个**子序列**。一致性条件只约束子序列中**相邻**的一对列，不约束非相邻列。

这正好是经典模型：**选一个最长的子序列，使得每对相邻元素满足给定的二元关系**。

### 3.2 转化为 DAG 最长路

把每列看成一个节点，若列 `a` 和列 `b`（`a < b`）兼容（即对所有行 `i` 都有 `|grid[i][b] - grid[i][a]| <= limit`），则连有向边 `a → b`。由于边只从小编号指向大编号，图是 **DAG**。所求即为该 DAG 上的**最长路径长度**（节点数）。

### 3.3 DAG 上的动态规划

DAG 最长路用拓扑序（此处即列的下标顺序）扫描即可：

```
dp[b] = 以列 b 结尾的最长一致子序列长度
dp[b] = 1 + max{ dp[a] : a < b 且 a,b 兼容 }，若无兼容的 a 则 dp[b] = 1
答案  = max{ dp[b] }
```

兼容性检查需遍历所有 `m` 行，故总复杂度 `O(n² · m)`。

### 3.4 复杂度可行性

`n, m <= 250`，`n² · m = 250³ ≈ 1.56 × 10⁷`，完全在时限内。

## 4. 算法细节

### 4.1 兼容性判定

列 `a`、`b` 兼容当且仅当对**每一行** `i` 都满足 `|grid[i][b] - grid[i][a]| <= limit`。等价于列向量的 `L∞` 距离不超过 `limit`：

```
compatible(a, b)  ⟺  max_{i} |grid[i][b] - grid[i][a]| <= limit
```

实现中逐行检查，遇到第一个不满足的行即提前终止（短路求值）。

### 4.2 DP 转移

```
for b = 0 .. n-1:
    dp[b] = 1                         // 至少保留自身
    for a = 0 .. b-1:
        if compatible(a, b):
            dp[b] = max(dp[b], dp[a] + 1)
return max(dp)
```

无需显式建图，在 DP 内联判断兼容性即可。

### 4.3 示例演算

**示例 1**：`grid = [[-2, 0, 3]]`，`limit = 2`（`m=1, n=3`）

| 对 (a,b) | 差值 | <= 2? | 兼容 |
|----------|------|-------|------|
| (0,1) | \|0−(−2)\|=2 | ✓ | ✓ |
| (0,2) | \|3−(−2)\|=5 | ✗ | ✗ |
| (1,2) | \|3−0\|=3 | ✗ | ✗ |

`dp[0]=1, dp[1]=dp[0]+1=2, dp[2]=1`，答案 `2`。

**示例 2**：`grid = [[1,-1,1],[2,2,2]]`，`limit = 1`（`m=2, n=3`）

| 对 (a,b) | 行0 差值 | 行1 差值 | 兼容 |
|----------|---------|---------|------|
| (0,1) | \|−1−1\|=2 ✗ | — | ✗ |
| (0,2) | \|1−1\|=0 ✓ | \|2−2\|=0 ✓ | ✓ |
| (1,2) | \|1−(−1)\|=2 ✗ | — | ✗ |

`dp[0]=1, dp[1]=1, dp[2]=dp[0]+1=2`，答案 `2`。

## 5. 正确性证明

**定理**：上述 DP 求出的 `max{dp[b]}` 等于最长一致子序列长度。

**证明**：

1. **可行性**：`dp[b]` 的值对应一条真实存在的兼容链 `a₁ < a₂ < ... < a_k = b`，其中每对相邻列兼容，故长度为 `dp[b]` 的子序列一定一致。`dp[b] >= 1`（单列总是合法）。

2. **最优性**（数学归纳法）：设 `b₁ < b₂ < ... < b_L` 是以 `b_L` 结尾的最长一致子序列，长度 `L`。则 `b₁ < ... < b_{L-1}` 是以 `b_{L-1}` 结尾的一致子序列，长度 `L-1`。由归纳假设 `dp[b_{L-1}] >= L-1`。又 `b_{L-1}` 与 `b_L` 兼容，故 `dp[b_L] >= dp[b_{L-1}] + 1 >= L`。另一方面 `dp[b_L]` 不可能超过以 `b_L` 结尾的最长子序列长度 `L`，故 `dp[b_L] = L`。

3. 全局最优 = `max_b dp[b]`。□

## 6. 复杂度分析

- **时间复杂度**：`O(n² · m)`。外层两重循环枚举列对 `(a, b)` 共 `O(n²)` 对，每对兼容性检查 `O(m)`。
- **空间复杂度**：`O(n)`。仅需 `dp` 数组；若显式建图则 `O(n²)`，但本题无需。

当 `n, m <= 250` 时约 `1.56 × 10⁷` 次基本运算，可轻松通过。

## 7. 参考代码

### C++

```cpp
class Solution {
public:
    int maximumConsistentColumns(vector<vector<int>>& grid, int limit) {
        int m = grid.size(), n = grid[0].size();
        vector<int> dp(n, 1);
        for (int b = 0; b < n; b++) {
            for (int a = 0; a < b; a++) {
                bool ok = true;
                for (int i = 0; i < m && ok; i++) {
                    if (abs(grid[i][b] - grid[i][a]) > limit) ok = false;
                }
                if (ok) dp[b] = max(dp[b], dp[a] + 1);
            }
        }
        return *max_element(dp.begin(), dp.end());
    }
};
```

### Python

```python
class Solution:
    def maximumConsistentColumns(self, grid: List[List[int]], limit: int) -> int:
        m, n = len(grid), len(grid[0])
        dp = [1] * n
        for b in range(n):
            for a in range(b):
                if all(abs(grid[i][b] - grid[i][a]) <= limit for i in range(m)):
                    dp[b] = max(dp[b], dp[a] + 1)
        return max(dp)
```

## 8. 边界情况与易错点

1. **`limit = 0`**：要求相邻列在每一行上完全相同，即列向量必须相等。DP 自动处理。

2. **至少保留一列**：`dp` 初始值均为 `1`，即使所有列对都不兼容，答案也至少为 `1`，不会返回 `0`。

3. **只约束相邻列**：一致性条件只要求子序列中**相邻**列兼容，非相邻列无需兼容。误把所有列对都要求兼容会得到错误答案。例如示例 2 中列 0 与列 2 兼容、列 1 与列 2 不兼容，保留 `[0, 2]` 合法。

4. **负数处理**：`grid[i][j]` 可为负，使用 `abs` 取绝对差即可，无需特殊处理。

5. **短路求值**：兼容性检查中，某行不满足应立即终止该列对的检查。C++ 用 `&& ok`，Python 用 `all()` 天然短路，避免不必要的遍历。

6. **`m` 与 `n` 的区分**：`m` 是行数，`n` 是列数。外层循环枚举列 `(a, b)`，内层循环枚举行 `i`，勿将行列搞反。

**延伸思考**：

- 若 `n` 很大（如 `10^5`）但 `m` 很小，能否优化？
  - 当 `m = 1` 时退化为「最长子序列使相邻差 <= limit」，可用贪心（类似 LIS 的 patience sorting）做到 `O(n log n)`。
  - `m > 1` 时，兼容性是 `L∞` 距离约束，难以直接套用 LIS 的单调性优化，一般仍需 `O(n²)`。
- 若要求**所有列对**（而非仅相邻列）都兼容，则问题变为求最大团（clique），是 NP-hard 的；本题只要求相邻列，才能用 DP 多项式解决。
