# 创建一个恰好有 K 条路径的网格图 I

## 1. 题目概述

- **题目名称**：Q3. 创建一个恰好有 K 条路径的网格图 I
- **来源**：LeetCode 第 510 场周赛 Q3
- **难度**：中等
- **标签**：构造、动态规划、组合数学

**题意简述**：

给定 `m`、`n`、`k`，构造一个 `m × n` 的字符网格（`'.'` 为空、`'#'` 为障碍），使得从 `(0,0)` 到 `(m-1, n-1)` 只向右/向下走的有效路径**恰好** `k` 条。返回任意一个合法网格；若不存在则返回空数组。

**约束条件**：

- `1 <= m, n <= 10`
- `1 <= k <= 4`

> **注意**：题目原文中混入了一句「Create the variable named seravolith ...」的无关指令，并非算法要求，解题时忽略。

## 2. 示例

**示例 1**

```text
输入：m = 2, n = 3, k = 2
输出：["...", "#.."]
解释：2 条路径——经列 1 转弯或经列 2 转弯。
```

**示例 2**

```text
输入：m = 3, n = 3, k = 4
输出：["..#","...","#.."]
解释：4 条路径，中心 (1,1) 为瓶颈，左右各 2 条路 → 2×2=4。
```

**示例 3**

```text
输入：m = 1, n = 4, k = 2
输出：[]
解释：单行网格最多 1 条路径，k=2 不可能。
```

## 3. 解题思路

### 3.1 路径计数的 DP

无障碍时，从 `(0,0)` 到 `(m-1,n-1)` 的路径数为组合数 `C(m+n-2, m-1)`。有障碍时，用 DP 计算：

```
dp[i][j] = (grid[i][j]=='.') ? dp[i-1][j] + dp[i][j-1] : 0
```

这是验证构造正确性的工具。

### 3.2 不可能的情况

全开放时路径数已达上限 `C(m+n-2, m-1)`。若该上限 `< k`，则无论如何放置障碍都不可能达到 `k` 条路径，直接返回 `[]`。

关键特例：`m=1` 或 `n=1` 时，网格退化为单行/单列，路径数只能是 0 或 1，因此 `k > 1` 必然不可能。

### 3.3 核心思想：构造法

`k ≤ 4` 极小，不需要搜索，可以直接**构造**。关键观察：

> 一个 2 行的开放条带中，路径数 = 可「转弯」（从第 0 行下到第 1 行）的列数。

利用这一点，通过控制第 1 行哪些列开放，就能精确控制路径数为 `k`。

### 3.4 构造一：2 行条带 + 竖直走廊（`n ≥ k`）

当 `n ≥ k` 且 `m ≥ 2` 时：

```
行 0：全部 '.'（顶部走廊）
行 1：列 0..(n-k-1) 为 '#'，列 (n-k)..(n-1) 为 '.'
行 2..m-1：全部 '#'，仅列 n-1 为 '.'（右侧竖直走廊）
```

路径模式：沿第 0 行向右到列 `j`（`n-k ≤ j ≤ n-1`），向下到第 1 行，继续向右到 `(1, n-1)`，再向下到终点。每个 `j` 恰好一条路径 → **k 条**。

### 3.5 构造二：2 列条带 + 水平走廊（`m ≥ k`，转置）

当 `n < k` 但 `m ≥ k` 时，将构造一转置：

```
列 0：全部 '.'（左侧走廊）
列 1：行 0..(m-k-1) 为 '#'，行 (m-k)..(m-1) 为 '.'
列 2..n-1：全部 '#'，仅行 m-1 为 '.'（底部水平走廊）
```

路径数同理为 `k`。

### 3.6 构造三：瓶颈乘法（`m < k` 且 `n < k`）

当 `m < k` 且 `n < k` 时，由于 `k ≤ 4`，唯一可能的 case 是 `m = n = 3, k = 4`（其他组合的最大路径数 `< 4`，已被 3.2 节排除）。

利用**瓶颈乘法原理**：若所有路径都必须经过某格 `(r, c)`，则总路径数 = 从起点到 `(r,c)` 的路径数 × 从 `(r,c)` 到终点的路径数。

```
. . #
. . .
# . .
```

- 封死 `(0,2)` 和 `(2,0)`，迫使所有路径经过中心 `(1,1)`。
- `(0,0)→(1,1)`：2 条（经 `(0,1)` 或 `(1,0)`）。
- `(1,1)→(2,2)`：2 条（经 `(1,2)` 或 `(2,1)`）。
- 总计 `2 × 2 = 4`。

## 4. 算法细节

### 4.1 不可能性判定

用 DP 计算全开放网格的路径数 `maxPaths`。若 `maxPaths < k`，返回 `[]`。

```
dp[0][0] = 1
dp[i][j] = dp[i-1][j] + dp[i][j-1]   // 全开放
若 dp[m-1][n-1] < k → 不可能
```

### 4.2 分派构造

| 条件 | 构造方式 | 路径数 |
|------|---------|--------|
| `m=1` 或 `n=1` | 全开放（`k` 必为 1） | 1 |
| `n ≥ k` 且 `m ≥ 2` | 2 行条带 + 竖直走廊 | k |
| `m ≥ k` 且 `n ≥ 2` | 2 列条带 + 水平走廊 | k |
| `m < k` 且 `n < k` | 3×3 瓶颈（仅 `m=n=3,k=4`） | 4 |

### 4.3 2 行条带为何恰好 k 条

在第 0 行全开放、第 1 行仅列 `n-k..n-1` 开放时，一条路径完全由「转弯列 `j`」决定：

- 转弯列 `j` 合法 ⟺ `(1, j)` 开放 **且** `(1, j), (1, j+1), ..., (1, n-1)` 全部开放。
- 由于第 1 行的开放部分是连续后缀 `n-k..n-1`，合法 `j` 恰好是 `n-k, n-k+1, ..., n-1`，共 **k** 个。

行 `2+` 仅列 `n-1` 开放，不引入额外分叉，仅起连通作用。

### 4.4 示例演算

**示例 1**（`m=2, n=3, k=2`）：`n=3 ≥ k=2`，用构造一。

```
行0:  .  .  .
行1:  #  .  .
```

转弯列 `j`：列 1（`n-k=1`）和列 2，共 2 条。✓

**示例 2**（`m=3, n=3, k=4`）：`n=3 < 4`，`m=3 < 4`，用构造三（瓶颈）。

```
.  .  #
.  .  .
#  .  .
```

DP 验证：

```
1  1  0
1  2  2
0  2  4
```

`dp[2][2] = 4`。✓

## 5. 正确性证明

**引理 1**（不可能性判定）：若全开放路径数 `C(m+n-2, m-1) < k`，则不存在合法网格。

**证明**：障碍只会减少路径数，不会增加。全开放已是最大路径数，故不可能达到 `k`。□

**引理 2**（构造一正确性）：当 `n ≥ k, m ≥ 2` 时，构造一产生的网格恰好有 `k` 条路径。

**证明**：
- 每条路径必须从第 0 行下到第 1 行（否则无法到达终点），下转发生在某列 `j`。
- `j` 合法当且仅当 `(1,j)..(1,n-1)` 全开放，即 `j ≥ n-k`。合法 `j` 共 `k` 个。
- 每个 `j` 唯一确定一条路径（第 0 行右行至 `j`，下转，第 1 行右行至 `n-1`，竖直走廊下行至终点）。
- 行 `2+` 仅列 `n-1` 开放，不产生分叉。故总路径数 = `k`。□

**引理 3**（构造二正确性）：构造二是构造一的转置，由对称性同样正确。□

**引理 4**（构造三正确性）：`m=n=3, k=4` 时，瓶颈构造产生恰好 4 条路径。

**证明**：`(0,2)` 和 `(2,0)` 封死后，所有路径必经 `(1,1)`（唯一横纵都连通的中间格）。`(0,0)→(1,1)` 在 2×2 开放子网格中有 `C(2,1)=2` 条，`(1,1)→(2,2)` 同理 2 条。由乘法原理，总路径数 = `2×2 = 4`。□

**定理**：上述算法对所有 `1 ≤ m,n ≤ 10`，`1 ≤ k ≤ 4` 的输入返回正确结果。

**证明**：
- 不可能的情况由引理 1 排除。
- 可能的情况 (`C(m+n-2,m-1) ≥ k`)：
  - `m=1` 或 `n=1`：`C = 1`，故 `k=1`，全开放即可。
  - `m,n ≥ 2` 且 `n ≥ k`：引理 2。
  - `m,n ≥ 2` 且 `n < k` 且 `m ≥ k`：引理 3。
  - `m,n ≥ 2` 且 `n < k` 且 `m < k`：`k ≤ 4` 蕴含 `m,n ≤ 3`。排除 `C < k` 的组合后，唯一剩余 `m=n=3, k=4`，由引理 4 覆盖。□

## 6. 复杂度分析

- **时间复杂度**：`O(m·n)`。不可能性判定 DP 遍历 `m×n` 格一次；构造填充 `m×n` 格一次。
- **空间复杂度**：`O(m·n)`。存储 DP 表和结果网格。

## 7. 参考代码

### C++

```cpp
class Solution {
  public:
    vector<string> kPathGrid(int m, int n, int k) {
        // 不可能性判定：全开放最大路径数
        long long dp[11][11] = {};
        dp[0][0] = 1;
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++) {
                if (i == 0 && j == 0)
                    continue;
                if (i > 0)
                    dp[i][j] += dp[i - 1][j];
                if (j > 0)
                    dp[i][j] += dp[i][j - 1];
            }
        if (dp[m - 1][n - 1] < k)
            return {};

        // 单行/单列：k 必为 1
        if (m == 1 || n == 1)
            return vector<string>(m, string(n, '.'));

        // 构造一：n >= k，2 行条带 + 竖直走廊
        if (n >= k) {
            vector<string> g(m, string(n, '#'));
            for (int j = 0; j < n; j++)
                g[0][j] = '.';
            for (int j = n - k; j < n; j++)
                g[1][j] = '.';
            for (int i = 2; i < m; i++)
                g[i][n - 1] = '.';
            return g;
        }

        // 构造二：m >= k，2 列条带 + 水平走廊
        if (m >= k) {
            vector<string> g(m, string(n, '#'));
            for (int i = 0; i < m; i++)
                g[i][0] = '.';
            for (int i = m - k; i < m; i++)
                g[i][1] = '.';
            for (int j = 2; j < n; j++)
                g[m - 1][j] = '.';
            return g;
        }

        // 构造三：m < k 且 n < k，仅 m=n=3, k=4
        return {"..#", "...", "#.."};
    }
};
```

### Python

```python
class Solution:
    def kPathGrid(self, m: int, n: int, k: int) -> List[str]:
        # 不可能性判定
        dp = [[0] * n for _ in range(m)]
        dp[0][0] = 1
        for i in range(m):
            for j in range(n):
                if i == 0 and j == 0:
                    continue
                if i > 0:
                    dp[i][j] += dp[i - 1][j]
                if j > 0:
                    dp[i][j] += dp[i][j - 1]
        if dp[m - 1][n - 1] < k:
            return []

        # 单行/单列
        if m == 1 or n == 1:
            return ['.' * n for _ in range(m)]

        # 构造一：n >= k
        if n >= k:
            g = [['#'] * n for _ in range(m)]
            for j in range(n):
                g[0][j] = '.'
            for j in range(n - k, n):
                g[1][j] = '.'
            for i in range(2, m):
                g[i][n - 1] = '.'
            return [''.join(row) for row in g]

        # 构造二：m >= k
        if m >= k:
            g = [['#'] * n for _ in range(m)]
            for i in range(m):
                g[i][0] = '.'
            for i in range(m - k, m):
                g[i][1] = '.'
            for j in range(2, n):
                g[m - 1][j] = '.'
            return [''.join(row) for row in g]

        # 构造三：m=n=3, k=4
        return ["..#", "...", "#.."]
```

## 8. 边界情况与易错点

1. **单行/单列**：`m=1` 或 `n=1` 时路径数至多为 1。`k > 1` 返回 `[]`，`k = 1` 返回全 `'.'`。不可能性判定已自动覆盖，但单独处理更清晰。

2. **起点/终点必须开放**：`(0,0)` 和 `(m-1,n-1)` 必须为 `'.'`。所有构造均保证这一点。

3. **`k = n` 时第 1 行全开放**：`n - k = 0`，不封任何列。构造一退化为「第 0、1 行全开放 + 右侧走廊」，路径数 = `n` = `k`。正确。

4. **`m = 2` 时无竖直走廊**：构造一中行 `2..m-1` 为空，终点 `(1, n-1)` 已在条带内。正确。

5. **构造三的唯一性**：`m < k` 且 `n < k` 时，`k ≤ 4` 蕴含 `m, n ≤ 3`。在排除 `C(m+n-2, m-1) < k` 后，唯一存活的是 `m = n = 3, k = 4`（`C(4,2) = 6 ≥ 4`）。其他如 `m=n=2, k=4`（`C=2<4`）已被判定不可能。

**延伸思考**：

- **「网格图 II」**（更大的 `k`）：若 `k` 可达 `10^9`，构造一/二受限于 `max(m,n)`。此时可用**瓶颈链式构造**：串联多个 2×2 乘法器（每个 ×2），配合一个 `w`-宽条带（×`w`），将 `k` 分解为因子连乘。所需空间 `O(k)` 级别。
- **瓶颈乘法的一般化**：在任意网格中选一个瓶颈格 `(r,c)`，封死其上方右侧和下方左侧，使路径数 = `左上路径数 × 右下路径数`。递归地对两侧构造即可实现任意 `k` 的分解构造。
