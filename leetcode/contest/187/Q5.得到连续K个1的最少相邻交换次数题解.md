## 1. 题目概述

- **题目名称**：Q5. 得到连续 K 个 1 的最少相邻交换次数（187 场 Q3 同类题）
- **链接**：[1703. 得到连续 K 个 1 的最少相邻交换次数](https://leetcode.cn/problems/minimum-adjacent-swaps-for-k-consecutive-ones/)
- **难度**：困难
- **标签**：贪心、数组、前缀和、滑动窗口、逆序对

**题意简述**：
给定一个只含 `0` 和 `1` 的数组 `nums` 和一个整数 `k`。每次操作可以交换相邻两个元素。求让数组中出现 **连续 `k` 个 1** 所需的最少相邻交换次数。

**约束条件**：
- `1 <= nums.length <= 10^5`
- `nums[i]` 为 `0` 或 `1`
- `1 <= k <= sum(nums)`

## 2. 示例

**示例 1**

```text
输入：nums = [1,0,0,1,0,1], k = 2
输出：1
解释：把最后两个 1 相邻即可，例如 [1,0,0,0,1,1]。
```

**示例 2**

```text
输入：nums = [1,0,0,0,0,0,1,1], k = 3
输出：5
解释：把最左边的 1 移到右侧，得到 [0,0,0,0,0,1,1,1]。
```

**示例 3**

```text
输入：nums = [1,1,0,1], k = 2
输出：0
```

## 3. 解题思路

### 3.1 暴力思路
枚举最终连续 `k` 个 `1` 出现的位置，再枚举用哪 `k` 个 `1` 去填充，模拟冒泡交换计数。复杂度 `O(n^2)`，不可行。

### 3.2 核心观察：逆序对 = 相邻交换次数
把数组看成由 `1` 和 `0` 组成的序列。要让某 `k` 个 `1` 聚到一起，只能通过让它们与挡在中间的 `0` 交换。每发生一次相邻交换，恰好消除一个“选中的 `1` 在 `0` 左边且需要越过它”的逆序对。

因此，对固定的一组 `k` 个 `1`，把它们变成连续块所需的交换次数 = 它们需要跨越的 `0` 的总数，也就是这组 `1` 与相关 `0` 之间的逆序对数。

![选定窗口与目标位置](images/lc1703_window_move.svg)

### 3.3 核心观察：平移后的问题变成“到中位数的最小距离”
设所有 `1` 的位置为 `pos[0], pos[1], ..., pos[m-1]`。考虑一个窗口 `pos[i..i+k-1]`，最终把它们放到连续的 `k` 个位置上，设起始位置为 `t`，则第 `j` 个 `1` 的目标位置是 `t + j`。

交换次数为：

$$\sum_{j=0}^{k-1} \big| pos[i+j] - (t+j) \big| = \sum_{j=0}^{k-1} \big| \big(pos[i+j] - j\big) - t \big|$$

令 $q_j = pos[i+j] - j$。问题转化为：在数组 $q$ 中选一个数 $t$，使 $\sum |q_j - t|$ 最小。经典结论是：**取 $t$ 为 $q$ 的中位数时最小**。

![平移后求中位数](images/lc1703_median_transform.svg)

### 3.4 算法流程
1. 收集所有 `1` 的下标到数组 `pos`。
2. 对 `pos` 求前缀和 `prefix`。
3. 枚举窗口左端点 `i`，窗口覆盖 `pos[i..i+k-1]`，中点 `mid = i + k/2`。
4. 用前缀和在 `O(1)` 时间内计算该窗口的代价：
   - 左半部分到中位数的距离之和；
   - 右半部分到中位数的距离之和。
5. 所有窗口中取最小值。

### 3.5 示例演算
`nums = [1,0,0,1,0,1], k = 2`：
- `pos = [0, 3, 5]`。
- 窗口 `[0,3]`：`q = [0, 2]`，中位数 `1`，代价 `|0-1|+|2-1| = 2`。
- 窗口 `[3,5]`：`q = [3, 4]`，中位数 `3`，代价 `|3-3|+|4-3| = 1`。
- 最小值为 `1`。

## 4. 算法细节

- 设窗口覆盖 `pos[l..r]`，`mid = l + k // 2`，`median = pos[mid] - (mid - l)`。
- 左半段和 = `median * (mid - l) - (prefix[mid-1] - prefix[l-1])`。
- 右半段和 = `(prefix[r] - prefix[mid]) - median * (r - mid)`。
- 注意 `l = 0` 时 `prefix[-1] = 0`。

## 5. 正确性证明

**引理 1**：存在最优解，使得最终连续的 `k` 个 `1` 由原数组中某 `k` 个 `1` 按原有相对顺序移动得到。

**证明**：相邻交换不会改变相同元素的相对顺序。最终连续块里有 `k` 个 `1`，它们必然来自原数组中的 `k` 个 `1`，且顺序不变。 ∎

**引理 2**：对于固定的 `k` 个 `1`，其原始位置为 `p_0 < p_1 < ... < p_{k-1}`，把它们移动到连续位置所需的最少交换次数为 $\min_t \sum_{j=0}^{k-1} |q_j - t|$，其中 $q_j = p_j - j$。

**证明**：由于相对顺序不变，第 `j` 个 `1` 的目标位置必为 `t + j`。交换次数等于每个 `1` 移动的距离之和，即 $\sum |p_j - (t+j)| = \sum |(p_j - j) - t|$。 ∎

**引理 3**：$\min_t \sum_{j=0}^{k-1} |q_j - t|$ 在 $t$ 取 $q$ 的中位数时达到最小。

**证明**：对绝对值函数求和，左侧导数为负、右侧导数为正，导数由负变正的点即为中位数，故中位数处取最小。 ∎

**定理**：上述滑动窗口 + 前缀和算法返回最少相邻交换次数。

**证明**：由引理 1，只需枚举原数组中 `k` 个 `1` 组成的窗口。对每个窗口，引理 2 给出代价表达式，引理 3 给出最优目标位置。算法用前缀和在 `O(1)` 时间内求出每个窗口的最优代价，再取全局最小，故正确。 ∎

## 6. 复杂度分析

- **时间复杂度**：`O(n)`，其中 `n = nums.length`。收集 `1` 的位置 `O(n)`，枚举窗口 `O(m) <= O(n)`。
- **空间复杂度**：`O(n)`，存储 `pos` 与前缀和数组。

## 7. 参考代码

### C++

```cpp
class Solution {
public:
    int minMoves(vector<int>& nums, int k) {
        vector<int> pos;
        for (int i = 0; i < (int)nums.size(); ++i)
            if (nums[i] == 1) pos.push_back(i);

        int m = pos.size();
        vector<long long> prefix(m + 1, 0);
        for (int i = 0; i < m; ++i)
            prefix[i + 1] = prefix[i] + pos[i];

        long long ans = LLONG_MAX;
        for (int i = 0; i + k <= m; ++i) {
            int l = i, r = i + k - 1;
            int mid = l + k / 2;
            long long median = pos[mid] - (mid - l);

            long long left = median * (mid - l) - (prefix[mid] - prefix[l]);
            long long right = (prefix[r + 1] - prefix[mid + 1]) - median * (r - mid);
            ans = min(ans, left + right);
        }
        return (int)ans;
    }
};
```

### Python

```python
class Solution:
    def minMoves(self, nums: list[int], k: int) -> int:
        pos = [i for i, x in enumerate(nums) if x == 1]
        m = len(pos)
        prefix = [0] * (m + 1)
        for i, p in enumerate(pos):
            prefix[i + 1] = prefix[i] + p

        ans = float('inf')
        for i in range(m - k + 1):
            l, r = i, i + k - 1
            mid = l + k // 2
            median = pos[mid] - (mid - l)

            left = median * (mid - l) - (prefix[mid] - prefix[l])
            right = (prefix[r + 1] - prefix[mid + 1]) - median * (r - mid)
            ans = min(ans, left + right)
        return int(ans)
```

## 8. 边界情况与易错点

1. **`k = 1`**：单个 `1` 已经满足条件，答案为 `0`。
2. **`k = sum(nums)`**：所有 `1` 都要聚在一起，窗口只有一个，算法仍正确。
3. **下标计算**：`median = pos[mid] - (mid - l)` 不要忘记减去偏移 `(mid - l)`。
4. **前缀和开 long long**：`n` 最大 `10^5`，但位置差最大也是 `10^5`，乘积可达 `10^10`，需 64 位整数。
5. **中位数对偶数 `k` 的处理**：取 `mid = l + k // 2`（上中位数或下中位数均可，任意中位数都最优）。
