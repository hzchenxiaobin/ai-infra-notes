## 1. 题目概述

- **题目名称**：Q1. 移动后的最大曼哈顿距离
- **链接**：[3968. 移动后的最大曼哈顿距离](https://leetcode.cn/problems/maximum-manhattan-distance-after-all-moves/)
- **来源**：LeetCode 第 507 场周赛 Q1
- **难度**：中等
- **标签**：数学、字符串、计数

**题意简述**：

给定由 `'U'`、`'D'`、`'L'`、`'R'`、`'_'` 组成的字符串 `moves`，从原点 `(0,0)` 出发逐字符移动。`'_'` 可独立替换为任意方向。返回执行完所有移动后距离原点的**最大曼哈顿距离** `|x| + |y|`。

**约束条件**：

- `1 <= moves.length <= 10^5`
- `moves` 仅由 `'U'`、`'D'`、`'L'`、`'R'`、`'_'` 组成

## 2. 示例

**示例 1**

```text
输入：moves = "L_D_"
输出：4
解释：L→(-1,0)，_→D→(-1,-1)，D→(-1,-2)，_→L→(-2,-2)
     曼哈顿距离 = |−2| + |−2| = 4
```

**示例 2**

```text
输入：moves = "U_R"
输出：3
解释：U→(0,1)，_→U→(0,2)，R→(1,2)
     曼哈顿距离 = |1| + |2| = 3
```

---

## 3. 解题思路

### 3.1 暴力思路

每个 `_` 有 4 种选择，共 `4^(num_)` 种组合，枚举取最大。指数级，不可行。

### 3.2 核心观察：拆分固定部分与自由部分

![曼哈顿距离拆分：固定 + 自由](images/max_manhattan_concept.svg)

**关键观察**：将移动分为**固定移动**（U/D/L/R）和**自由移动**（`_`）。

- 固定移动决定净位移 `(net_x, net_y)`，贡献 `|net_x| + |net_y|`
- 每个自由移动 `_` 可**独立**选择方向，使曼哈顿距离**增加 1**

**为什么每个** `_` **恰好增加 1？**

曼哈顿距离 `|x| + |y|` 中，`|x|` 和 `|y|` 是独立可增的。无论当前 `(x, y)` 在哪：

- 若 `net_x >= 0`：选 `R` 使 `x` 增加 1，`|x|` 增加 1
- 若 `net_x < 0`：选 `L` 使 `x` 减少 1，`|x|` 增加 1
- 同理可对 `y` 轴操作

因此每个 `_` 都能使 `|x| + |y|` 增加 1，且选择互不干扰。

![_ 的方向选择：按净位移符号使 |x| 或 |y| 增 1](images/max_manhattan_underscore.svg)

### 3.3 公式

```text
答案 = |net_x| + |net_y| + num_underscore
```

其中：
- `net_x = count('R') - count('L')`
- `net_y = count('U') - count('D')`
- `num_underscore = count('_')`

### 3.4 示例演算

`moves = "L_D_"`：

| 字符 | net_x | net_y | 说明 |
|------|-------|-------|------|
| L | -1 | 0 | |
| _ | -1 | 0 | 自由，待分配 |
| D | -1 | -1 | |
| _ | -1 | -1 | 自由，待分配 |

固定：`|net_x| + |net_y| = 1 + 1 = 2`，`num_ = 2`
答案 = `2 + 2 = 4` ✓

## 4. 算法细节

1. **净位移统计**：遍历 `moves`，`U/D` 改变 `y`，`R/L` 改变 `x`，`_` 计入 `under`。
2. **绝对值处理**：最终 `|net_x| + |net_y|`，C++ 用 `abs`，Python 内置 `abs`。
3. `_` **的方向选择**：无需显式选择，公式 `|net_x| + |net_y| + under` 隐含每个 `_` 沿净位移符号方向移动使 `|x|` 或 `|y|` 增 1。

---

## 5. 正确性证明

**引理 1**：每个 `_` 可使曼哈顿距离 `|x|+|y|` 增加 1，且不减少固定部分的贡献。

**证明**：固定移动后的净位移为 `(net_x, net_y)`。对任一 `_`：若 `net_x >= 0` 选 `R` 使 `x` 增 1，`|x|` 增 1（`x >= 0` 时 `|x|` 随 `x` 单调增）；若 `net_x < 0` 选 `L` 使 `x` 减 1，`|x|` 增 1。各 `_` 独立选择，互不干扰，故总增加 `under`。∎

**定理**：最大曼哈顿距离 = `|net_x| + |net_y| + num_underscore`。

**证明**：由引理 1，`under` 个 `_` 各增 1，下界 `|net_x|+|net_y|+under` 可达。又每个 `_` 单步移动至多改变 `|x|+|y|` 一个单位，故该上界不可超越。公式成立。∎

---

## 6. 复杂度分析

- **时间复杂度**：`O(n)`。遍历一次统计各方向计数。
- **空间复杂度**：`O(1)`。仅用几个计数变量。

## 7. 参考代码

### C++

```cpp
class Solution {
public:
    int maxDistance(string moves) {
        int x = 0, y = 0, under = 0;
        for (char c : moves) {
            if (c == 'U') y++;
            else if (c == 'D') y--;
            else if (c == 'R') x++;
            else if (c == 'L') x--;
            else under++;
        }
        return abs(x) + abs(y) + under;
    }
};
```

### Python

```python
class Solution:
    def maxDistance(self, moves: str) -> int:
        x = y = under = 0
        for c in moves:
            if c == 'U': y += 1
            elif c == 'D': y -= 1
            elif c == 'R': x += 1
            elif c == 'L': x -= 1
            else: under += 1
        return abs(x) + abs(y) + under
```

## 8. 边界情况与易错点

1. **全** `_`：固定部分为 0，答案 = `num_`，每个 `_` 贡献 1。
2. **固定移动互相抵消**：如 `"UDLR"`，`net_x=0, net_y=0`，无 `_` 则答案为 0。
3. `_` **选择方向取决于净位移符号**：若 `net_x >= 0` 选 `R`，`net_x < 0` 选 `L`。但公式 `|net_x| + under` 已隐含此选择，无需显式计算。
